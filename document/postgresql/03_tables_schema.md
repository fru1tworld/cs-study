# PostgreSQL 테이블 및 스키마

> 공식 문서: https://www.postgresql.org/docs/current/ddl.html

---

## 테이블 생성

### 기본 구문

```sql
CREATE TABLE table_name (
    column_name data_type [constraints],
    ...
);
```

### 예제

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    username VARCHAR(50) NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 제약조건 (Constraints)

### 1. NOT NULL

NULL 값을 허용하지 않습니다.

```sql
CREATE TABLE products (
    name VARCHAR(100) NOT NULL,
    price NUMERIC(10, 2) NOT NULL
);
```

### 2. UNIQUE

중복 값을 허용하지 않습니다.

```sql
CREATE TABLE users (
    email VARCHAR(255) UNIQUE,
    -- 또는 테이블 레벨
    UNIQUE (email)
);

-- 복합 유니크
CREATE TABLE user_roles (
    user_id INTEGER,
    role_id INTEGER,
    UNIQUE (user_id, role_id)
);
```

### 3. PRIMARY KEY

행을 고유하게 식별합니다 (UNIQUE + NOT NULL).

```sql
-- 단일 컬럼
CREATE TABLE users (
    id SERIAL PRIMARY KEY
);

-- 복합 기본키
CREATE TABLE order_items (
    order_id INTEGER,
    product_id INTEGER,
    quantity INTEGER,
    PRIMARY KEY (order_id, product_id)
);
```

### 4. FOREIGN KEY

다른 테이블과의 참조 무결성을 보장합니다.

```sql
CREATE TABLE orders (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    total NUMERIC(10, 2)
);

-- 상세 옵션
CREATE TABLE order_items (
    id SERIAL PRIMARY KEY,
    order_id INTEGER,
    product_id INTEGER,
    FOREIGN KEY (order_id)
        REFERENCES orders(id)
        ON DELETE CASCADE
        ON UPDATE CASCADE,
    FOREIGN KEY (product_id)
        REFERENCES products(id)
        ON DELETE RESTRICT
);
```

**참조 동작 옵션:**

| 옵션 | 설명 |
|------|------|
| `CASCADE` | 부모 삭제/수정 시 자식도 삭제/수정 |
| `RESTRICT` | 자식이 있으면 부모 삭제/수정 불가 |
| `SET NULL` | 부모 삭제/수정 시 자식을 NULL로 |
| `SET DEFAULT` | 부모 삭제/수정 시 자식을 기본값으로 |
| `NO ACTION` | RESTRICT와 유사 (기본값) |

### 5. CHECK

조건을 만족하는 값만 허용합니다.

```sql
CREATE TABLE products (
    price NUMERIC CHECK (price > 0),
    quantity INTEGER CHECK (quantity >= 0),
    discount NUMERIC CHECK (discount >= 0 AND discount <= 100)
);

-- 복합 CHECK
CREATE TABLE reservations (
    start_date DATE,
    end_date DATE,
    CHECK (end_date > start_date)
);
```

### 6. DEFAULT

기본값을 설정합니다.

```sql
CREATE TABLE articles (
    status VARCHAR(20) DEFAULT 'draft',
    views INTEGER DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT CURRENT_TIMESTAMP,
    uuid UUID DEFAULT gen_random_uuid()
);
```

### 7. EXCLUSION

범위 겹침 등을 방지합니다.

```sql
CREATE EXTENSION IF NOT EXISTS btree_gist;

CREATE TABLE reservations (
    room_id INTEGER,
    period TSTZRANGE,
    EXCLUDE USING GIST (room_id WITH =, period WITH &&)
);
```

## IDENTITY 컬럼 (PostgreSQL 10+)

SQL 표준 방식의 자동 증가 컬럼입니다.

```sql
-- GENERATED ALWAYS: 항상 자동 생성 (명시적 삽입 불가)
CREATE TABLE users (
    id INTEGER GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    name VARCHAR(100)
);

-- GENERATED BY DEFAULT: 기본 자동 생성 (명시적 삽입 가능)
CREATE TABLE products (
    id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name VARCHAR(100)
);

-- 시작값과 증가값 지정
CREATE TABLE orders (
    id INTEGER GENERATED ALWAYS AS IDENTITY (START WITH 1000 INCREMENT BY 1) PRIMARY KEY
);
```

## Generated Columns (PostgreSQL 12+)

다른 컬럼으로부터 계산되는 컬럼입니다.

```sql
-- STORED: 디스크에 저장됨
CREATE TABLE products (
    price NUMERIC,
    quantity INTEGER,
    total NUMERIC GENERATED ALWAYS AS (price * quantity) STORED
);

-- PostgreSQL 18+: VIRTUAL (저장하지 않고 읽기 시 계산)
CREATE TABLE products (
    price NUMERIC,
    quantity INTEGER,
    total NUMERIC GENERATED ALWAYS AS (price * quantity) VIRTUAL
);

-- 예제: 전체 이름
CREATE TABLE people (
    first_name VARCHAR(50),
    last_name VARCHAR(50),
    full_name VARCHAR(101) GENERATED ALWAYS AS (first_name || ' ' || last_name) STORED
);
```

## 테이블 수정

### 컬럼 추가

```sql
ALTER TABLE users ADD COLUMN phone VARCHAR(20);
ALTER TABLE users ADD COLUMN age INTEGER DEFAULT 0 NOT NULL;
```

### 컬럼 삭제

```sql
ALTER TABLE users DROP COLUMN phone;
ALTER TABLE users DROP COLUMN IF EXISTS phone;
```

### 컬럼 이름 변경

```sql
ALTER TABLE users RENAME COLUMN username TO name;
```

### 데이터 타입 변경

```sql
ALTER TABLE users ALTER COLUMN phone TYPE VARCHAR(30);

-- 타입 변환이 필요한 경우
ALTER TABLE users ALTER COLUMN age TYPE VARCHAR(10) USING age::VARCHAR;
```

### 기본값 변경

```sql
ALTER TABLE users ALTER COLUMN is_active SET DEFAULT FALSE;
ALTER TABLE users ALTER COLUMN is_active DROP DEFAULT;
```

### 제약조건 추가/삭제

```sql
-- NOT NULL
ALTER TABLE users ALTER COLUMN email SET NOT NULL;
ALTER TABLE users ALTER COLUMN email DROP NOT NULL;

-- UNIQUE
ALTER TABLE users ADD CONSTRAINT unique_email UNIQUE (email);
ALTER TABLE users DROP CONSTRAINT unique_email;

-- CHECK
ALTER TABLE products ADD CONSTRAINT positive_price CHECK (price > 0);
ALTER TABLE products DROP CONSTRAINT positive_price;

-- FOREIGN KEY
ALTER TABLE orders ADD CONSTRAINT fk_user
    FOREIGN KEY (user_id) REFERENCES users(id);
ALTER TABLE orders DROP CONSTRAINT fk_user;
```

### 테이블 이름 변경

```sql
ALTER TABLE users RENAME TO members;
```

## 스키마 (Schema)

스키마는 테이블, 뷰, 함수 등의 논리적 그룹입니다.

### 스키마 생성

```sql
CREATE SCHEMA sales;
CREATE SCHEMA IF NOT EXISTS inventory;
```

### 스키마 내 객체 생성

```sql
CREATE TABLE sales.orders (
    id SERIAL PRIMARY KEY,
    amount NUMERIC
);

CREATE TABLE inventory.products (
    id SERIAL PRIMARY KEY,
    stock INTEGER
);
```

### 검색 경로 (Search Path)

```sql
-- 현재 검색 경로 확인
SHOW search_path;

-- 검색 경로 설정
SET search_path TO sales, public;

-- 특정 사용자의 기본 검색 경로
ALTER USER myuser SET search_path TO sales, public;
```

### public 스키마

- 기본 스키마
- 모든 사용자가 접근 가능
- 명시적 스키마 없이 생성된 객체가 위치

```sql
-- 아래 두 문장은 동일
CREATE TABLE users (...);
CREATE TABLE public.users (...);
```

### 스키마 삭제

```sql
DROP SCHEMA sales;
DROP SCHEMA sales CASCADE;  -- 포함된 객체 모두 삭제
```

## 테이블 상속 (Inheritance)

```sql
CREATE TABLE cities (
    name VARCHAR(100),
    population BIGINT,
    elevation INTEGER
);

CREATE TABLE capitals (
    country VARCHAR(100)
) INHERITS (cities);

-- capitals는 cities의 모든 컬럼을 상속
INSERT INTO capitals VALUES ('Seoul', 9776000, 38, 'South Korea');

-- 부모 테이블 조회 시 자식 데이터도 포함
SELECT * FROM cities;

-- 부모 테이블만 조회
SELECT * FROM ONLY cities;
```

## 테이블 파티셔닝

대용량 테이블을 작은 단위로 분할합니다.

### Range 파티셔닝

```sql
CREATE TABLE orders (
    id SERIAL,
    order_date DATE NOT NULL,
    amount NUMERIC
) PARTITION BY RANGE (order_date);

CREATE TABLE orders_2024 PARTITION OF orders
    FOR VALUES FROM ('2024-01-01') TO ('2025-01-01');

CREATE TABLE orders_2025 PARTITION OF orders
    FOR VALUES FROM ('2025-01-01') TO ('2026-01-01');

-- 기본 파티션 (범위에 해당하지 않는 데이터용)
CREATE TABLE orders_default PARTITION OF orders DEFAULT;
```

### List 파티셔닝

```sql
CREATE TABLE sales (
    id SERIAL,
    region VARCHAR(50) NOT NULL,
    amount NUMERIC
) PARTITION BY LIST (region);

CREATE TABLE sales_asia PARTITION OF sales
    FOR VALUES IN ('Korea', 'Japan', 'China');

CREATE TABLE sales_europe PARTITION OF sales
    FOR VALUES IN ('Germany', 'France', 'UK');
```

### Hash 파티셔닝

```sql
CREATE TABLE logs (
    id SERIAL,
    user_id INTEGER NOT NULL,
    message TEXT
) PARTITION BY HASH (user_id);

CREATE TABLE logs_0 PARTITION OF logs
    FOR VALUES WITH (MODULUS 4, REMAINDER 0);
CREATE TABLE logs_1 PARTITION OF logs
    FOR VALUES WITH (MODULUS 4, REMAINDER 1);
CREATE TABLE logs_2 PARTITION OF logs
    FOR VALUES WITH (MODULUS 4, REMAINDER 2);
CREATE TABLE logs_3 PARTITION OF logs
    FOR VALUES WITH (MODULUS 4, REMAINDER 3);
```

### 파티션 관리

```sql
-- 파티션 추가
CREATE TABLE orders_2026 PARTITION OF orders
    FOR VALUES FROM ('2026-01-01') TO ('2027-01-01');

-- 파티션 분리
ALTER TABLE orders DETACH PARTITION orders_2024;

-- 파티션 연결
ALTER TABLE orders ATTACH PARTITION orders_2024
    FOR VALUES FROM ('2024-01-01') TO ('2025-01-01');
```

## 임시 테이블 (Temporary Tables)

세션 종료 시 자동 삭제됩니다.

```sql
-- 세션 종료 시 삭제
CREATE TEMPORARY TABLE temp_results (
    id INTEGER,
    value TEXT
);

-- 트랜잭션 종료 시 삭제
CREATE TEMPORARY TABLE temp_data (
    id INTEGER
) ON COMMIT DROP;

-- 트랜잭션 종료 시 데이터만 삭제
CREATE TEMPORARY TABLE temp_data (
    id INTEGER
) ON COMMIT DELETE ROWS;
```

## UNLOGGED 테이블

WAL에 기록하지 않아 빠르지만, 장애 시 데이터 손실 가능합니다.

```sql
CREATE UNLOGGED TABLE cache (
    key VARCHAR(255) PRIMARY KEY,
    value TEXT,
    expires_at TIMESTAMPTZ
);
```

## 테이블 정보 조회

```sql
-- psql 명령어
\d users           -- 테이블 구조
\d+ users          -- 상세 정보
\dt                -- 테이블 목록
\dt schema.*       -- 특정 스키마 테이블
\dn                -- 스키마 목록

-- SQL로 조회
SELECT column_name, data_type, is_nullable, column_default
FROM information_schema.columns
WHERE table_name = 'users';

-- 테이블 크기
SELECT pg_size_pretty(pg_total_relation_size('users'));
```
