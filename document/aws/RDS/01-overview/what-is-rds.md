# Amazon RDS란?

## 개요

Amazon Relational Database Service(Amazon RDS)는 AWS 클라우드에서 관계형 데이터베이스를 쉽게 설정, 운영 및 확장할 수 있게 해주는 웹 서비스입니다. 비용 효율적이고 크기 조정이 가능한 용량을 제공하면서 일반적인 데이터베이스 관리 작업을 자동화합니다.

## 주요 이점

### 관리 부담 감소
- 하드웨어 프로비저닝, 데이터베이스 설정, 패치 및 백업 등의 시간 소모적인 관리 작업 자동화
- 관리형 서비스로서 인프라 관리 대신 애플리케이션에 집중 가능

### 확장성
- 몇 번의 클릭 또는 API 호출로 컴퓨팅 및 스토리지 리소스 조정 가능
- 다운타임 없이 스토리지 자동 확장 지원

### 고가용성
- Multi-AZ 배포를 통한 자동 장애 조치 지원
- 읽기 전용 복제본(Read Replica)을 통한 읽기 트래픽 분산

### 보안
- VPC를 통한 네트워크 격리
- 저장 데이터 및 전송 데이터 암호화
- IAM을 통한 접근 제어

## 지원 데이터베이스 엔진

| 엔진 | 설명 |
|------|------|
| **Amazon Aurora** | MySQL 및 PostgreSQL 호환, AWS 자체 개발 클라우드 네이티브 DB |
| **MySQL** | 오픈소스 관계형 데이터베이스 |
| **PostgreSQL** | 고급 오픈소스 객체-관계형 데이터베이스 |
| **MariaDB** | MySQL과 호환되는 오픈소스 데이터베이스 |
| **Oracle** | 상용 엔터프라이즈 데이터베이스 |
| **SQL Server** | Microsoft의 상용 관계형 데이터베이스 |
| **Db2** | IBM의 엔터프라이즈 데이터베이스 |

## 공유 책임 모델

### AWS의 책임
- 기본 인프라 관리 (하드웨어, 네트워크, 데이터 센터)
- 데이터베이스 소프트웨어 설치 및 패치
- 자동 백업 수행
- 장애 감지 및 복구

### 고객의 책임
- 데이터베이스 스키마 설계 및 최적화
- 쿼리 튜닝 및 인덱스 관리
- 데이터 관리 및 애플리케이션 로직
- 접근 제어 및 보안 정책 설정
- 암호화 키 관리 (고객 관리형 키 사용 시)

## DB 인스턴스

DB 인스턴스는 Amazon RDS의 기본 구성 요소입니다. 클라우드에서 실행되는 격리된 데이터베이스 환경으로, 하나 이상의 사용자 생성 데이터베이스를 포함할 수 있습니다.

### 인스턴스 제한
- 기본적으로 AWS 계정당 최대 40개의 DB 인스턴스
- SQL Server: 라이선스 포함 모델의 경우 최대 10개
- Oracle: 라이선스 포함 모델의 경우 최대 10개, BYOL 모델의 경우 최대 40개

## 리전 및 가용 영역

Amazon RDS는 전 세계 여러 AWS 리전에서 사용 가능하며, 각 리전 내 여러 가용 영역(AZ)에 걸쳐 고가용성을 제공합니다.

- ** 단일 AZ 배포**: 비용 효율적, 개발/테스트 환경에 적합
- **Multi-AZ 배포**: 프로덕션 환경을 위한 고가용성 구성

## 비용

Amazon RDS는 사용한 만큼만 지불하는 종량제 요금 모델을 제공합니다:

- ** 온디맨드 인스턴스**: 시간 단위로 청구, 장기 약정 없음
- ** 예약 인스턴스**: 1년 또는 3년 약정으로 할인 제공
- ** 스토리지 비용**: 프로비저닝된 스토리지 용량에 따라 청구
- **I/O 비용**: 데이터베이스 I/O 요청에 따라 청구 (일부 스토리지 유형)
- ** 백업 스토리지**: 자동 백업 보존 기간에 따른 비용

### AWS 프리 티어
신규 AWS 고객은 12개월 동안 매월 다음을 무료로 사용 가능:
- db.t2.micro 또는 db.t3.micro 인스턴스 750시간
- 범용(SSD) 스토리지 20GB
- 자동 백업 스토리지 20GB
