# 읽기 전용 복제본 (Read Replicas)

## 개요

읽기 전용 복제본은 원본 DB 인스턴스의 읽기 전용 복사본입니다. 읽기 트래픽을 분산하여 원본 인스턴스의 부하를 줄이고, 읽기 작업을 확장할 수 있습니다.

## 작동 원리

```
┌─────────────────────────────────────────────────────────────────┐
│                        VPC                                       │
│                                                                  │
│  ┌─────────────┐     ┌─────────────┐     ┌─────────────┐        │
│  │   Source    │     │  Replica 1  │     │  Replica 2  │        │
│  │   (Primary) │ ──▶ │  (읽기전용) │     │  (읽기전용) │        │
│  │ 읽기/쓰기   │비동기│             │     │             │        │
│  └─────────────┘ 복제└─────────────┘     └─────────────┘        │
│        │                   │                   │                 │
│        ▼                   ▼                   ▼                 │
│  ┌─────────────────────────────────────────────────────┐        │
│  │              애플리케이션                            │        │
│  │    쓰기 → 원본  /  읽기 → 복제본(들)                │        │
│  └─────────────────────────────────────────────────────┘        │
└─────────────────────────────────────────────────────────────────┘
```

### 비동기식 복제
- 원본 인스턴스의 변경 사항을 비동기적으로 복제
- 약간의 복제 지연(Replica Lag) 발생 가능
- 네이티브 데이터베이스 엔진 복제 기능 사용

## 지원 데이터베이스 엔진

| 엔진 | 복제 방식 | 최대 복제본 수 |
|-----|---------|--------------|
| MySQL | 네이티브 복제 | 15개 |
| MariaDB | 네이티브 복제 | 15개 |
| PostgreSQL | 스트리밍 복제 | 15개 |
| Oracle | Active Data Guard | 5개 |
| SQL Server | Always On AG | 5개 |
| Db2 | HADR | 1개 |

## 읽기 복제본 생성

### AWS 콘솔

1. RDS 콘솔에서 원본 DB 인스턴스 선택
2. **Actions** → **Create read replica** 클릭
3. 복제본 설정 구성
4. **Create read replica** 클릭

### AWS CLI

```bash
aws rds create-db-instance-read-replica \
    --db-instance-identifier mydb-replica \
    --source-db-instance-identifier mydb \
    --db-instance-class db.m6g.large \
    --availability-zone us-east-1b
```

### 크로스 리전 복제본

```bash
aws rds create-db-instance-read-replica \
    --db-instance-identifier mydb-replica-west \
    --source-db-instance-identifier arn:aws:rds:us-east-1:123456789012:db:mydb \
    --db-instance-class db.m6g.large \
    --region us-west-2 \
    --kms-key-id arn:aws:kms:us-west-2:123456789012:key/...
```

## 사용 시나리오

### 1. 읽기 확장
```
┌─────────────┐
│    원본     │ ← 쓰기 트래픽
└─────────────┘
       │
       ├──────────────┬──────────────┐
       ▼              ▼              ▼
┌───────────┐  ┌───────────┐  ┌───────────┐
│ Replica 1 │  │ Replica 2 │  │ Replica 3 │ ← 읽기 트래픽
└───────────┘  └───────────┘  └───────────┘
```

** 적합한 경우:**
- 읽기 비율이 높은 워크로드
- 리포팅/분석 쿼리 분리
- 읽기 성능 향상 필요

### 2. 재해 복구

```
┌─────────────────────┐     ┌─────────────────────┐
│    us-east-1        │     │    us-west-2        │
│                     │     │                     │
│  ┌───────────┐      │     │  ┌───────────┐      │
│  │   원본    │ ─────┼────▶│  │  복제본   │      │
│  └───────────┘      │     │  └───────────┘      │
│                     │     │  (재해 시 승격)     │
└─────────────────────┘     └─────────────────────┘
```

** 장점:**
- 리전 장애 대비
- 수동 승격으로 독립 인스턴스 전환
- RTO 단축

### 3. 비즈니스 분석

```
프로덕션 워크로드 → 원본 인스턴스
리포팅/분석 워크로드 → 읽기 복제본
```

** 장점:**
- 프로덕션 성능 영향 없음
- 대용량 쿼리 분리 실행

## 읽기 복제본 승격

### 독립 인스턴스로 승격

읽기 복제본을 독립적인 읽기/쓰기 가능한 DB 인스턴스로 승격:

```bash
aws rds promote-read-replica \
    --db-instance-identifier mydb-replica
```

### 승격 과정
1. 복제 중지
2. 재부팅 (선택적)
3. 독립 인스턴스로 전환
4. 읽기/쓰기 가능 상태

### 승격 시 고려사항
- 승격 후 원본과의 복제 관계 해제
- 승격 취소 불가
- DNS 엔드포인트 유지

## 복제 지연 (Replica Lag)

### 지연 원인
- 높은 쓰기 트래픽
- 복제본 리소스 부족
- 네트워크 대기 시간
- 장시간 실행 쿼리

### 모니터링

```bash
# CloudWatch 메트릭 확인
aws cloudwatch get-metric-statistics \
    --namespace AWS/RDS \
    --metric-name ReplicaLag \
    --dimensions Name=DBInstanceIdentifier,Value=mydb-replica \
    --start-time 2024-01-15T00:00:00Z \
    --end-time 2024-01-15T23:59:59Z \
    --period 300 \
    --statistics Average
```

### 지연 최소화 방법
```
✅ 복제본 인스턴스 클래스 증가
✅ IOPS 프로비저닝 증가
✅ 장시간 트랜잭션 분리
✅ 병렬 복제 활성화 (지원 시)
```

## 계층적 복제

### 복제본의 복제본

```
원본 ──▶ 복제본 1 ──▶ 복제본 1-1
              │
              └──▶ 복제본 1-2
```

** 지원 엔진:**
- MySQL ✅
- MariaDB ✅
- PostgreSQL ✅
- Oracle ❌
- SQL Server ❌
- Db2 ❌

### 제한사항
- 최대 깊이 제한 존재
- 각 레벨에서 지연 누적

## 비용

### 요금 구조
| 항목 | 비용 |
|-----|-----|
| 인스턴스 비용 | 원본과 동일 (시간당) |
| 스토리지 비용 | 원본과 동일 |
| 동일 리전 데이터 전송 | 무료 |
| 크로스 리전 데이터 전송 | 리전 간 전송 요금 |

## Multi-AZ와 읽기 복제본 조합

```
┌─────────────────────────────────────────────────────────────────┐
│                        us-east-1                                 │
│  ┌─────────────┐     ┌─────────────┐                            │
│  │   Primary   │     │   Standby   │  ← Multi-AZ (고가용성)     │
│  │ (us-east-1a)│ ──▶ │ (us-east-1b)│                            │
│  └─────────────┘     └─────────────┘                            │
│        │                                                         │
│        ▼                                                         │
│  ┌─────────────┐                                                │
│  │  Replica    │  ← 읽기 확장                                   │
│  │ (us-east-1c)│                                                │
│  └─────────────┘                                                │
└─────────────────────────────────────────────────────────────────┘
```

### 조합 시 장점
- 고가용성 (Multi-AZ)
- 읽기 확장 (Read Replica)
- 재해 복구 (크로스 리전 복제본)

## 모범 사례

### 설계
```
✅ 읽기/쓰기 분리 아키텍처 설계
✅ 복제본별 용도 명확화 (읽기 확장, DR, 분석)
✅ 복제본과 원본의 인스턴스 클래스 적절히 선택
```

### 운영
```
✅ 복제 지연 모니터링 및 알림 설정
✅ 정기적인 승격 테스트
✅ 사용하지 않는 복제본 삭제
```

### 애플리케이션
```
✅ 읽기 쿼리를 복제본으로 라우팅
✅ 복제 지연 허용 가능한 쿼리만 복제본 사용
✅ 연결 풀링으로 연결 관리
```
