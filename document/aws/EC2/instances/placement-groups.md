# EC2 배치 그룹 (Placement Groups)

## 개요

배치 그룹은 인스턴스를 논리적으로 그룹화하여 저지연 통신을 가능하게 하거나 하드웨어 장애 영향을 최소화합니다. 배치 그룹 사용에는 추가 요금이 없습니다.

## 배치 전략 비교

| 전략 | 목적 | 네트워크 지연 | 장애 격리 | 최대 인스턴스 |
|------|------|--------------|-----------|--------------|
| **Cluster** | 고성능 컴퓨팅 | 최저 | 낮음 | 계정 한도 |
| **Spread** | 고가용성 | 높음 | 최고 | AZ당 7개 |
| **Partition** | 분산 워크로드 | 중간 | 높음 | 파티션당 수백 개 |

## 클러스터 배치 그룹 (Cluster)

### 개념

```
┌─────────────────────────────────────────────────────────────┐
│                     단일 가용 영역                           │
│  ┌───────────────────────────────────────────────────────┐  │
│  │                    단일 랙                             │  │
│  │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐    │  │
│  │  │ i-1 │ │ i-2 │ │ i-3 │ │ i-4 │ │ i-5 │ │ i-6 │    │  │
│  │  └─────┘ └─────┘ └─────┘ └─────┘ └─────┘ └─────┘    │  │
│  │           ↕ 저지연, 고대역폭 연결                     │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### 특징

- 모든 인스턴스가 동일한 랙에 배치
- 인스턴스 간 10Gbps 이상의 네트워크 대역폭
- 마이크로초 수준의 지연 시간
- Enhanced Networking 지원 인스턴스 권장

### 사용 사례

- 고성능 컴퓨팅 (HPC)
- 빅데이터 분석
- 실시간 애널리틱스
- 밀결합 분산 애플리케이션

### 제한 사항

- 단일 가용 영역에만 배포 가능
- 동일한 인스턴스 타입 권장
- 랙 장애 시 모든 인스턴스 영향
- 용량 부족 시 시작 실패 가능

### 생성 및 사용

```bash
# 클러스터 배치 그룹 생성
aws ec2 create-placement-group \
    --group-name my-cluster-pg \
    --strategy cluster

# 배치 그룹에 인스턴스 시작
aws ec2 run-instances \
    --image-id ami-xxxxxxxxx \
    --instance-type c5n.18xlarge \
    --placement "GroupName=my-cluster-pg"
```

### 모범 사례

1. 한 번에 모든 인스턴스 시작 (용량 확보)
2. 동일한 인스턴스 타입 사용
3. Enhanced Networking 활성화
4. EFA(Elastic Fabric Adapter) 고려

## 분산 배치 그룹 (Spread)

### 개념

```
┌─────────────────────────────────────────────────────────────┐
│                      가용 영역 A                             │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐                     │
│  │  랙 1   │  │  랙 2   │  │  랙 3   │                     │
│  │ ┌─────┐ │  │ ┌─────┐ │  │ ┌─────┐ │                     │
│  │ │ i-1 │ │  │ │ i-2 │ │  │ │ i-3 │ │  ...  (최대 7개)   │
│  │ └─────┘ │  │ └─────┘ │  │ └─────┘ │                     │
│  └─────────┘  └─────────┘  └─────────┘                     │
└─────────────────────────────────────────────────────────────┘
```

### 특징

- 각 인스턴스가 서로 다른 랙에 배치
- 하드웨어 장애 격리 최대화
- 다중 가용 영역에 걸쳐 생성 가능

### 사용 사례

- 미션 크리티컬 애플리케이션
- 고가용성이 중요한 워크로드
- 데이터베이스 클러스터
- 중요 서비스 인스턴스

### 제한 사항

- 가용 영역당 최대 7개 인스턴스
- 인스턴스 간 네트워크 지연 증가
- Dedicated Host 미지원

### 생성 및 사용

```bash
# 분산 배치 그룹 생성
aws ec2 create-placement-group \
    --group-name my-spread-pg \
    --strategy spread \
    --spread-level rack

# 배치 그룹에 인스턴스 시작
aws ec2 run-instances \
    --image-id ami-xxxxxxxxx \
    --instance-type m5.large \
    --placement "GroupName=my-spread-pg" \
    --count 7
```

### 분산 수준

| 수준 | 설명 |
|------|------|
| **rack** | 랙 수준 분산 (기본값) |
| **host** | 호스트 수준 분산 (Outposts에서만) |

## 파티션 배치 그룹 (Partition)

### 개념

```
┌─────────────────────────────────────────────────────────────┐
│                      가용 영역 A                             │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │ 파티션 1    │  │ 파티션 2    │  │ 파티션 3    │         │
│  │ (랙 1)      │  │ (랙 2)      │  │ (랙 3)      │         │
│  │ ┌───┐┌───┐ │  │ ┌───┐┌───┐ │  │ ┌───┐┌───┐ │         │
│  │ │i-1││i-2│ │  │ │i-3││i-4│ │  │ │i-5││i-6│ │         │
│  │ └───┘└───┘ │  │ └───┘└───┘ │  │ └───┘└───┘ │         │
│  │ ┌───┐┌───┐ │  │ ┌───┐┌───┐ │  │ ┌───┐┌───┐ │         │
│  │ │i-7││i-8│ │  │ │i-9││i10│ │  │ │i11││i12│ │         │
│  │ └───┘└───┘ │  │ └───┘└───┘ │  │ └───┘└───┘ │         │
│  └─────────────┘  └─────────────┘  └─────────────┘         │
└─────────────────────────────────────────────────────────────┘
```

### 특징

- 논리적 파티션으로 인스턴스 그룹화
- 파티션 간 하드웨어 공유 없음
- 각 파티션에 여러 인스턴스 배치 가능
- 파티션 정보를 애플리케이션에서 조회 가능

### 사용 사례

- HDFS, HBase, Cassandra
- Kafka 클러스터
- 대규모 분산 복제 워크로드
- 토폴로지 인식 애플리케이션

### 제한 사항

- 가용 영역당 최대 7개 파티션
- Dedicated Host 미지원

### 생성 및 사용

```bash
# 파티션 배치 그룹 생성 (7개 파티션)
aws ec2 create-placement-group \
    --group-name my-partition-pg \
    --strategy partition \
    --partition-count 7

# 특정 파티션에 인스턴스 시작
aws ec2 run-instances \
    --image-id ami-xxxxxxxxx \
    --instance-type r5.large \
    --placement "GroupName=my-partition-pg,PartitionNumber=1"
```

### 파티션 정보 조회

```bash
# 인스턴스 메타데이터에서 파티션 번호 확인
TOKEN=$(curl -X PUT "http://169.254.169.254/latest/api/token" \
    -H "X-aws-ec2-metadata-token-ttl-seconds: 21600")

curl -H "X-aws-ec2-metadata-token: $TOKEN" \
    http://169.254.169.254/latest/meta-data/placement/partition-number
```

## 배치 그룹 관리

### 조회

```bash
# 모든 배치 그룹 조회
aws ec2 describe-placement-groups

# 특정 배치 그룹 조회
aws ec2 describe-placement-groups --group-names my-cluster-pg
```

### 삭제

```bash
# 배치 그룹 삭제 (인스턴스가 없어야 함)
aws ec2 delete-placement-group --group-name my-cluster-pg
```

### 인스턴스 이동

기존 인스턴스를 배치 그룹으로 이동하려면:

1. 인스턴스 중지
2. 배치 그룹 수정
3. 인스턴스 시작

```bash
# 인스턴스를 배치 그룹으로 이동
aws ec2 modify-instance-placement \
    --instance-id i-xxxxxxxxx \
    --group-name my-spread-pg
```
