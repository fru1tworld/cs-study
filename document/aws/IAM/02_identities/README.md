## 자격 증명 비교

| 항목 | 루트 사용자 | IAM 사용자 | IAM 역할 |
|------|-----------|-----------|---------|
| 권한 제한 | 불가능 | 가능 | 가능 |
| 자격 증명 유형 | 장기 | 장기 | 임시 |
| 공유 가능 | 금지 | 권장하지 않음 | 위임 목적으로 설계 |
| MFA 지원 | 예 | 예 | 예 |
| 삭제 가능 | 불가능 | 가능 | 가능 |

## 권장 사용 패턴

```
┌─────────────────────────────────────────────────────────────────┐
│                         AWS 계정                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  루트 사용자 (긴급 상황에만 사용, MFA 필수)                       │
│       │                                                         │
│       ▼                                                         │
│  관리자 IAM 사용자 (일상 관리 작업)                               │
│       │                                                         │
│       ├──→ 개발자 그룹 ──→ 개발자 사용자들                       │
│       │                                                         │
│       ├──→ 운영자 그룹 ──→ 운영자 사용자들                       │
│       │                                                         │
│       └──→ 서비스 역할들 ──→ EC2, Lambda 등                      │
│                                                                 │
│  외부 사용자 ──→ 자격 증명 공급자 ──→ IAM 역할                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 선택 가이드

### IAM 사용자 선택
- 개별 사람이 AWS에 직접 접근해야 할 때
- CodeCommit 접근이 필요할 때
- 역할을 지원하지 않는 도구 사용 시

### IAM 역할 선택
- AWS 서비스가 다른 서비스에 접근해야 할 때
- 교차 계정 접근이 필요할 때
- 임시 접근만 필요할 때
- 연합 사용자 접근 시

### 그룹 사용
- 여러 사용자에게 동일한 권한을 부여할 때
- 직무 기반 접근 제어 구현 시
