# AWS 계정 루트 사용자

## 루트 사용자란?

AWS 계정을 처음 생성할 때 자동으로 만들어지는 단일 자격 증명입니다. 계정 생성 시 사용한 이메일 주소와 비밀번호로 로그인하며, 계정 내 모든 AWS 서비스와 리소스에 완전한 접근 권한을 가집니다.

## 루트 사용자의 특징

### 무제한 권한
- 모든 AWS 서비스에 접근 가능
- 모든 리소스에 대한 전체 권한
- 권한을 제한하는 IAM 정책 적용 불가

### 보안 위험
- 모든 권한을 가지므로 손상 시 치명적
- 일상 작업에 사용하면 안 됨
- 자격 증명이 노출되면 전체 계정 위험

## 루트 사용자만 수행 가능한 작업

### 계정 관리
| 작업 | 설명 |
|------|------|
| 계정 설정 변경 | 계정 이름, 이메일, 비밀번호, 액세스 키 변경 |
| 계정 폐쇄 | AWS 계정 완전 삭제 |
| IAM 권한 복구 | 실수로 모든 관리자 권한 제거 시 복구 |
| 지원 플랜 변경 | AWS Support 플랜 변경 |

### 결제 관련
| 작업 | 설명 |
|------|------|
| 결제 정보 접근 | IAM 사용자의 결제 접근 활성화 |
| 세금 송장 조회 | 특정 세금 관련 송장 확인 |
| 예약 인스턴스 판매 | EC2 Reserved Instance Marketplace 등록 |

### 보안 설정
| 작업 | 설명 |
|------|------|
| S3 버킷 MFA Delete | S3 버킷의 MFA Delete 설정 |
| GovCloud 가입 | AWS GovCloud(US) 계정 생성 |
| CloudFront 키 페어 | CloudFront 키 페어 생성 |

## 루트 사용자 보안 모범 사례

### 1. 즉시 MFA 활성화

```
루트 사용자 → 보안 자격 증명 → MFA 디바이스 할당
```

권장 MFA 유형:
- 하드웨어 보안 키 (YubiKey 등)
- 가상 MFA 앱 (Authenticator 앱)
- 하드웨어 TOTP 토큰

### 2. 액세스 키 생성 금지

```
⚠️ 루트 사용자의 액세스 키는 절대 생성하지 마세요!

이유:
- 장기 자격 증명으로 노출 위험이 높음
- 프로그래밍 방식 접근에는 IAM 역할 사용
- 이미 있다면 즉시 삭제
```

### 3. 강력한 비밀번호 사용

비밀번호 요구사항:
- 최소 14자 이상
- 대문자, 소문자, 숫자, 특수문자 포함
- 다른 사이트와 중복 금지
- 정기적 변경

### 4. 일상 작업에 사용 금지

```
올바른 사용 패턴:
1. 관리자 IAM 사용자 생성
2. 일상 작업은 IAM 사용자로 수행
3. 루트 사용자는 필수 작업에만 사용
```

### 5. 자격 증명 안전 보관

권장 방법:
- 비밀번호 관리자 사용
- 복구 이메일 보안 강화
- 복수의 관리자에게 접근 정보 분산 보관

## AWS Organizations에서의 루트 사용자

### 관리 계정에서 멤버 계정 관리

Organizations를 사용하면 관리 계정에서 멤버 계정의 루트 자격 증명을 중앙에서 관리할 수 있습니다.

```
관리 계정
├── 멤버 계정 A → 루트 자격 증명 제거 가능
├── 멤버 계정 B → 루트 자격 증명 제거 가능
└── 멤버 계정 C → 루트 자격 증명 제거 가능
```

### 가능한 작업
- 멤버 계정 루트 자격 증명 삭제
- 비밀번호 복구 허용/차단
- 특정 특권 작업을 위한 임시 접근 부여

## 루트 사용자 vs IAM 관리자

| 항목 | 루트 사용자 | IAM 관리자 |
|------|------------|-----------|
| 권한 제한 | 불가능 | 가능 |
| MFA 강제 | 수동 설정 | 정책으로 강제 가능 |
| 활동 감사 | 제한적 | CloudTrail 완전 지원 |
| 삭제 가능 | 불가능 | 가능 |
| 일상 사용 | 권장하지 않음 | 권장 |

## 응급 상황 대응

### 루트 자격 증명 분실 시

1. 이메일로 비밀번호 재설정
2. MFA 분실 시 AWS Support 연락
3. 계정 복구 절차 진행

### 루트 자격 증명 노출 시

1. 즉시 비밀번호 변경
2. 액세스 키가 있다면 삭제
3. MFA 디바이스 교체
4. CloudTrail 로그 검토
5. 의심스러운 리소스 확인 및 삭제
