## 빠른 시작

### IAM을 사용해야 하는 이유

1. ** 보안 강화**: 최소 권한 원칙을 적용하여 필요한 권한만 부여
2. ** 중앙 집중 관리**: 모든 AWS 리소스 접근을 한 곳에서 관리
3. ** 감사 및 규정 준수**: CloudTrail과 통합하여 모든 작업 기록
4. ** 비용 절감**: 무료로 제공되는 핵심 보안 서비스

### 핵심 구성 요소

```
AWS 계정
├── 루트 사용자 (최초 생성, 일상 사용 금지)
├── IAM 사용자 (사람 또는 서비스용)
├── IAM 그룹 (사용자 모음)
├── IAM 역할 (임시 권한 위임)
└── IAM 정책 (권한 정의 문서)
```

### 권장 시작 순서

1. 루트 사용자에 MFA 설정
2. 관리자 IAM 사용자 생성
3. 루트 사용자 대신 관리자 사용자로 작업
4. 필요한 사용자와 그룹 생성
5. 최소 권한 정책 적용
