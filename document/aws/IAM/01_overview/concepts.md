# IAM 핵심 개념

## 용어 정의

### 리소스 (Resources)
IAM에 저장된 사용자, 그룹, 역할, 정책, 자격 증명 공급자 객체입니다.

### 자격 증명 (Identities)
IAM 리소스 객체로, 작업 및 리소스에 대한 권한을 정의합니다.
- IAM 사용자
- IAM 그룹
- IAM 역할

### 엔터티 (Entities)
AWS가 인증에 사용하는 IAM 리소스 객체입니다.
- IAM 사용자
- IAM 역할

### 보안 주체 (Principals)
AWS 계정 루트 사용자, IAM 사용자, IAM 역할을 사용하여 AWS에 로그인하고 요청하는 사람 또는 애플리케이션입니다.

## ARN (Amazon Resource Name)

모든 IAM 리소스는 고유한 ARN을 가집니다.

### ARN 형식

```
arn:partition:service:region:account:resource
```

### IAM ARN 예시

```
# IAM 사용자
arn:aws:iam::123456789012:user/johndoe

# IAM 그룹
arn:aws:iam::123456789012:group/developers

# IAM 역할
arn:aws:iam::123456789012:role/S3Access

# 관리형 정책
arn:aws:iam::123456789012:policy/MyPolicy

# 인스턴스 프로파일
arn:aws:iam::123456789012:instance-profile/Webserver
```

### 특수 ARN 패턴

```
# 모든 IAM 사용자
arn:aws:iam::123456789012:user/*

# 경로 기반 사용자
arn:aws:iam::123456789012:user/division_abc/*

# 루트 계정
arn:aws:iam::123456789012:root
```

## 요청 컨텍스트

AWS에 요청을 보낼 때 포함되는 정보입니다.

### 요청 구성 요소

| 구성 요소 | 설명 | 예시 |
|-----------|------|------|
| **Actions/Operations** | 수행할 작업 | `ec2:StartInstances` |
| **Resources** | 작업 대상 리소스 | S3 버킷, EC2 인스턴스 |
| **Principal** | 요청을 보내는 주체 | IAM 사용자, 역할 |
| **Environment data** | IP 주소, 시간 등 | `aws:SourceIp` |
| **Resource data** | 리소스 관련 데이터 | 태그, 소유자 |

## 정책 평가 로직

### 단일 계정 내 평가

```
1. 명시적 거부 확인 → 거부가 있으면 즉시 거부
2. SCP 확인 (Organizations) → 허용하지 않으면 암묵적 거부
3. 리소스 기반 정책 확인 → 허용하면 다음 단계
4. 자격 증명 기반 정책 확인 → 허용하면 다음 단계
5. 권한 경계 확인 → 허용하면 다음 단계
6. 세션 정책 확인 → 허용하면 최종 허용
```

### 교차 계정 평가

다른 계정의 리소스에 접근할 때는 **양쪽 모두**에서 허용해야 합니다:

```
계정 A의 사용자 → 계정 B의 S3 버킷 접근

필요 조건:
1. 계정 A의 자격 증명 기반 정책에서 허용
2. 계정 B의 리소스 기반 정책에서 허용
```

## 신뢰 영역 (Zone of Trust)

### 신뢰 영역 내부
- 같은 AWS 계정 내의 리소스
- 정책 평가가 비교적 단순

### 신뢰 영역 외부
- 다른 AWS 계정의 리소스
- 양쪽 계정 모두에서 명시적 허용 필요

## 정책 유형 간 상호작용

```
┌─────────────────────────────────────────────────────────┐
│                    최종 권한 결정                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌─────────────┐   ┌─────────────┐   ┌─────────────┐   │
│  │     SCP     │ ∩ │  자격증명   │ ∩ │   권한      │   │
│  │  (최대한도) │   │  기반 정책  │   │   경계      │   │

│                           │                             │
│                    ┌──────┴──────┐                     │
│                    │   리소스    │                     │
│                    │  기반 정책  │                     │
│                    └─────────────┘                     │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

## 암묵적 거부 vs 명시적 거부

### 암묵적 거부 (Implicit Deny)
- 정책에 해당 권한이 없는 경우
- 다른 정책의 허용으로 재정의 가능

### 명시적 거부 (Explicit Deny)
- 정책에서 명시적으로 `"Effect": "Deny"` 설정
- 다른 어떤 정책으로도 재정의 불가

```json
// 명시적 거부 예시
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Deny",
            "Action": "s3:DeleteBucket",
            "Resource": "*"
        }
    ]
}
```
