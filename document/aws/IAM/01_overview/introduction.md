# AWS IAM 소개

## IAM이란?

AWS Identity and Access Management(IAM)는 AWS 리소스에 대한 접근을 안전하게 제어하는 웹 서비스입니다. IAM을 사용하면 사용자가 접근할 수 있는 AWS 리소스를 제어하는 권한을 중앙에서 관리할 수 있습니다.

## 핵심 기능

### 1. 공유 접근 (Shared Access)
다른 사람에게 AWS 계정의 리소스를 관리하고 사용할 수 있는 권한을 부여할 수 있습니다. 비밀번호나 액세스 키를 공유하지 않아도 됩니다.

### 2. 세분화된 권한 (Granular Permissions)
각 사용자에게 서로 다른 리소스에 대해 서로 다른 권한을 부여할 수 있습니다. 예를 들어, 일부 사용자에게는 EC2, S3, DynamoDB에 대한 전체 접근 권한을, 다른 사용자에게는 일부 S3 버킷에 대한 읽기 전용 권한만 부여할 수 있습니다.

### 3. EC2 애플리케이션의 안전한 접근
EC2 인스턴스에서 실행되는 애플리케이션이 다른 AWS 리소스에 안전하게 접근할 수 있도록 자격 증명을 제공합니다.

### 4. 다중 인증 (MFA)
추가적인 보안을 위해 2단계 인증을 추가할 수 있습니다.

### 5. 자격 증명 연동 (Identity Federation)
기업 네트워크나 인터넷 자격 증명 공급자의 사용자가 임시 자격 증명으로 AWS 계정에 접근할 수 있습니다.

### 6. AWS 서비스와의 통합
대부분의 AWS 서비스와 통합되어 일관된 접근 제어를 제공합니다.

### 7. 최종 일관성 (Eventually Consistent)
IAM 변경사항은 전 세계 데이터센터에 점진적으로 복제됩니다. 변경 후 즉시 모든 곳에서 적용되지 않을 수 있습니다.

### 8. 무료 사용
IAM은 추가 비용 없이 제공됩니다. IAM Access Analyzer의 일부 고급 기능만 별도 요금이 적용됩니다.

## IAM 작동 원리

### 인증 (Authentication)
사용자가 AWS에 요청을 보내면, AWS는 먼저 해당 사용자의 신원을 확인합니다.

```
사용자 → 로그인 정보 제출 → AWS 신원 확인 → 인증 완료
```

### 인가 (Authorization)
인증된 사용자가 특정 작업을 요청하면, AWS는 해당 사용자에게 적용된 정책을 평가하여 권한을 확인합니다.

```
인증된 사용자 → 작업 요청 → 정책 평가 → 허용/거부
```

## 접근 제어 기본 규칙

1. ** 기본 거부 (Default Deny)**: 모든 요청은 기본적으로 거부됩니다
2. ** 명시적 허용 (Explicit Allow)**: 정책에서 명시적으로 허용해야 접근 가능
3. ** 명시적 거부 우선 (Explicit Deny Wins)**: 하나의 정책이라도 거부하면 최종 거부

```
기본 거부 → 허용 정책 확인 → 거부 정책 확인 → 최종 결정
     ↓              ↓               ↓
   거부          허용 가능        무조건 거부
```

## 서비스 가용성

IAM은 ** 최종 일관성(Eventually Consistent)** 특성을 가집니다:
- 변경사항이 전 세계 데이터센터에 복제되는 데 시간이 걸릴 수 있음
- 중요한 변경 후에는 잠시 대기 후 다음 작업 수행 권장
- 전 세계 어디서나 낮은 지연 시간으로 접근 가능
