# Amazon ECS란?

## 개요

**Amazon Elastic Container Service (ECS)** 는 컨테이너화된 애플리케이션을 쉽게 배포, 관리, 확장할 수 있는 완전 관리형 컨테이너 오케스트레이션 서비스입니다.

## 핵심 아키텍처 계층

ECS는 3가지 핵심 계층으로 구성됩니다:

### 1. Capacity (용량)
컨테이너가 실행되는 인프라 계층입니다.

| 옵션 | 설명 | 특징 |
|------|------|------|
| **ECS Managed Instances** | AWS가 EC2 인스턴스 전체 관리 | 자동 프로비저닝, 패칭, 스케일링 |
| **AWS Fargate** | 서버리스 컴퓨팅 | 인프라 관리 불필요, 사용한 만큼만 비용 |
| **Amazon EC2** | 직접 EC2 인스턴스 관리 | 최대 제어 권한 |
| **On-premises (ECS Anywhere)** | 온프레미스 서버/VM | 하이브리드 환경 지원 |

### 2. Controller (컨트롤러)
컨테이너에서 실행되는 애플리케이션을 배포하고 관리하는 계층입니다.

- ECS 스케줄러가 태스크 배치 및 관리
- 서비스 정의에 따라 원하는 상태 유지
- 실패한 태스크 자동 복구

### 3. Provisioning (프로비저닝)
ECS와 상호작용하는 방법을 제공합니다.

- **AWS Management Console**: 웹 기반 UI
- **AWS CLI**: 명령줄 인터페이스
- **AWS SDKs**: 프로그래밍 방식 접근
- **AWS CDK**: Infrastructure as Code
- **AWS Copilot**: 컨테이너 앱 전용 CLI

### Task Definition (태스크 정의)
- 애플리케이션의 청사진
- JSON 형식으로 컨테이너 설정 정의
- CPU, 메모리, 네트워킹, 볼륨 등 지정

### Task (태스크)
- Task Definition의 인스턴스화된 실행 단위
- 하나 이상의 컨테이너로 구성
- 수명 주기가 있는 일회성 실행

### Service (서비스)
- 지정된 수의 태스크를 지속적으로 실행/유지
- 실패한 태스크 자동 재시작
- 로드 밸런서와 통합 가능

### Cluster (클러스터)
- 태스크와 서비스의 논리적 그룹화
- 리전별로 생성
- 여러 인프라 유형 혼합 가능

## 주요 기능

### Auto Scaling
- **Cluster Auto Scaling**: EC2 인스턴스 용량 자동 조정
- **Service Auto Scaling**: 태스크 수 자동 조정

### 통합 서비스
- **Amazon ECR**: 컨테이너 이미지 저장소
- **Elastic Load Balancing**: 트래픽 분산
- **Amazon CloudWatch**: 모니터링 및 로깅
- **AWS IAM**: 접근 제어
- **AWS Secrets Manager**: 민감한 데이터 관리

## 사용 사례

1. ** 마이크로서비스 아키텍처**: 서비스별 독립적 배포 및 확장
2. ** 배치 처리**: 대규모 데이터 처리 작업
3. **ML 워크로드**: 모델 학습 및 추론
4. **CI/CD 파이프라인**: 자동화된 빌드/테스트/배포
5. ** 하이브리드 배포**: 클라우드와 온프레미스 통합

## 요금

- **Amazon ECS 자체**: 무료
- ** 인프라 비용만 발생**:
  - EC2 인스턴스 사용 시: EC2 요금
  - Fargate 사용 시: vCPU 및 메모리 요금
- ** 추가 서비스**: ECR, CloudWatch 등 별도 과금
