# Confluent Intelligence 신규 기능: A2A, 다변량 이상 탐지, Cosmos DB 및 Amazon S3 Vectors 벡터 검색 등

> **원문:** [New in Confluent Intelligence: A2A, Multivariate Anomaly Detection, Vector Search for Cosmos DB, Amazon S3 Vectors, and More](https://www.confluent.io/blog/2026-q1-confluent-intelligence-update/)
> **저자:** Confluent Staff | **게시일:** 2026년 2월 26일 | **읽는 시간:** 8분

AI 모델이 점점 더 범용화됨에 따라, 기업의 가치 동인은 더 이상 "어떤 대규모 언어 모델(LLM)을 사용하는가?"가 아니라 "신뢰할 수 있는 실시간 AI 의사결정을 위해 데이터를 어떻게 활용할 수 있는가?"입니다. 에이전트가 자율적으로 계획하고, 결정하고, 행동하는 에이전틱 AI 시스템은 보유한 컨텍스트만큼만 유용합니다. 해당 컨텍스트가 오래되었거나, 단편적이거나, 취약한 포인트-투-포인트 통합 뒤에 갇혀 있다면, 아무리 뛰어난 모델도 제대로 된 결과를 제공하지 못합니다.

Confluent Intelligence는 이 문제를 해결하기 위해 만들어졌습니다. Apache Kafka와 Apache Flink를 결합한 통합 데이터 스트리밍 플랫폼인 Confluent Cloud에서 실시간으로 컨텍스트가 풍부하고 신뢰할 수 있는 AI 시스템을 구축하기 위한 완전 관리형 서비스입니다. Confluent Intelligence를 사용하면 운영 이벤트를 스트리밍하고, 외부 데이터로 지속적으로 보강하며, 내장 ML을 적용하고, 인프라를 일일이 연결하지 않고도 스트리밍 에이전트 및 기타 AI 애플리케이션에 최신 컨텍스트를 제공할 수 있습니다.

오늘, 우리는 스트리밍 에이전트, 내장 ML 함수, 그리고 MCP(Model Context Protocol) 지원에 걸친 새로운 기능으로 Confluent Intelligence를 확장합니다. 이러한 기능들을 함께 사용하면 이미 보유한 에이전트를 재사용하고, 이동 중인 복잡한 이상을 탐지하며, 더 많은 벡터 저장소를 RAG(검색 증강 생성) 워크플로우에 통합하고, 네트워킹을 보안하며, 에이전트가 Confluent Cloud의 실시간 데이터에 접근하는 방식을 표준화하는 것이 더 쉬워집니다.

## 2026년 1분기: Confluent Intelligence 신규 기능

Confluent Intelligence의 새로운 기능과 지원을 발표하게 되어 기쁩니다:

- **스트리밍 에이전트를 위한 Agent2Agent(A2A) 통합 (오픈 프리뷰):** 스트리밍 에이전트가 A2A를 지원하는 모든 플랫폼(예: LangChain, CrewAI, SAP, Salesforce)의 외부 에이전트와 협업하고 작업을 오케스트레이션할 수 있도록 하며, 이 모든 것이 안정적이고 재생 가능한 Kafka 백본 위에서 이루어집니다.

- **내장 ML 함수를 위한 다변량 이상 탐지 (얼리 액세스):** 여러 상관 메트릭에서 동시에 이상을 탐지하여, 노이즈가 많은 급등이 아닌 실제 문제를 플래그합니다.

- **Azure Cosmos DB 및 Amazon S3 Vectors를 위한 벡터 검색 (정식 출시):** Azure Cosmos DB와 Amazon S3 Vectors를 MongoDB, Pinecone, Elastic, PostgreSQL, SQL Server, Oracle 등에서 이미 사용하고 있는 것과 동일한 외부 테이블 및 검색 패브릭에 통합하여 Flink에서의 실시간 RAG 파이프라인을 확장합니다.

- **모델 추론, 외부 테이블 및 검색을 위한 AWS 및 Azure Private Link (정식 출시):** Flink에서 외부 데이터베이스, 벡터 저장소, REST API로의 프라이빗 VPC-대-VPC 연결을 통해 안전하게 모델을 호출하고 민감한 데이터로 실시간 스트림을 보강합니다 -- AI 트래픽을 공용 인터넷에 노출시키지 않습니다.

- **Confluent Cloud용 오픈 소스 MCP 서버에 대한 Confluent 지원:** 모든 MCP 클라이언트에 거버넌스가 적용된 실시간 데이터를 제공하는 오픈 소스 MCP 서버에 대해 벤더 공식 지원을 받을 수 있습니다.

*Confluent Intelligence 데모에서 A2A 통합 및 다변량 이상 탐지를 확인하세요.*

각 기능을 더 자세히 살펴보겠습니다.

## A2A 통합: 기존 에이전트를 재사용하고 연결하기

기업이 CRM, 데이터 웨어하우스, 운영 시스템, 맞춤형 애플리케이션 전반에 AI 에이전트를 도입함에 따라, 단독으로는 강력하지만 서로를 쉽게 발견하거나, 통신하거나, 업무를 조율할 수 없는 에이전트 고립 현상이 발생하곤 합니다. 오늘날 이러한 상호작용의 대부분은 포인트-투-포인트 HTTP로 연결되어 있어, 멀티 에이전트 시스템이 성장함에 따라 관찰, 감사, 확장이 어렵습니다.

스트리밍 에이전트를 위한 A2A 통합은 A2A 오픈 프로토콜을 Flink에 직접 통합하여, 이벤트 기반 에이전트가 A2A를 지원하는 모든 플랫폼(예: LangChain, SAP, Salesforce)의 에이전트와 재생 가능한 Kafka 이벤트 스트림 위에서 연결, 오케스트레이션, 협업할 수 있게 합니다. 이미 투자한 에이전트와 프레임워크를 그대로 유지하면서, 이제 이벤트 기반으로 전환하여 오래된 배치 스냅샷 대신 비즈니스의 실시간 상태에 기반하여 행동할 수 있습니다.

*스트리밍 에이전트는 A2A를 지원하는 모든 에이전트와 협업하고 오케스트레이션할 수 있습니다.*

### 스트리밍 A2A 사용 사례: 이벤트 기반 인시던트 워크플로우 오케스트레이션

Confluent, ServiceNow, Salesforce, 그리고 맞춤형 복구 에이전트를 활용하여 통신 인시던트 관리를 운영하는 상황을 상상해 보세요:

- 스트리밍 에이전트가 Kafka 토픽의 텔레메트리 데이터를 모니터링하여 높은 우선순위의 알림(예: 특정 지역에서의 통화 끊김 급증)을 감지합니다.

- 네트워크 이상이 플래그되면, 스트리밍 에이전트는 A2A를 사용하여:

  - ServiceNow 인시던트 분류 에이전트를 트리거하여 최근 로그, 구성 변경 사항, 영향받은 4G/5G 기지국 정보로 티켓을 보강합니다.

  - Salesforce 에이전트를 호출하여 영향받은 기업 및 개인 가입자에 대한 최신 계정 및 서비스 수준 계약(SLA) 컨텍스트를 가져옵니다.

- 각 외부 에이전트가 작업을 완료하면, 결과가 Kafka에 다시 기록되어 복구 에이전트로 스트리밍되며, 이 에이전트가 트래픽 재라우팅, 타겟 고객 커뮤니케이션 발송, 또는 사람에게 에스컬레이션할지 여부를 결정합니다.

이 실시간 이벤트 기반 워크플로우는 Confluent Intelligence와 A2A를 통해 연결되므로, 전체 멀티 에이전트 상호작용을 불변 로그의 일부로 재생하고, 모든 결정을 검사하며, 프로덕션에 영향을 주지 않으면서 새로운 로직을 안전하게 반복 개선할 수 있습니다.

## 다변량 이상 탐지: 단순 검사로는 놓치는 것을 잡아내기

실제 시스템은 단일 메트릭이 임계값을 넘어서 장애가 발생하는 경우가 드뭅니다. 정상적인 카드 거래도 새로운 기기, 인식할 수 없는 위치, 또는 여러 계정에 걸친 비정상적인 지출 속도와 결합되면 의심스러울 수 있습니다. 전통적인 단일 변수 이상 탐지는 이러한 상관관계를 놓치거나, 노이즈나 이상치가 기준선을 왜곡할 때 거짓 양성으로 팀의 주의를 분산시키는 경우가 많습니다.

내장 ML 함수를 위한 다변량 이상 탐지는 여러 메트릭을 단일 벡터로 처리하고 견고한 통계를 적용하여 결합된 행동이 비정상적으로 보일 때를 탐지함으로써 이를 변화시킵니다. 이는 다양한 데이터 조합에서 이상을 찾아 새로운 유형의 실시간 AI 사용 사례를 가능하게 하며, 단일 신호 이상을 요구하는 복잡한 시나리오를 해결하는 데 훨씬 더 강력한 방법입니다. 결과적으로 노이즈 알림이 최소화되고 고객과 비즈니스에 중요한 실제 문제의 조기 탐지가 가능해집니다.

이 기능은 Flink SQL의 새로운 `ML_DETECT_ANOMALIES_ROBUST` 함수를 사용합니다.

*다변량 이상 탐지는 노이즈를 줄이고 정확도를 향상시킵니다.*

### 다변량 사용 사례: 산업 장비의 예측 유지보수

펌프 집합에 대한 예측 유지보수를 생각해 보세요. 장애는 단일 수치로 설명되지 않을 수 있습니다. 초기 이상 징후를 정확하게 탐지하려면 온도, 압력, 진동을 함께 모니터링해야 합니다.

Confluent Intelligence의 다변량 이상 탐지를 사용하면:

- 다양한 공장의 센서 데이터를 Kafka 토픽으로 스트리밍합니다.

- Flink SQL에서 `ROW(temp, pressure, vibration)` 같은 벡터에 `ML_DETECT_ANOMALIES_ROBUST`를 사용하여 시간에 따른 정상 운영 행동을 모델링합니다.

- 각 메트릭이 개별적으로는 정상 범위 내에 있더라도, 메트릭의 조합이 정상에서 벗어나는 이벤트를 플래그합니다.

- 해당 이상 이벤트를 이벤트 기반 스트리밍 에이전트로 전달하여, 유지보수 이력과 결합하고 자산 관리 시스템에서 자동으로 작업 지시서를 생성하거나 우선순위를 지정합니다.

동일한 패턴은 사기 탐지에도 적용되며, 거래 금액, 가맹점 카테고리, 기기 지문, 위치를 벡터로 모델링할 수 있습니다. 또는 전자상거래 시나리오에서 페이지 조회수, 장바구니 이탈, 결제 지연 시간이 함께 일시적인 블립이 아닌 실제 전환 문제를 신호하는 경우에도 적용됩니다.

## Cosmos DB 및 Amazon S3 Vectors를 위한 벡터 검색: RAG 스택 보강하기

오늘날 기업은 운영 데이터베이스, 분석 시스템, 여러 벡터 저장소에 데이터가 분산되어 있습니다. 운영 이벤트는 Kafka로 스트리밍될 수 있지만, 임베딩은 Cosmos DB나 S3 Vectors에 있어 팀이 수집, 임베딩, 벡터 검색, 추론을 취약한 포인트-투-포인트 연결로 하나씩 엮어야 합니다.

*Flink 벡터 검색이 이제 Cosmos DB와 S3 Vectors를 지원합니다.*

Azure Cosmos DB 및 Amazon S3 Vectors에 대한 Flink 벡터 검색은 두 시스템을 MongoDB, Elastic, Couchbase, Pinecone과 함께 Confluent의 외부 테이블 및 검색 패브릭에서 일급 벡터 공급자로 만들어 이 격차를 해소합니다.

이번 릴리스로 다음을 수행할 수 있습니다:

- Flink SQL에서 직접 Cosmos DB 또는 S3 Vectors를 쿼리하여 LLM에 프롬프트하기 전에 실시간으로 k-최근접 이웃(kNN) 결과를 검색합니다.

- 여러 마이크로서비스를 체이닝하는 대신, 수집, 변환, 임베딩 생성, 벡터 검색, 모델 추론을 처리하는 단일 스트리밍 파이프라인을 실행합니다.

- 스트리밍 에이전트와 RAG 애플리케이션을 관련 컨텍스트에 기반시켜 환각을 완화하고 응답 품질을 향상시킵니다.

- 외부 테이블과 검색을 통해 벡터를 제자리에서 쿼리하여 추가 운영 저장소로 데이터를 복제할 필요를 없앱니다.

- 지원되는 모든 벡터 데이터베이스에서 동일한 Flink 및 스트리밍 에이전트 패턴을 사용하여 미래에 대비합니다.

Amazon S3 Vectors의 경우, S3 데이터 레이크 및 Amazon Bedrock 같은 AWS 서비스와 쉽게 통합되는 저비용의 내구성 있는 벡터 스토리지도 제공됩니다. 이를 통해 스트리밍 에이전트가 동일한 Flink 작업에서 S3에서 검색한 시맨틱 컨텍스트와 Bedrock 모델을 결합할 수 있습니다.

### 벡터 검색 사용 사례: 최신 컨텍스트를 활용한 실시간 고객 지원

Cosmos DB 또는 S3 Vectors에 있는 지식 베이스 문서와 최근 통화 녹취록에서 프롬프트 시점에 가장 관련성 높은 컨텍스트가 필요한 고객 지원 에이전트를 구축한다고 가정해 보세요.

Confluent Intelligence를 사용하면:

- 스트리밍 에이전트가 Kafka 토픽에서 들어오는 지원 티켓과 사용자 이벤트를 모니터링합니다.

- 각 티켓에 대해 Flink SQL이 Cosmos DB 또는 S3 Vectors에 대해 벡터 검색을 수행하여 가장 의미적으로 관련성 높은 콘텐츠를 검색합니다.

- 스트리밍 에이전트가 이러한 벡터 검색 결과를 실시간 스트림(예: 로열티 상태, 최근 브라우징 및 장바구니 활동, 주문/환불 이력)과 결합하고 LLM을 호출하여 초개인화된 추천을 생성합니다.

이 모든 것이 동일한 Flink 스트림 처리 파이프라인에서 실행되므로, 별도의 수집 작업, 맞춤형 RAG 서비스, 벡터 데이터베이스와 데이터 소스 간의 임시 동기화에 따른 추가 비용과 복잡성을 피하면서 AI와 데이터 처리 워크플로우를 통합할 수 있습니다.

## 모델 추론, 외부 테이블 및 벡터 검색을 위한 AWS 및 Azure Private Link: AI를 위한 보안 네트워킹

많은 팀에게 있어, AI 워크플로우를 민감한 기록 시스템에 연결하는 마지막 단계의 문제는 API나 스키마가 아닙니다. 바로 네트워킹입니다.

모델 추론, 외부 테이블 및 벡터 검색을 위한 AWS 및 Azure Private Link는 Flink에서 외부 모델 및 시스템으로의 프라이빗 VPC-대-VPC 연결을 가능하게 합니다. 따라서 Confluent Intelligence(스트리밍 에이전트 포함)가 공용 인터넷을 거치지 않고 LLM을 호출하고 외부 데이터베이스, 벡터 저장소, REST 엔드포인트의 민감하고 독점적인 기업 데이터로 실시간 스트림을 보강할 수 있습니다. 많은 조직에서 이는 프로덕션 배포의 장애물을 해소하고 정보 보안 컴플라이언스를 보장하는 데 도움이 됩니다.

Private Link를 통해 다음을 수행할 수 있습니다:

- **AI 트래픽을 프라이빗하고 컴플라이언스에 맞게 유지:** Private Link를 사용하여 CRM, ERP(전사적 자원 관리), 벡터 저장소, REST API에 대한 조회가 프라이빗 네트워크 경로를 통해 이루어지도록 합니다.

- **Kafka 스트림을 기록 시스템과 안전하게 결합:** 엄격한 보안 요구사항을 충족하면서 스트리밍 에이전트에 최신의 완전한 컨텍스트(예: 고객 기록, 주문, 정책)를 제공합니다.

- **네트워킹 복잡성 감소:** 임시 프록시와 터널을 Confluent의 관리형 네트워킹에 기반한 표준화되고 클라우드 네이티브한 프라이빗 연결로 대체합니다.

## Confluent Cloud용 오픈 소스 MCP 서버 지원: 모든 MCP 클라이언트를 위한 프로덕션 지원

Anthropic의 MCP를 도입하는 팀은 오픈 소스 Confluent MCP 서버를 사용하여 AI 에이전트가 Confluent Cloud -- Kafka, Flink, 커넥터, Tableflow, Schema Registry 등 -- 에 간단하고 표준화된 방식으로 접근하기를 원합니다. 지금까지 이러한 팀은 대부분 자체적으로 MCP 서버를 배포하고 운영하며, 각 MCP 클라이언트(예: Claude Desktop, Goose, Gemini CLI)에 연결하고, 공식 벤더 지원 없이 문제를 처리해야 했습니다.

Confluent는 이제 Confluent Cloud용 오픈 소스 MCP 서버에 대한 공식 지원을 제공하여, MCP 기반 에이전트와 Confluent의 실시간 거버넌스 적용 데이터 사이의 프로덕션 준비 완료, 벤더 공식 지원 브릿지로 전환합니다.

이 지원은 다음을 의미합니다:

- **MCP 에이전트에 최신 Confluent Cloud 데이터 및 운영에 대한 직접 액세스를 제공합니다** -- 단순히 Kafka 토픽 관리를 넘어 커넥터, Flink, Tableflow, 빌링 및 메트릭 API 지원 등을 포함합니다. 이는 역압(backpressure) 같은 문제를 디버깅하는 데에도 활용할 수 있습니다.

- **자연어로 Confluent를 관리합니다** -- 코드 없이 MCP 클라이언트에서 토픽을 구성하고, Flink SQL을 실행하며, 데이터 인프라와 상호작용합니다.

- **GitHub 또는 계정 팀을 통해 이슈를 제기하면**, Confluent 엔지니어링이 정의된 SLA에 따라 문제를 해결합니다.

## Confluent Intelligence로 실시간 컨텍스트 풍부한 시스템 구축 시작하기

AI의 잠재력을 최대한 활용하려면, AI 에이전트와 애플리케이션이 비즈니스의 실시간 상태를 보고, 이해하고, 이에 따라 행동할 수 있어야 합니다. Confluent Intelligence는 컨텍스트 엔지니어링을 위한 스트리밍 및 스트림 처리, ML, RAG를 위한 벡터 검색, 스트리밍 에이전트, 그리고 MCP 및 A2A 같은 오픈 프로토콜을 완전 관리형 거버넌스 플랫폼에서 결합하여 이를 가능하게 합니다.
