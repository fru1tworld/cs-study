# RFC 4632: Classless Inter-Domain Routing (CIDR)

## 개요

RFC 4632는 IPv4 주소를 할당하면서 주소 공간을 보존하고 글로벌 라우팅 테이블의 성장을 제한하는 전략을 문서화한 문서입니다.

## 핵심 문제점과 해결책

### 기존 문제점

원래의 IPv4 주소 체계는 세 가지 고정된 클래스(A, B, C)를 사용했는데, 이로 인해 두 가지 심각한 문제가 발생했습니다:

1. **Class B 주소의 급격한 고갈**: Class B 주소가 빠르게 소진되고 있었음
2. **라우팅 테이블의 기하급수적 증가**: 전 세계 라우팅 테이블이 급격히 커지고 있었음

### CIDR의 해결책

CIDR은 이러한 경직된 클래스 시스템을 **유연하고 계층적인 프리픽스 기반 주소 체계**로 대체했습니다.

## 핵심 개념

### 프리픽스 표기법 (Prefix Notation)

주소는 `a.b.c.d/X` 형식으로 표기됩니다. 여기서 X는 네트워크 부분을 정의하는 비트 수를 나타냅니다.

**예시:**
- `172.16.0.0/16`: 기존 Class B 네트워크에 해당, 65,536개의 주소 보유
- `192.168.1.0/24`: 256개의 주소 보유
- `10.0.0.0/8`: 16,777,216개의 주소 보유

### 유연한 크기 조정 (Flexible Sizing)

기존의 세 가지 고정 클래스와 달리, CIDR은 **2의 거듭제곱 크기의 모든 주소 블록**을 허용합니다:

| 프리픽스 | 주소 개수 | 설명 |
|---------|----------|------|
| /8 | 16,777,216 | 대규모 조직용 |
| /16 | 65,536 | 중규모 조직용 |
| /24 | 256 | 소규모 네트워크용 |
| /32 | 1 | 단일 호스트 |

## 계층적 주소 분배

주소 공간은 여러 레지스트리 레벨을 통해 분배됩니다:

```
IANA (Internet Assigned Numbers Authority)
    ↓ /8 블록 할당
Regional Internet Registries (RIRs)
    ↓ 더 작은 블록 할당
Internet Service Providers (ISPs)
    ↓ 추가 블록 위임
최종 고객 (End Customers)
```

### 분배 구조

1. **IANA**: /8 블록을 지역 인터넷 레지스트리(RIR)에 할당
2. **지역 인터넷 레지스트리 (RIR)**: ISP에 더 작은 블록을 할당
   - ARIN (북미)
   - RIPE NCC (유럽, 중동, 중앙아시아)
   - APNIC (아시아 태평양)
   - LACNIC (라틴 아메리카, 카리브해)
   - AFRINIC (아프리카)
3. **ISP**: 고객에게 추가적인 블록 위임

이 분산 모델은 이전의 중앙 집중식 권한 체계를 대체했습니다.

## 라우팅 규칙

### 1. 최장 프리픽스 매칭 (Longest-Prefix Matching)

포워딩은 **최장 매칭 라우팅**을 사용합니다. 더 구체적인 프리픽스가 우선순위를 가집니다.

**예시:**
- 목적지: `192.168.1.100`
- 라우팅 테이블:
  - `192.168.0.0/16` → Router A
  - `192.168.1.0/24` → Router B
- 결과: `/24`가 더 구체적이므로 **Router B**로 전송

### 2. 집계 라우트 (Aggregate Routes)

집계 라우트는 해당 집계에는 매칭되지만 특정 서브넷에는 매칭되지 않는 트래픽을 **폐기(discard)**해야 합니다.

## 서브넷 마스크와 CIDR 표기법

### 서브넷 마스크 대응표

| CIDR 표기 | 서브넷 마스크 | 호스트 수 |
|-----------|--------------|----------|
| /8 | 255.0.0.0 | 16,777,214 |
| /16 | 255.255.0.0 | 65,534 |
| /24 | 255.255.255.0 | 254 |
| /25 | 255.255.255.128 | 126 |
| /26 | 255.255.255.192 | 62 |
| /27 | 255.255.255.224 | 30 |
| /28 | 255.255.255.240 | 14 |
| /29 | 255.255.255.248 | 6 |
| /30 | 255.255.255.252 | 2 |
| /31 | 255.255.255.254 | 2 (점대점) |
| /32 | 255.255.255.255 | 1 |

## 영향과 결과

### 긍정적 효과

- **IPv4 고갈 지연**: CIDR 배포로 IPv4 주소 고갈 속도가 늦춰짐
- **라우팅 성장 안정화**: 1990년대를 통해 라우팅 테이블 성장이 안정화됨
- **효율적인 주소 사용**: 필요한 만큼만 주소를 할당할 수 있게 됨

### 한계점

2005년 이후로, 다음과 같은 이유로 기하급수적 성장이 재개되었습니다:
- **멀티호밍 (Multi-homing)**: 여러 ISP에 연결하기 위한 추가 경로 광고
- **트래픽 엔지니어링**: 더 구체적인 경로 광고가 전 세계적으로 필요해짐

## 실제 활용 예시

### 주소 할당 예시

회사에 1,000개의 호스트가 필요한 경우:

```
기존 클래스 방식:
- Class C (/24) = 256개 주소 → 부족
- Class B (/16) = 65,536개 주소 → 과다

CIDR 방식:
- /22 = 1,024개 주소 → 적절!
```

### 슈퍼넷팅 예시

여러 개의 /24 네트워크를 하나로 집약:

```
192.168.0.0/24
192.168.1.0/24
192.168.2.0/24
192.168.3.0/24
    ↓ 집약
192.168.0.0/22 (하나의 라우팅 엔트리)
```

## 요약

CIDR은 인터넷의 지속적인 성장을 가능하게 한 핵심 기술입니다. 클래스 기반 주소 체계의 비효율성을 해결하고, 유연한 주소 할당과 효율적인 라우팅을 가능하게 했습니다. 오늘날 모든 네트워크 엔지니어가 반드시 이해해야 하는 기본 개념입니다.
