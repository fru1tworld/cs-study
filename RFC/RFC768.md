## 1. 소개 (Introduction)

### 1.1 정의
- UDP는 패킷 스위칭 컴퓨터 통신 환경에서 **데이터그램 모드**의 통신을 제공하기 위해 정의됨
- **interconnected set of computer networks** 환경에서 동작

### 1.2 기본 프로토콜
- IP (Internet Protocol)를 하위 프로토콜로 사용

### 1.3 주요 특성
- **최소한의 프로토콜 메커니즘** 제공
- 애플리케이션 프로그램이 다른 프로그램에 메시지를 전송하는 절차 제공
- **트랜잭션 지향적** (transaction oriented)

### 1.4 보장하지 않는 것
- 전달 보장 없음 (no guaranteed delivery)
- 중복 방지 없음 (no duplicate protection)

### 1.5 권장 사항
- 순서가 보장된 신뢰성 있는 데이터 스트림 전달이 필요한 경우 → **TCP (Transmission Control Protocol) 사용 권장**

## 2. 형식 (Format)

### 2.1 UDP 헤더 다이어그램

```
                  0      7 8     15 16    23 24    31
                 +--------+--------+--------+--------+
                 |     Source      |   Destination   |
                 |      Port       |      Port       |
                 +--------+--------+--------+--------+
                 |                 |                 |
                 |     Length      |    Checksum     |
                 +--------+--------+--------+--------+
                 |
                 |          data octets ...
                 +---------------- ...

                      User Datagram Header Format
```

### 2.2 필드 상세 설명

| 필드명 | 크기 | 비트 위치 | 설명 |
|--------|------|-----------|------|
| Source Port | 16비트 | 0-15 | 송신 포트 번호 |
| Destination Port | 16비트 | 16-31 | 수신 포트 번호 |
| Length | 16비트 | 32-47 | 데이터그램 전체 길이 |
| Checksum | 16비트 | 48-63 | 체크섬 |

---

## 3. 필드 설명 (Fields)

### 3.1 Source Port (송신 포트)
- **크기**: 16비트
- **특성**: 선택적 필드 (optional field)
- **용도**: 응답이 필요할 때 송신 프로세스의 포트 표시
- **미사용 시**: 0으로 설정

### 3.2 Destination Port (수신 포트)
- **크기**: 16비트
- **특성**: 필수 필드
- **용도**: 특정 인터넷 목적지 주소 컨텍스트 내에서 의미를 가짐
- **설명**: 목적지 호스트의 특정 애플리케이션/프로세스 식별

### 3.3 Length (길이)
- **크기**: 16비트
- **단위**: 옥텟 (octets)
- **범위**: 헤더와 데이터를 포함한 사용자 데이터그램의 **전체 길이**
- **최솟값**: 8 (헤더만 있는 경우)
- **계산**: Length = UDP 헤더(8 옥텟) + 데이터 길이

### 3.4 Checksum (체크섬)
- **크기**: 16비트
- **계산 방식**: 16비트 1의 보수 합(one's complement sum)의 1의 보수
- **계산 범위**:
  1. IP 헤더에서 추출한 의사 헤더 (pseudo header)
  2. UDP 헤더
  3. 데이터
- **패딩**: 데이터가 홀수 옥텟인 경우, 마지막에 0 옥텟 패딩 추가 (체크섬 계산용, 실제 전송 안 함)

#### 체크섬 특수 규칙

| 상황 | 처리 방법 |
|------|-----------|
| 계산 결과가 0인 경우 | 모두 1로 전송 (0xFFFF) |
| 전송된 값이 0인 경우 | 송신자가 체크섬을 생성하지 않았음을 의미 |
| 체크섬 미사용 시 | 디버깅 또는 상위 레벨 프로토콜에서 체크섬 불필요한 경우 |

## 4. 의사 헤더 (Pseudo Header)

### 4.1 의사 헤더 다이어그램

```
                  0      7 8     15 16    23 24    31
                 +--------+--------+--------+--------+
                 |          source address           |
                 +--------+--------+--------+--------+
                 |        destination address        |
                 +--------+--------+--------+--------+
                 |  zero  |protocol|   UDP length    |
                 +--------+--------+--------+--------+
```

### 4.2 의사 헤더 필드 설명

| 필드명 | 크기 | 설명 |
|--------|------|------|
| Source Address | 32비트 | IP 헤더의 송신 IP 주소 |
| Destination Address | 32비트 | IP 헤더의 수신 IP 주소 |
| Zero | 8비트 | 0으로 고정 |
| Protocol | 8비트 | IP 프로토콜 번호 (UDP = 17) |
| UDP Length | 16비트 | UDP 데이터그램 길이 (UDP 헤더의 Length 필드와 동일) |

### 4.3 의사 헤더의 목적
- UDP 데이터그램이 **올바른 목적지에 도착했는지 검증**
- 의사 헤더는 **개념적으로 UDP 헤더 앞에 위치**
- **체크섬 계산에만 사용** (실제 전송되지 않음)

---

## 5. 사용자 인터페이스 (User Interface)

### 5.1 필수 기능
사용자 인터페이스는 다음 기능을 제공해야 함:

| 기능 | 설명 |
|------|------|
| 새 수신 포트 생성 | 새로운 수신 포트를 만들 수 있어야 함 |
| 데이터 수신 | 수신 포트에서 데이터 옥텟을 반환하고, 송신 포트와 송신 주소를 표시 |
| 데이터 송신 | 데이터, 송신 포트, 송신 주소, 수신 포트, 수신 주소를 지정하여 데이터그램 전송 |

---

## 6. IP 인터페이스 (IP Interface)

### 6.1 UDP 모듈의 역할
- 인터넷 헤더에서 다음 필드들을 확인해야 함:
  - **송신 주소** (source address)
  - **수신 주소** (destination address)
  - **프로토콜 필드** (protocol field)

### 6.2 UDP-IP 관계
- UDP는 IP 위에서 동작하는 상위 계층 프로토콜
- IP가 제공하는 데이터그램 서비스를 그대로 활용
