## 요약 형식 규칙

### 개조식 작성 원칙
```markdown
## 섹션 제목

### 하위 주제
- 핵심 포인트 1
  - 세부 사항 a
  - 세부 사항 b
- 핵심 포인트 2
```

### JWT 구조 다이어그램
```
Header.Payload.Signature

예시:
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.
eyJzdWIiOiIxMjM0NTY3ODkwIiwibmFtZSI6IkpvaG4gRG9lIiwiaWF0IjoxNTE2MjM5MDIyfQ.
SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c
```

### 등록된 클레임 표
```markdown
| 클레임 | 전체 이름 | 설명 | 타입 |
|--------|----------|------|------|
| iss | Issuer | 토큰 발급자 | StringOrURI |
| sub | Subject | 토큰 주체 | StringOrURI |
| aud | Audience | 토큰 수신자 | StringOrURI |
| exp | Expiration Time | 만료 시간 | NumericDate |
| nbf | Not Before | 활성화 시간 | NumericDate |
| iat | Issued At | 발급 시간 | NumericDate |
| jti | JWT ID | 고유 식별자 | String |
```

## 검증 방법

### 완료 후 체크
1. JWT의 세 부분(Header, Payload, Signature)이 설명되었는지 확인
2. 모든 등록된 클레임이 설명되었는지 확인
3. JWT 생성 및 검증 절차가 설명되었는지 확인
4. 보안 고려사항이 완전히 설명되었는지 확인
