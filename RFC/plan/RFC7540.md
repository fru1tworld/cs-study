## 요약 형식 규칙

### 개조식 작성 원칙
```markdown
## 섹션 제목

### 하위 주제
- 핵심 포인트 1
  - 세부 사항 a
  - 세부 사항 b
- 핵심 포인트 2
```

### 프레임 구조 다이어그램
```
+-----------------------------------------------+
|                 Length (24)                   |
+---------------+---------------+---------------+
|   Type (8)    |   Flags (8)   |
+-+-------------+---------------+-------------------------------+
|R|                 Stream Identifier (31)                      |
+=+=============================================================+
|                   Frame Payload (0...)                      ...
+---------------------------------------------------------------+
```

### 프레임 타입 표
```markdown
| Type | 이름 | 코드 |
|------|------|------|
| DATA | 데이터 | 0x0 |
| HEADERS | 헤더 | 0x1 |
| PRIORITY | 우선순위 | 0x2 |
| RST_STREAM | 스트림 리셋 | 0x3 |
| SETTINGS | 설정 | 0x4 |
| PUSH_PROMISE | 푸시 약속 | 0x5 |
| PING | 핑 | 0x6 |
| GOAWAY | 연결 종료 | 0x7 |
| WINDOW_UPDATE | 윈도우 업데이트 | 0x8 |
| CONTINUATION | 연속 | 0x9 |
```
