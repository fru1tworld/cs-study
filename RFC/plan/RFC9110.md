## 요약 형식 규칙

### 개조식 작성 원칙
```markdown
## 섹션 제목

### 하위 주제
- 핵심 포인트 1
  - 세부 사항 a
  - 세부 사항 b
- 핵심 포인트 2
```

### HTTP 메서드 표
```markdown
| 메서드 | 안전 | 멱등 | 설명 |
|--------|------|------|------|
| GET | Yes | Yes | 리소스 조회 |
| HEAD | Yes | Yes | 헤더만 조회 |
| POST | No | No | 리소스 생성/처리 |
| PUT | No | Yes | 리소스 대체 |
| DELETE | No | Yes | 리소스 삭제 |
| CONNECT | No | No | 터널 설정 |
| OPTIONS | Yes | Yes | 옵션 조회 |
| TRACE | Yes | Yes | 루프백 테스트 |
```

### 상태 코드 범위 표
```markdown
| 범위 | 분류 | 의미 |
|------|------|------|
| 1xx | Informational | 요청 수신, 처리 계속 |
| 2xx | Successful | 요청 성공 |
| 3xx | Redirection | 추가 동작 필요 |
| 4xx | Client Error | 클라이언트 오류 |
| 5xx | Server Error | 서버 오류 |
```
