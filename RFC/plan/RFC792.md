# RFC 792 요약 작업 지시서

## 대상 문서
- **RFC 번호**: 792
- **제목**: Internet Control Message Protocol (ICMP)
- **원본 URL**: https://www.rfc-editor.org/rfc/rfc792

---

## 작업 목표
RFC 792 원문을 **개조식으로 완전 요약**하되, **어떠한 내용도 누락하지 않아야 함**.

---

## 작업 지침

### 1. 원문 섹션별 완전 매핑
원본 RFC의 모든 섹션을 식별하고, 각 섹션의 **모든 내용**을 빠짐없이 포함해야 합니다.

### 2. 누락 방지 체크리스트

다음 항목들이 반드시 포함되어야 합니다:

#### 문서 메타정보
- [ ] RFC 번호, 발행일
- [ ] 프로토콜 번호 (Protocol = 1)
- [ ] 문서 상태

#### Introduction
- [ ] ICMP의 목적과 역할
- [ ] IP와의 관계 (IP의 일부로서 구현)
- [ ] 게이트웨이/호스트에서의 사용

#### 모든 ICMP 메시지 타입
각 메시지 타입별로 다음 항목 포함:
- [ ] Type 번호
- [ ] Code 값들과 의미
- [ ] 메시지 형식 다이어그램
- [ ] 각 필드 설명
- [ ] 언제 생성되는지
- [ ] 어떻게 처리해야 하는지

#### 메시지 타입 전체 목록
- [ ] Type 0: Echo Reply
- [ ] Type 3: Destination Unreachable
  - [ ] Code 0: net unreachable
  - [ ] Code 1: host unreachable
  - [ ] Code 2: protocol unreachable
  - [ ] Code 3: port unreachable
  - [ ] Code 4: fragmentation needed and DF set
  - [ ] Code 5: source route failed
- [ ] Type 4: Source Quench
- [ ] Type 5: Redirect
  - [ ] Code 0: network
  - [ ] Code 1: host
  - [ ] Code 2: ToS and network
  - [ ] Code 3: ToS and host
- [ ] Type 8: Echo Request
- [ ] Type 11: Time Exceeded
  - [ ] Code 0: TTL exceeded in transit
  - [ ] Code 1: fragment reassembly time exceeded
- [ ] Type 12: Parameter Problem
- [ ] Type 13: Timestamp Request
- [ ] Type 14: Timestamp Reply
- [ ] Type 15: Information Request (deprecated)
- [ ] Type 16: Information Reply (deprecated)

#### 공통 헤더 구조
- [ ] Type 필드 (8비트)
- [ ] Code 필드 (8비트)
- [ ] Checksum 필드 (16비트)
- [ ] 체크섬 계산 방법

#### 에러 메시지 공통 규칙
- [ ] ICMP 에러 메시지에 대한 ICMP 에러를 보내면 안 됨
- [ ] 첫 번째 fragment가 아닌 경우 ICMP 보내면 안 됨
- [ ] 브로드캐스트/멀티캐스트에 대한 ICMP 제한

## 검증 방법

### 완료 후 체크
1. 모든 Type 번호가 포함되었는지 확인
2. 각 Type의 모든 Code 값이 설명되었는지 확인
3. 모든 메시지 형식 다이어그램이 포함되었는지 확인
4. 에러 메시지 생성 규칙이 명시되었는지 확인
