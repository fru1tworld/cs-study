# RFC 768 요약 작업 지시서

## 대상 문서
- **RFC 번호**: 768
- **제목**: User Datagram Protocol (UDP)
- **원본 URL**: https://www.rfc-editor.org/rfc/rfc768

---

## 작업 목표
RFC 768 원문을 **개조식으로 완전 요약**하되, **어떠한 내용도 누락하지 않아야 함**.

---

## 작업 지침

### 1. 원문 섹션별 완전 매핑
원본 RFC의 모든 섹션을 식별하고, 각 섹션의 **모든 내용**을 빠짐없이 포함해야 합니다.

### 2. 누락 방지 체크리스트

다음 항목들이 반드시 포함되어야 합니다:

#### 문서 메타정보
- [ ] RFC 번호, 발행일, 저자
- [ ] 문서 상태 (Standard, Informational 등)
- [ ] 관련/대체 RFC 번호

#### 핵심 내용
- [ ] Introduction/개요 - 프로토콜의 목적과 배경
- [ ] Format/형식 - 헤더 구조 다이어그램 (ASCII 아트 포함)
- [ ] 각 필드 설명 - 모든 필드의 크기, 용도, 유효값
- [ ] Pseudo Header - 체크섬 계산을 위한 의사 헤더 구조
- [ ] 체크섬 계산 규칙 (0일 경우 처리 방법 포함)
- [ ] User Interface - 사용자 인터페이스 설명 (있는 경우)
- [ ] IP Interface - IP 계층과의 인터페이스
- [ ] Protocol Application - 적용 사례 (Name Server, TFTP 등)
- [ ] Protocol Number - 프로토콜 식별 번호 (17)

#### 수치/상수 값
- [ ] 헤더 크기 (8 옥텟)
- [ ] Length 필드 최소값 (8)
- [ ] 프로토콜 번호 (17)
- [ ] 포트 번호 범위

## 요약 형식 규칙

### 개조식 작성 원칙
```markdown
## 섹션 제목

### 하위 주제
- 핵심 포인트 1
  - 세부 사항 a
  - 세부 사항 b
- 핵심 포인트 2
```

### 다이어그램 보존
원본의 ASCII 다이어그램은 **그대로 유지**하거나 **더 명확하게 개선**할 수 있으나, 정보 손실 없이 유지해야 함.

### 표 활용
관련 정보는 표로 정리하여 가독성 향상:
```markdown
| 필드명 | 크기 | 설명 |
|--------|------|------|
| ... | ... | ... |
```

## 검증 방법

### 완료 후 체크
1. 원본 RFC의 각 섹션을 순회하며 대응 내용이 있는지 확인
2. 원본에 있는 모든 숫자값/상수가 요약본에 있는지 확인
3. 원본에 있는 모든 용어/정의가 요약본에 있는지 확인
4. ASCII 다이어그램이 정보 손실 없이 포함되었는지 확인

### 비교 검토
- 원본 문장 수 vs 요약 항목 수 비교
- 누락된 섹션이 없는지 목차 대조
