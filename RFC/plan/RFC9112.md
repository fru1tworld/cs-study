## 요약 형식 규칙

### 개조식 작성 원칙
```markdown
## 섹션 제목

### 하위 주제
- 핵심 포인트 1
  - 세부 사항 a
  - 세부 사항 b
- 핵심 포인트 2
```

### HTTP/1.1 메시지 구조
```
HTTP-message = start-line CRLF
               *( field-line CRLF )
               CRLF
               [ message-body ]
```

### 요청 메시지 예시
```
GET /path HTTP/1.1
Host: example.com
User-Agent: Mozilla/5.0
Accept: text/html
                          <- 빈 줄 (CRLF)
[message-body]
```

### 응답 메시지 예시
```
HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234
                          <- 빈 줄 (CRLF)
[message-body]
```

### Chunked 인코딩 형식
```
chunked-body = *chunk
               last-chunk
               trailer-section
               CRLF

chunk = chunk-size [ chunk-ext ] CRLF
        chunk-data CRLF
```

### 전송 코딩 표
```markdown
| 코딩 | 설명 |
|------|------|
| chunked | 청크 단위 전송 |
| compress | LZW 압축 |
| deflate | zlib 압축 |
| gzip | GNU zip 압축 |
```

## 검증 방법

### 완료 후 체크
1. 메시지 형식이 완전히 설명되었는지 확인
2. 요청 라인과 상태 라인이 상세히 설명되었는지 확인
3. Chunked 전송 코딩이 완전히 설명되었는지 확인
4. 연결 관리가 완전히 설명되었는지 확인
5. 보안 고려사항(Request Smuggling, Response Splitting)이 설명되었는지 확인
