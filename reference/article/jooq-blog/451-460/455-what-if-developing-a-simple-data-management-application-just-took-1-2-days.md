# 간단한 데이터 관리 애플리케이션 개발이 1-2일이면 된다면?

> 원문: https://blog.jooq.org/what-if-developing-a-simple-data-management-application-just-took-1-2-days/

*2014년 5월 28일 Lukas Eder 작성*

"애플리케이션 개발이 1-2일이면 끝난다면 어떨까요? 제가 직접 10번의 클릭만으로 만들 수 있다면요? 개발자 여러분이 더 이상 필요 없다면요?"

이것은 관리자들의 오랜 꿈입니다. 그리고 엔지니어들은 보통 이런 가능성에 회의적입니다.

## 데이터 변환과 탐색

저는 항상 데이터 조작에 대한 비절차적 접근 방식에 열정을 가지고 있었습니다. SQL과 XSLT가 좋은 예입니다. 이 글을 쓰기 약 15년 전에 Ergon에서 만든 JTT(Java Table Tool)라는 Swing 데스크톱 애플리케이션을 경험했습니다.

### JTT의 기능

- 접근 가능한 모든 테이블 개요
- 필터링 및 그룹화 기능이 있는 편집 가능한 레코드 목록
- 개별 레코드에 대한 상세 팝업
- 외래 키 관계를 위한 탭이 있는 "자식(Children)" 뷰
- 기술적인 ID 대신 관련 연결 데이터 표시
- CSV, XLS 또는 PDF 형식으로 내보내기 기능

이것은 제가 본 것 중 기술적으로 가장 훌륭한 제품 중 하나였습니다. 정말 간결하고, 단순하면서도 강력했습니다.

Ergon은 JTT를 내부적으로 "회계, 송장 관리, CRM, HR 도구"로 사용했습니다. 저는 이 도구를 상용화해야 한다고 주장했으며, Eclipse RAP 같은 기술로 현대화할 수 있다고 제안했습니다.

## Espresso Logic의 발견

나중에 저는 유사한 기능이 이미 Espresso Logic의 Live Browser를 통해 존재한다는 것을 발견했습니다. InfoQ와 DZone에서 Val Huber의 기사를 통해 이를 알게 되었습니다. 이 도구는 자동화된 REST API 생성과 반응형 데이터 기반 애플리케이션을 제공합니다.

### Live Browser 기능

- 테이블 선택 및 탐색
- 필터링 기능
- 상세 뷰
- 외래 키 탐색
- 자식 레코드 탐색
- 데이터 조작
- 역할 기반, 행/열 수준 권한
- 반응형 프로그래밍 기능

이 도구는 실제로 앞서 언급한 반응형 REST API 위에 구축되어 있어, 모든 훌륭한 기능을 상속받습니다.

저는 SaaS가 본격화되기 시작한 3년 전에 이 제품을 만들었으면 좋았을 텐데 하는 아쉬움이 있습니다.

## 사용 사례

Live Browser는 다음과 같은 용도로 활용될 수 있습니다:

- 데이터 탐색에서 Toad나 SQL Developer보다 우수한 범용 데이터베이스 검사 도구
- 비즈니스 요구사항에 대해 데이터베이스 스키마를 검증하기 위한 프로토타이핑 엔진

## 결론

저는 상업적 기회를 놓쳤음을 인정합니다. Espresso Logic이 이를 성취한 것을 축하하며, Caspio나 StratusBase 같은 제품들과 함께 신속한 애플리케이션 개발 도구 시장이 얼마나 붐비고 있는지 생각해봅니다.

이 분야는 HyperCard, FileMaker Pro, Microsoft Access의 유산을 잇는 시장입니다.

관리자들이 즉각적인 애플리케이션 개발을 꿈꾸는 동안, 특화된 도구들이 이제 이전보다 더 빠른 데이터 관리 애플리케이션 생성을 가능하게 합니다.
