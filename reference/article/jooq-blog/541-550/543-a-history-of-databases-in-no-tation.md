# "No-테이션"으로 본 데이터베이스의 역사

> 원문: https://blog.jooq.org/a-history-of-databases-in-no-tation/

2013년 11월 15일

최근 데이터베이스 기술 분야에서 흥미로운 일들이 벌어지고 있습니다. 인메모리 컴퓨팅과 전통적인 SQL 데이터베이스의 융합이 바로 그것입니다.

## 인메모리 컴퓨팅의 혁명

GridGain Systems의 설립자인 Nikita Ivanov는 DRAM의 가격 하락으로 인해 인메모리 데이터베이스가 이제 중소기업에게도 접근 가능해졌다고 강조했습니다. Ivanov에 따르면:

> "모든 기업의 99%는 10TB 미만의 트랜잭션 데이터를 보유하고 있습니다."

이러한 데이터를 이제 15,000달러 미만의 비용으로 메모리에 저장할 수 있게 되었습니다 - 이는 전통적인 데이터베이스 라이선스 비용에 비하면 무시해도 될 수준입니다.

## 데이터 처리 패러다임의 전환

Hazelcast의 Christoph Engelbert는 현대 데이터 처리의 패러다임 전환에 대해 발표했습니다:

> "데이터를 연산 쪽으로 이동시키는 것이 아니라, 연산을 데이터 쪽으로 이동시키는 것입니다."

저자는 이 원칙이 PL/SQL이나 T-SQL과 같은 절차적 SQL 언어를 사용하는 전통적인 OLAP 시스템에도 동일하게 적용된다고 지적합니다. 전통적인 데이터베이스도 이러한 언어들을 통해 시스템 내에서 직접 복잡한 알고리즘을 실행할 수 있으며, 이는 현대 분산 컴퓨팅 접근 방식과 비교할 수 있는 원칙입니다.

## SQL의 부활

이 글은 ORM(Object-Relational Mapping) 프레임워크에 대한 불만족의 증가와 NoSQL의 미래에 대한 혼란이 "언어로서의 SQL의 부활"로 이어졌다고 연결 짓습니다. 저자는 이러한 추세에 대해 열정적으로 말합니다.

특히 O'Reilly의 Strata Conference에서 연구원 Mark Madsen이 선보인 유머러스한 "History of NoSQL(NoSQL의 역사)" 티셔츠를 언급합니다. 이 시각적 코멘터리는 SQL의 귀환을 암시합니다.

티셔츠의 내용은 다음과 같은 데이터베이스 기술의 순환적 역사를 보여줍니다:

```
1970년대: 관계형 데이터베이스 등장
1980년대: SQL 표준화
1990년대: 객체지향 데이터베이스
2000년대: NoSQL 운동 시작
2010년대: NoSQL... 아, 잠깐, 우리 SQL이 그리워...
          NewSQL의 등장
          "SQL이 돌아왔다!"
```

저자는 이 티셔츠가 데이터베이스 기술이 결국 입증된 SQL 패러다임으로 다시 순환한다는 점을 유머러스하게 보여준다고 설명합니다.

## 결론

이 글은 다음과 같은 관찰로 결론을 맺습니다:

> "SQL이 돌아왔습니다. 그리고 그 어느 때보다 강력합니다."

수년간의 대안적 데이터베이스 패러다임의 인기에도 불구하고, SQL은 현대 인프라 개선을 수용하면서도 지속적인 솔루션으로서의 위치를 유지하고 있습니다. 인메모리 컴퓨팅의 발전, 분산 처리의 새로운 패러다임, 그리고 NoSQL에 대한 환멸이 모두 SQL의 르네상스를 이끄는 요인이 되고 있습니다.
