# Java 8을 광고할 때 자신감이 어디 갔는가, Oracle?

> 원문: https://blog.jooq.org/wheres-the-self-confidence-when-advertising-java-8-oracle/

2013년 11월 17일 - Lukas Eder

저는 최근 Oracle의 Java 8 마케팅 방식에 대해 비판적인 시각을 갖게 되었습니다. Brian Goetz가 Java 8의 성과를 발표할 때, 그것을 "적절한 타협안(decent compromise)"으로 표현하는 모습을 보았습니다. 하지만 이것은 Java 8이 실제로 가져오는 혁신에 비해 너무 겸손한 표현이었습니다.

람다 표현식과 디펜더 메서드(defender methods, 인터페이스의 기본 메서드)는 Java 언어 초창기 이래로 가장 중요한 개선 사항 중 하나입니다. 이 기능들은 .NET의 성공적인 LINQ 구현과 견줄 수 있는 수준의 능력을 제공합니다.

저는 이전에 Java 8과 LINQ를 비교 분석한 글을 작성한 적이 있습니다. Java 커뮤니티는 이 논의를 긍정적으로 받아들였지만, .NET 커뮤니티는 Erik Meijer를 포함하여 회의적인 반응을 보였습니다. 일부에서 이런 비판이 있었음에도 불구하고, 더 넓은 Java 커뮤니티는 Java 8의 기능을 열렬히 기대하고 있었습니다.

제 핵심 주장은 이것입니다: Oracle은 Java 8을 진정으로 혁신적인 기술로서 더 큰 자신감과 열정을 가지고 광고했어야 했습니다. 머뭇거리며 소개하는 대신 말입니다.

Java 8은 단순한 "타협안"이 아닙니다. 이것은 Java 생태계에 함수형 프로그래밍 패러다임을 도입하는 진정한 게임 체인저입니다. Oracle은 이 사실을 더 자랑스럽게 알렸어야 했습니다.
