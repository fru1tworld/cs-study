# 소스 코드 주석이 문제를 나타낼 때

> 원문: https://blog.jooq.org/when-source-code-comments-indicate-trouble/

게시일: 2013년 12월 4일 | 저자: lukaseder

## 소개

이 글은 소스 코드 주석의 유머러스한 측면을 탐구하며, 개발자들이 코드에 남긴 재치 있는 발언을 통해 표현된 냉소주의를 다룹니다. 이러한 주석들은 수년 후 미래의 유지보수 담당자들이 발견하게 됩니다.

## 개인적인 경험

저자는 레거시 코드에서 공격적인 문서화를 발견한 개인적인 일화를 공유합니다:

> "이 메서드를 절대로 건드리지 마세요, 그렇지 않으면 모든 것이 폭발할 것입니다!!!!!"

아이러니하게도, 버그는 실제로 80자 들여쓰기가 된 500줄짜리 메서드 안에 있었습니다. 저자는 유머러스하게 원래 경고를 작성한 사람을 "겁쟁이!"라고 부르는 주석을 추가했다고 합니다.

## 주요 내용

이 글은 549개의 주목할 만한 코드 주석이 담긴 Stack Overflow 토론을 참조합니다. 한 확장된 예시는 더글러스 애덤스의 "전체 관점 소용돌이(Total Perspective Vortex)" 개념에 비유된 복잡한 매크로 솔루션을 설명합니다. 이것은 코드 복잡성에 대한 좌절감을 표현하는 정교한 이야기입니다.

## 핵심 포인트

- 개발자 유머는 나중에 발견되도록 의도된 코드 주석에서 번성합니다
- 재치 있는 문서화는 유지보수 작업 중에 코믹한 안도감을 제공할 수 있습니다
- 잘 작성된 주석은 프로그래머의 개성과 문제 있는 코드에 대한 좌절감을 보여줍니다

## 결론

이러한 재치 있는 발언들은 종종 기본적인 코드 품질 문제를 나타냅니다. 개발자들은 문제 있는 코드를 다루는 대처 메커니즘으로 냉소적인 주석을 사용합니다.

이 글은 프로그래머 주석의 창의적인(비록 냉소적이지만) 문화를 기념하면서, 그러한 주석이 종종 근본적인 코드 품질 문제를 나타낸다는 것을 인정합니다. 추가적인 창의적 프로그래머 주석 예시를 보려면 전체 Stack Overflow 스레드를 탐색해 보시기 바랍니다.
