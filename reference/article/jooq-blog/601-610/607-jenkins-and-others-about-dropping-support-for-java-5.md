# Jenkins(및 기타 프로젝트들)의 Java 5 지원 중단에 대하여

> 원문: https://blog.jooq.org/jenkins-and-others-about-dropping-support-for-java-5/

게시일: 2013년 7월 26일, 작성자: lukaseder

오픈 소스 개발자로서, 저는 사용자들을 위해 가능한 한 많은 합리적인 것들을 지원하려고 노력해왔습니다. 그러나 여기에는 Java 5 지원이 포함된 적이 없습니다. Java 5 자체가 인기 있는 Java 벤더들로부터 더 이상 거의 지원받지 못하기 때문입니다. 따라서 jOOQ는 컴파일 및 실행을 위해 Java 6 이상을 필요로 합니다.

Jenkins CI 서버의 창시자인 Kohsuke Kawaguchi가 흥미로운 이니셔티브를 시작했습니다. 그는 서한을 통해 다른 오픈 소스 프로젝트와 개발자들이 Java 5 지원을 중단하는 데 동참하도록 설득하고 있습니다. 이 변화는 대부분의 오픈 소스 프로젝트에게는 다소 사소하고 미미한 것이지만, Jenkins와 같은 지속적 통합 서버에게는 중대한 변화입니다. 그의 허락을 받아, Jenkins CI가 더 이상 Java 5를 지원하지 않아야 하는 이유에 대한 그의 서한을 인용합니다. 만약 여러분이 오픈 소스 개발자이고 Java 5 지원을 중단하고 싶거나 이미 중단했다면, 이 이니셔티브에 참여하세요:

## 무엇인가?

우리는 기준점을 세우고 있습니다: 2013년 9월 30일 이후의 릴리스부터 최소 런타임 환경으로 Java 6을 요구하기 시작할 것입니다.

우리는 사용자들에게 공정한 사전 통지를 제공하기 위해 이 메시지를 전달하고 있습니다. 이를 더 효과적으로 만들기 위해, 우리는 OSS 프로젝트들의 연합을 구축하고 있습니다. 이를 홍보하기 위한 간단한 웹사이트를 만들고, 사람들이 소식을 퍼뜨리도록 장려할 것입니다. 우리의 집단적인 프로젝트 이름과 로고가 메시지를 전파하는 데 도움이 될 것입니다.

우리는 OSS 프로젝트의 개발자들입니다. 사용자들이 우리 소프트웨어를 사용할 수 있도록 돕기 위해, 지금까지 최소 런타임으로 Java 6을 요구하는 것을 자제해왔습니다. 하지만 우리는 충분히 오래 그렇게 했다고 생각합니다. 이제 앞으로 나아갈 때입니다.

## 왜?

- 대부분의 Java VM 벤더들이 더 이상 Java 5를 지원하지 않습니다. 사람들은 그것을 사용해서는 안 됩니다.
- 실행 가능한 오픈 소스 Java 5 구현체가 없습니다.
- 더 새로운 Java를 요구하는 라이브러리의 수가 증가하고 있어 이를 사용할 수 없게 되고, 이는 더 많은 개발 노력, 더 적은 기능과 수정으로 이어집니다.
- 통합 테스트 비용을 증가시키고 있습니다. 점점 더 적은 수의 개발자들만이 실제로 Java 5를 가지고 있는데, 우리는 Java 5를 위해 더 많은 테스트를 실행하고 있습니다.
- 더 새로운 Java 런타임은 더 많은 기능을 가지고 있습니다. 더 많은 컬렉션 API, NIO 개선, 콘솔 접근, XML 지원, 컴파일러 API, 어노테이션 프로세서, 그리고 스크립팅 언어 인터페이스가 있습니다.
- 1.50 클래스 파일 형식은 분리 검증기(split verifier)와 함께 제공되어, 더 빠른 클래스 로딩을 가능하게 합니다.
- 우리의 집단적인 영향력을 이 일에 투입하면 더 많은 사용자들에게 도달하는 데 도움이 될 것입니다. 개별적으로 이 싸움을 선택하는 것은 더 어렵습니다.
- 이것이 성공한다면, 향후 버전에서 더 새로운 Java 런타임으로 이동하는 것이 더 쉬워질 것입니다.

## 사실

- Java 5는 거의 10년 전인 2004년에 출시되었습니다. 공식 지원은 2009년에 종료되었습니다.
- IBM조차도 2013년 9월 30일에 서버 측 Java 5 지원을 종료합니다.
