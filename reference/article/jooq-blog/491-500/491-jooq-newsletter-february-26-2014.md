## 오늘의 트윗

이번 뉴스레터에서는 두 가지 긍정적인 사용자 후기를 소개합니다:

Ben Hood는 jOOQ의 추상화에 대한 감사를 표현했습니다: "커스텀 해킹이 필요하다고 생각할 때쯤이면, 이전에 몰랐던 편리한 추상화가 나타나네요 :-)" (2014년 2월 21일)

Antoine Comte는 프랑스어 트윗에서 MyBatis에서 jOOQ로 직접 전환할 것을 추천했습니다. (2014년 2월 12일)

## jOOQ 3.4 전망

이 뉴스레터에서는 다가오는 릴리스에서 예상되는 기능들을 개략적으로 설명합니다:

- Professional Edition을 위한 IBM Informix 지원
- 일반적인 문장 유형에 대한 타입 안전 DDL 지원
- Java 8 호환성을 갖춘 트랜잭션 관리
- 여러 데이터베이스에 걸친 TABLE 타입과 TABLE 값 반환 함수
- SQL 2 jOOQ 개선 및 UI 향상

## 커뮤니티 활동

Vlad Mihalcea는 저장 프로시저 통합을 위한 jOOQ의 장점을 탐구하는 기사를 발표했으며, JDBC와 JPA 제공 기능의 공백을 다루었습니다.

RebelLabs는 "당신의 지루한 오래된 데이터가 실제로 최신 기술보다 더 섹시하다"라는 제목의 게스트 포스트를 실었으며, 데이터 중심 개발의 관련성을 강조했습니다.

---

## SQL 영역 주제

성능 테스트: "SQL 개발자의 60%가 실패한다" - Use The Index, Luke는 개발자들 사이에서 SQL 인덱싱 지식에 광범위한 격차가 있음을 보여주는 퀴즈를 발표했습니다.

JDBC Boolean 호환성: 이 블로그 포스트에서는 SQL:1999에서 도입된 Boolean 타입이 여전히 데이터베이스 간에 일관되지 않게 지원되어 JDBC 개발을 복잡하게 만드는 방식을 검토했습니다.
