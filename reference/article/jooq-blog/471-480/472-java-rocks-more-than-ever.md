# Java는 그 어느 때보다 멋지다

> 원문: https://blog.jooq.org/java-rocks-more-than-ever/

Java와 C는 오랫동안 TIOBE 인덱스의 1위와 2위를 놓고 경쟁해왔습니다. 최근 JDK 8의 정식 출시와 함께, 우리는 다시 한번 확인할 수 있습니다:

Java는 정말 멋집니다! 그리고 거의 모든 종류의 애플리케이션을 구축하기에 최고의 플랫폼입니다.

Java가 이렇게 멋진 이유는 JVM 때문입니다. 뛰어난 하위 호환성 때문입니다. 접근하기 쉬운 구문과 API 때문입니다. 그리고 시중에서 사용할 수 있는 엄청난 양의 도구들 때문입니다. 제가 계속 나열할 수도 있지만, ZeroTurnaround의 RebelLabs가 Geert Bevin의 도움을 받아 이미 훌륭한 시리즈를 작성해 두었으니, 다시 정리할 필요는 없겠습니다.

## Java가 멋진 10가지 이유

### 1. Java 컴파일러

C++과 달리, Java 개발자들은 컴파일과 관련된 복잡한 문제들을 걱정할 필요가 없습니다. 링킹이나 최적화에 대해 신경 쓸 필요가 없죠. JIT 컴파일러가 런타임에 추가적인 컴파일을 원활하게 처리해줍니다.

### 2. Core API

JDK는 "매우 견고하고, 안정적이며, 잘 이해된 라이브러리 집합"을 제공합니다. 물론 일부 개발자들은 Google Guava나 Apache Commons 같은 도구로 이를 보완하기도 합니다.

### 3. 오픈소스

저자는 오픈소스 정신과 뜻을 같이하며, "자유로서의 무료(free-as-in-freedom)"이든 "공짜 맥주로서의 무료(free-as-in-beer)"이든 그 구분과 상관없이 커뮤니티 협력을 강조합니다.

### 4. Java 메모리 모델

Java의 스레딩과 메모리 모델은 도입 이후로 놀라울 정도로 안정적으로 유지되어 왔습니다. JDK 1.5(2004년)에서 단 한 번의 수정만 있었고, 이것이 액터(Actors)나 Fork/JOIN 같은 현대적인 패러다임을 가능하게 했습니다.

### 5. 고성능 JVM

JVM은 여러 언어들이 다양한 하드웨어 환경에서 인상적인 속도로 실행될 수 있게 해줍니다.

### 6. 바이트코드

바이트코드는 벤더 독립적인 기계 코드 추상화로서, 다양한 기술들에 걸쳐 예측 가능한 방식으로 생성, 조작, 변환될 수 있습니다.

### 7. 지능형 IDE

현대의 개발에는 정교한 도구가 필요합니다. 저자는 IDE 지원의 부족이 SQLJ의 실패에 기여했다고 언급합니다. 복잡한 엔터프라이즈 애플리케이션을 다루는 데 있어 IDE의 역량은 언어 채택에 매우 중요합니다.

### 8. 프로파일링 도구

저자는 Oracle의 무료 Java Mission Control(JDK 7u40 이후)을 강조합니다. 이를 통해 추측이 아닌 정확한 병목 지점 식별이 가능합니다.

### 9. 하위 호환성

Java는 중요한 기능들을 도입해왔습니다 - Streams API, 람다 표현식, 디폴트 메서드, 제네릭, enum - 그러면서도 "아무것도 깨뜨리지 않았습니다". 최소한의 키워드 추가를 제외하고는요.

### 10. 혁신과 함께하는 성숙함

마지막 포인트는 앞선 논의들을 종합합니다: Java는 검증된 안정성과 지속적인 발전을 결합합니다. Java 8의 혁신적인 기능들이 이를 잘 보여줍니다.

## 결론

Java는 성숙한 플랫폼으로서 그 어느 때보다 훌륭하며, Java 커뮤니티의 미래는 밝습니다.
