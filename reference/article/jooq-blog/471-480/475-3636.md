## SQL 영역 - 인메모리 컴퓨팅

이 섹션에서는 데이터베이스 시스템 간의 인메모리 컴퓨팅 접근 방식을 비교합니다:

SQL Server 접근 방식: 타입 안전한 인메모리 테이블 작업을 위해 T-SQL 임시 테이블을 사용합니다.

Oracle 접근 방식: 두 가지 옵션이 있습니다:
1. SQL TABLE OF OBJECT 타입
2. GLOBAL TEMPORARY TABLE

성능 비교: GLOBAL TEMPORARY TABLE은 TABLE OF OBJECT 타입보다 성능이 뛰어납니다. 이는 작업이 SQL 엔진 내에서만 수행되기 때문입니다. 반면 TABLE OF OBJECT 타입은 PL/SQL 엔진의 개입이 필요하며, 이는 상당한 양의 데이터를 처리할 때 문제가 될 수 있습니다.

참조: 이 주제에 대한 Stack Overflow 질문: https://stackoverflow.com/q/22558797/521799
