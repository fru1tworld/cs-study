# SQL 인젝션에 대한 유쾌한 분노

> 원문: https://blog.jooq.org/hilarious-rant-about-sql-injection/

최근 SQL 인젝션에 관한 제 글이 JCG(Java Code Geeks)에서 꽤 격렬한 반응을 불러일으켰습니다. 이걸 여러분께 공유하지 않을 수가 없네요! 일부 발췌입니다:

> "[...] ORM을 사용하면 SQL 인젝션 문제가 마법처럼 사라질 거라는 생각은 X나 해롭고, 근시안적이며, 그렇게 생각하는 사람은 민감한 부위를 세게 걷어차여 마땅하다. [...]"

그리고 그 발길질에서 살아남으셨다면...

> "[...] SQL 문이 없기 때문에, "'a'; TRUNCATE your_mom" 같은 것들이 정확히 그대로 저장/조회된다 [...]"

그리고 여러분의 엄마가 truncate에서 살아남으셨다면:

> "[...] ORM도 여전히 당신을 구해주지 못한다. 빌어먹을 입력값을 검증해라, 이 X아. [...]"

그러니 제발, 여러분의 코드를 정화(sanitise)하세요. 안 그러면 화난 남자가 여러분을 잡으러 올 겁니다!! :)
