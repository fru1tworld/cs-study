# Oracle 기능에 대한 배움은 끝이 없다

> 원문: https://blog.jooq.org/you-never-stop-learning-about-oracle-features/

Oracle은 "신비롭고 비밀스러운 것들을 데이터베이스에 대해 알려주는 신탁"이라는 이름에 걸맞게, SQL:2008 표준의 상당 부분을 구현하면서도 다른 데이터베이스에서는 찾아볼 수 없는 고유한 기능들을 제공합니다. 이러한 기능들 중 일부는 미래의 SQL 표준이 될 가능성이 높습니다.

## PARTITION BY 절

대부분의 개발자들은 PARTITION BY 절을 윈도우 함수(분석 함수)와 함께 사용하는 것으로 알고 있습니다. 또한 Oracle의 MODEL 절에서도 PARTITION BY를 볼 수 있습니다. 하지만 가장 숨겨진 사용 사례가 있는데, 바로 OUTER JOIN 절에서의 사용입니다.

## 파티션 외부 조인(Partitioned Outer Join)

파티션 외부 조인은 OUTER JOIN과 PARTITION BY 절을 결합한 특수한 구문입니다. 이 기능을 사용하면 조인의 어느 한쪽을 파티션으로 지정하여, OUTER JOIN이 해당 파티션에 대해 어떤 레코드도 생성하지 않는 경우에도 모든 파티션에 대해 최소한 하나의 (비어 있을 수 있는) 레코드가 존재하도록 할 수 있습니다.

간단히 말해서: 이 절은 결과 집합을 빈 레코드로 채우는 데 도움이 됩니다.

Oracle에서 빈 행을 채워야 할 필요가 있을 때마다 파티션 외부 조인을 사용하면 됩니다. 이 기능은 jOOQ 2.5.0에서 지원될 예정입니다.

## 결론

Oracle과 같은 성숙한 데이터베이스 시스템에서 고급 기능을 지속적으로 발견하고 학습하는 것은 데이터베이스를 다루는 개발자에게 매우 중요합니다. 특히 이러한 고급 SQL 기능들은 언젠가 SQL 표준의 일부가 될 수 있기 때문에 미리 알아두면 좋습니다.
