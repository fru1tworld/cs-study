# SQL 도구, 순위 매기기

> 원문: https://blog.jooq.org/sql-tooling-the-ranking/

2012년 4월 13일 lukaseder 작성

데이터베이스를 빠르게 설정하고 실행해야 할 때, 도구는 매우 중요해집니다. jOOQ를 개발하고 새로운 데이터베이스에 대한 통합을 추가할 때, 저는 새 데이터베이스, 스키마, 사용자, 역할, 권한 등 필요한 모든 것을 간단한 대화 상자에서 "다음 다음 다음"을 클릭하여 생성할 수 있는 단순한 방법을 제공하는 데이터베이스를 정말 좋아합니다. 결국, 우리는 21세기에 살고 있고, 저는 더 이상 펀치 카드로 소프트웨어를 구성하고 싶지 않습니다.

## 데이터베이스 도구 범주

jOOQ 개발을 하면서 저는 꽤 많은 데이터베이스와 그 도구들을 접해봤습니다. 저는 이것들을 세 가지 범주로 나누고 싶습니다. 이 구분은 jOOQ 개발의 관점에서 주관적이라는 점을 참고해 주세요. 이 데이터베이스들 대부분에 대해 저는 (Oracle과 MySQL을 제외하고) 실제 운영 경험이 없습니다. 운영 환경에 들어가면 상황이 크게 달라질 수 있습니다. 자, 여기 범주들이 있습니다:

### "원하는 모든 것을 갖춘" 데이터베이스들

이 친구들은 무료로 제공되는 표준 배포판에 이미 훌륭한 도구가 통합되어 함께 제공됩니다. 아무런 구성 없이 도구를 시작하고 바로 사용하기 쉽습니다. 이 도구는 실제로 직관적인 리치 클라이언트이며, 수천 페이지의 매뉴얼을 읽거나 이곳저곳 구글링할 필요가 없고, 애드온을 얻기 위해 추가 라이선스 비용을 지불할 필요도 없습니다. 이 범주에는 (알파벳 순서로) 다음이 포함됩니다:

* CUBRID와 Eclipse-RCP 기반 CUBRID Manager. 신규 사용자에게 매우 좋은 도구입니다.
* DB2와 Eclipse-RCP 기반 IBM Data Studio. IBM이 Eclipse를 만들었습니다. Data Studio를 만들지 않았다면 부끄러웠을 것입니다.
* Postgres와 pgAdminIII. 매우 매우 보기 좋고 빠릅니다.
* SQL Server와 SQL Server Management Studio. 이것은 아마도 모든 것 중에서 가장 완벽할 것입니다. 수많은 속성과 구성 팝업에서 길을 잃을 수 있습니다.
* Sybase SQL Anywhere와 Sybase ASE, 둘 다 Sybase Central이라는 동일한 도구를 공유합니다. 약간 구식으로 보이지만, 모든 관리 작업을 쉽게 수행할 수 있습니다.

### "충분한 도구"를 갖춘 데이터베이스들

이 데이터베이스들은 "충분한" 도구를 가지고 있습니다. 이것은 스크립팅이 가능한 통합 콘솔과 함께 제공된다는 것을 의미합니다. 그중 일부는 일반적으로 인기가 있어서, 이러한 데이터베이스를 관리하기 위한 무료 오픈소스 도구가 존재합니다. 여기에는 MySQL과 Oracle이 포함됩니다. 여기 "괜찮은" 것들이 있습니다:

* H2. 웹 기반 콘솔이 실제로 꽤 보기 좋습니다. DHTML 기반 자동 완성과 스크립팅 기능이 있습니다. 이 정도면 충분합니다.
* Ingres. 이 공룡은 Windows 95 이후로 UI 컴포넌트를 업그레이드하지 않은 것 같지만, 필요한 만큼 잘 작동합니다.
* MySQL과 phpMyAdmin. 이것은 MySQL 관리를 위한 매우 좋은, 독립적인, 오픈소스 PHP 애플리케이션입니다. Apache, MySQL, PHP, Perl 배포판인 XAMPP를 사용하여 MySQL과 함께 쉽게 설치할 수 있습니다. 네, 저는 "다음 다음 다음" 패턴을 사용하여 완전한 것들을 설치하는 것을 좋아합니다!
* Oracle. 스크립팅을 위한 sql*plus가 있고, 사용자 인터페이스를 갖춘 많은 상용 및 오픈소스 제품이 있습니다. 제가 가장 좋아하는 것은 Toad와 Toad Extensions입니다 - 정말 좋은 무료 Eclipse 플러그인입니다. 추가 라이선스 비용을 지불하면 Oracle Enterprise Manager와 기타 매우 매우 멋진 도구에 접근할 수 있다는 점을 언급할 가치가 있습니다. 돈이 있다면, 여기서 불평할 수 없습니다.

### "그 외" 데이터베이스들...

여기서는 DDL이 담긴 *.sql 파일을 로드하는 것으로 돌아가게 됩니다. 벤더로부터의 도움은 없습니다.

* Derby. 저는 어떤 도구도 알지 못합니다. 제가 틀렸다면 알려주세요.
* HSQLDB. 통합 콘솔에서 SQL을 실행할 수 있지만, 구문 강조, 검사, 자동 완성 등을 제공하지 않습니다. 아마도 SQuirreL SQL이나 다른 범용 SQL 도구를 사용하는 것이 더 나을 것입니다.
* SQLite. 행운을 빕니다! 이 데이터베이스는 정말 미니멀합니다!
