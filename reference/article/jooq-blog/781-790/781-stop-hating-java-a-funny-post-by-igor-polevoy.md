# "Java를 미워하지 마라" - Igor Polevoy의 재미있는 포스트

> 원문: https://blog.jooq.org/stop-hating-java-a-funny-post-by-igor-polevoy/

2011년 7월 25일 lukaseder 작성

방금 Igor Polevoy(jOOQ와 비슷하지만 약간 다른 초점을 가진 제품인 ActiveJDBC의 창시자)가 쓴 Java/Ruby에 관한 재미있는 글을 발견했습니다. 일부 발췌:

> Ruby 개발자들은 [...] 모두 여러 가지 유행에 빠져든 것 같습니다: Ruby/Rails/Apple MBP/iPhone/TextMate. 그들은 "친환경적"이고, "민주적"으로 보이며, 헐렁한 옷을 입고, "느긋합니다".

그의 발언에 씩 웃지 않을 수 없었는데, 최근 Ruby 개발자로부터 받았던 비꼬는 댓글이 떠올랐기 때문입니다. 그 발언은 jOOQ가 Ruby가 아닌 Java로 작성되었기 때문에 레거시라는 것이었습니다... :-) 어쨌든, 저는 언어나 개발자를 판단하는 데 관여하지 않겠습니다. Igor의 의견이 흥미롭다고 생각하는 이유는, Java가 너무 많은 것을 표준화하고 있다는 그의 말에 동감하기 때문입니다:

> 표준 - Java 역사상 가장 큰 실패작은 Java Enterprise였습니다. 이것은 설명이 필요 없기를 바랍니다. 표준은 Java의 재앙입니다.

Igor는 강한 의견을 가지고 있으며, XML, RMI, Java 5의 동시성 패키지 등 다른 훌륭한 표준들을 너무 많이 무시하고 있을 수도 있습니다. 하지만 더 논란이 되는 표준에는 EJB, JPA, 그리고 특히 Criteria Query API가 포함되어 있으며, jOOQ는 이에 대한 실용적인 대안이 되고자 합니다. Igor의 블로그를 좀 더 팔로우할 가치가 있을 것입니다. 전체 글은 여기서 읽을 수 있습니다:

http://igorpolevoy.blogspot.com/2011/07/stop-hating-java-2.html
