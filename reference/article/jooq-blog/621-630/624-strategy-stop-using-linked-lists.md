# 전략: 연결 리스트 사용을 중단하라

> 원문: https://blog.jooq.org/strategy-stop-using-linked-lists/

저는 LinkedList의 목적을 이해합니다. Java 6 이후로 Deque 타입이 있다는 것도 좋아합니다. 하지만 List의 구현체로서 LinkedList를 유용하게 사용하는 경우는 거의 없습니다.

이것이 잘 설명된 High Scalability의 흥미로운 기사가 있습니다:

http://highscalability.com/blog/2013/5/22/strategy-stop-using-linked-lists.html

원문 기사는 여기에서 확인할 수 있습니다:

http://www.futurechips.org/thoughts-for-researchers/quick-post-linked-lists.html#more-818

연결 리스트의 "학문적" 장점은 모든 사람에게 분명합니다(삽입, 삭제 연산의 big-O 이점). 하지만 하드웨어, 메모리, 힙과 관련된 "현실 세계"의 단점들은 연결 리스트를 사용할 때 빠르게 불리하게 작용합니다. 이는 C에서와 마찬가지로 Java에서도 동일하게 적용됩니다.

대부분의 경우, LinkedList를 사용해야 한다고 생각할 때, 단순히 배열을 사용하는 것이 더 나을 수 있습니다. 이는 조기 최적화의 좋은 예입니다.

## 댓글

### Peter Verhas (2013년 5월 27일)

"절대 연결 리스트를 사용하지 마라"고 말하는 것은 문제가 있습니다. 몇 가지 중요한 점이 있습니다:

- 조기 최적화는 모든 악의 근원입니다. 실제로 성능 문제가 발생했을 때만 성능에 대해 걱정해야 합니다.
- 문맥이 중요합니다. 연결 리스트는 특정 환경과 목적에 적합합니다.
- 현대 CPU 캐시 구조는 단순한 big-O 분석과 다릅니다.
- 파일 시스템은 연결 리스트를 효과적으로 사용하며, 특정 목적에 맞게 최적화되어 있습니다.
- Java는 포인터가 아닌 객체 참조를 통해 메모리를 처리하므로 C와 다릅니다.
- 실제 측정과 코드 예제 없이는 이러한 기사들은 추측에 불과합니다.

### lukaseder (2013년 5월 27일)

비판을 인정합니다. "절대"라는 단어 선택이 적절하지 않았습니다. 아마도 주목을 끌기 위한 기법이었을 것입니다. 개발자들이 LinkedList를 사용해야 한다고 생각할 때 배열을 사용하는 것이 더 나을 수 있다는 점에 동의하며, 이는 조기 최적화를 나타냅니다.
