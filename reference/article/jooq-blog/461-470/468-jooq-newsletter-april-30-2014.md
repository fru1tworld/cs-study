# jOOQ 뉴스레터: 2014년 4월 30일

> 원문: https://blog.jooq.org/jooq-newsletter-april-30-2014/

Java, SQL 그리고 jOOQ. 멋진 Java 및 SQL 코드 작성에서 얻은 모범 사례와 교훈.

## 오늘의 트윗

### Santiago M. Mola

jOOQ가 데이터베이스별 기능을 지원하는 것에 대해 감사를 표하며, 이 프레임워크가 "여러분의 DB가 지원하는 거의 모든 타입이나 기능을 올바르게 지원한다"고 말했습니다.

### Vlad Mihalcea

jOOQ 문서를 살펴보면서 자신의 SQL 지식이 확장되었다고 공유했으며, 이전에 익숙하지 않았던 윈도우 함수와 기타 고급 개념들을 발견했습니다.

### Tom Bujok

jOOQ 스티커를 받고 기뻐하며, 관심 있는 사용자들은 홍보 자료를 받기 위해 팀에 연락해 달라고 초대했습니다.

---

## 도서 홍보 – SQL Performance Explained

"SQL Performance Explained" 도서를 쿠폰 코드 "jOOQ"를 사용하여 10% 할인된 가격으로 홍보합니다. 인덱스를 이해하면 개발자가 약 90%의 성능 문제를 해결할 수 있으며, 약 6시간의 독서 투자 가치가 있다고 강조합니다.

웹사이트: http://sql-performance-explained.com

---

## 커뮤니티 존 – 활발한 기여자들

이 섹션에서는 커뮤니티가 기여한 여러 기사를 소개합니다:

### Petri Kainulainen

jOOQ와 Spring Framework를 사용한 CRUD 작업 구현 및 페이지네이션/정렬에 대한 튜토리얼을 게시했습니다.

### Markus Winand

OFFSET과 KEYSET 페이지네이션 전략의 차이점을 강조했습니다.

### Michael Hughes (codinginthetrenches.com)

SQL DSL의 이점과 저장 프로시저에 대한 JDBC/JPA 대안 대비 jOOQ의 장점을 탐구했습니다.

### Krisztian Horvath (SequenceIQ)

Apache Phoenix를 통한 HBase와 jOOQ의 호환성을 시연했습니다.

### Harmeet Singh

프로덕션 배포 전 컴파일 타임 오류를 방지하는 jOOQ의 코드 생성기를 칭찬했습니다.

## SQL 존 – 데이터베이스 뷰

이 섹션에서는 데이터베이스 뷰의 주요 사용 사례를 설명합니다:

- 기본 구조를 변경하지 않고 필드 수정을 가능하게 하는 스키마 추상화
- 복잡한 조인 모델링 단순화
- 데이터베이스별 기능 마스킹
- 권한 관리 및 테이블 수준 접근 제어
- 스키마 진화 중 하위 호환성 유지
- 추가 보안 레이어 구현

출처 참조: 뷰 응용에 관한 Stack Overflow 토론

## SQL 존 – 누적 합계 계산

뉴스레터는 계산 작업에 데이터베이스 윈도우 함수를 활용할 것을 권장합니다. 은행 거래에서 잔액 값을 계산하는 예를 통해, 기본 집계 함수를 넘어서는 정교한 계산에 대한 SQL의 역량을 보여줍니다.

관련 기사: "NoSQL? No, SQL! – 누적 합계 계산 방법"
