# jOOQ 3.2 상용 라이선스 및 지원 제공

> 원문: https://blog.jooq.org/jooq-3-2-offering-commercial-licensing-and-support/

2013년 10월 9일 lukaseder 작성

4년 전, Java 데이터베이스 미들웨어 시장은 JPA를 구현하는 다양한 ORM들이 지배하고 있었습니다. 이 패러다임은 대안들에 의해 거의 도전받지 않았습니다. Java 언어 생태계에서 SQL을 일급 시민으로 만드는 API에 대한 공백이 있었고, jOOQ가 이 공백을 채우기 위해 등장했습니다. jOOQ를 통해 SQL 작성에 많이 관여하는 개발자들은 마침내 Java 언어에서 SQL을 거의 네이티브하게 표현할 수 있는 도구를 갖게 되었으며, Java 6+ 컴파일러의 기능을 활용하여 SQL 문의 구문과 타입 정확성을 검증할 수 있게 되었습니다. 지난 4년 동안 jOOQ는 점점 더 인기를 얻고 성숙해졌습니다. jOOQ는 대규모의 복잡한 데이터베이스에서 운영하는 많은 까다로운 고객들에 의해 사용되고 있습니다. 그 고객들로부터 받은 피드백은 만장일치로 긍정적이었지만, 한 가지 부족한 것이 있었습니다. 그 한 가지 부족한 것은 전문적인 지원, 보증, 그리고 따라서 상용 라이선스였습니다. 새로운 jOOQ 3.2에서는 훌륭한 새 기능들을 도입하는 것 외에도, 우리가 운영하는 방식에서 꽤 많은 것들을 변경하고 있습니다.

jOOQ 3.1 이후, 우리는 이제 jOOQ 3.2를 발표했습니다.

Data Geekery GmbH에서 우리는 오픈 소스를 믿습니다. 하지만 우리는 또한 상용 소프트웨어가 가능하게 하는 창조적인 힘도 믿습니다. 이것이 우리가 다음과 같은 듀얼 라이선스 전략을 구현하기로 선택한 이유입니다:

- 인기 있는 오픈 소스 데이터베이스와 함께 jOOQ를 사용하는 사용자들은 계속해서 Maven Central, GitHub, 또는 jOOQ 웹사이트에서 우리의 고품질, 통합 테스트된 Apache Software License 2.0 라이선스의 무료 오픈 소스 배포판을 얻을 수 있습니다. 동시에, 그들은 jOOQ User Group에서 영어로 제공되는 우리의 무료 커뮤니티 지원을 계속 이용할 수 있습니다.

- 상용 데이터베이스와 함께 jOOQ를 사용하는 사용자들은 우리의 경쟁력 있는 상용 jOOQ 구독 라이선스를 이용할 수 있으며, 이는 상용 데이터베이스 지원뿐만 아니라 영어, 독일어, 프랑스어로 제공되는 전문적인 지원, 릴리스 및 버그 수정에 대한 조기 액세스, 보증, 맞춤 엔지니어링 등을 포함합니다.

우리의 라이선스 모델에는 워크스테이션 라이선스와 상용 데이터베이스 지원 옵션이 포함됩니다.

물론, 오픈 소스 데이터베이스와 함께 jOOQ를 사용하는 고객들도 우리의 전문적인 지원과 서비스를 이용하고 싶다면 상용 라이선스 구독을 구매할 수 있습니다. 다음 FAQ에서 여러분이 가질 수 있는 몇 가지 질문에 답변하겠습니다:

## 왜 워크스테이션 라이선스를 제공하나요?

우리는 jOOQ의 주요 부가 가치가 고객의 개발 라이프사이클에 추가된다고 믿습니다. jOOQ는 개발자들이 JDBC나 다른 문자열 기반 접근 방식을 직접 사용했을 때 필요한 시간의 10%만에 SQL 코드를 작성할 수 있도록 도와줍니다. 동시에, jOOQ와 그 강력한 소스 코드 생성기는 개발 라이프사이클 초기에 코딩 실수를 방지하여 일반적인 SQL 실수의 90%를 예방합니다. 따라서 jOOQ는 개발 도구이며, 런타임에 가장 많은 가치를 추가하는 스토리지 도구인 데이터베이스와는 대조됩니다. 이것이 대부분의 데이터베이스가 서버 라이선스를 제공하는 것과 달리 jOOQ가 워크스테이션 라이선스를 제공하는 이유입니다.

## 왜 완전한 오픈 소스 라이브러리에 대한 지원만 제공하지 않나요?

우리는 모든 것을 오픈 소스로 제공하면서 비싼 지원 구독으로만 돈을 버는 벤더들을 알고 있습니다. 이것은 데이터베이스와 같은 서버 소프트웨어를 제공하는 벤더들에게는 실행 가능한 모델이며, 이 경우 24/7 지원이 매우 중요합니다. 또한 많은 추가적이고 비용이 많이 드는 컨설팅 노력이 필요한 매우 복잡한 소프트웨어를 제공하는 벤더들에게도 실행 가능한 모델입니다. jOOQ는 앞서 언급한 것들 중 어느 것도 아닙니다. 이것은 사용하기 매우 쉬운 미들웨어이며, SQL 자체 위에 거의 복잡성을 추가하지 않기 때문에 컨설팅 비즈니스를 생성하지 않을 것입니다. 우리는 다양한 기존 사용자 및 고객들과 함께 우리의 결정을 검토했으며, 종속적인 데이터베이스 라이선스 모델을 기반으로 하는 듀얼 라이선스 모델로 가장 많은 가치를 추가할 수 있다고 확고히 믿습니다. 결국, jOOQ 라이선스에 약간의 돈을 지출하면 DB2, Oracle, SQL Server, 또는 Sybase와 같은 강력한 데이터베이스를 사용하는 개발 라이프사이클에서 고객들에게 엄청난 양의 돈을 절약해 줄 것입니다.

## 왜 Apache License를 고수하나요?

Apache License는 지난 4년 동안 우리에게 잘 맞았습니다. 이것은 매우 자유로우면서도 동시에 우리의 상표를 보호합니다. 이것은 우리가 성장하고 많은 고품질 커뮤니티 피드백과 기여를 얻을 수 있게 해준 라이선스입니다. 우리는 *GPL 경로를 따르는 다른 듀얼 라이선스 회사들을 따르고 싶지 않습니다. 왜냐하면 우리는 듀얼 라이선스 *GPL 소프트웨어가 장기적으로 오픈 소스 소프트웨어의 정신에 진실하지 않다고 믿기 때문입니다. Apache License를 유지함으로써 우리는 오픈 소스에 대한 깊은 헌신을 표현하고자 합니다.

## 기여자 저작권은 어떻게 되나요?

우리는 저작권을 진지하게 다룹니다. 이것이 우리가 주요 기여자들에게 모든 저작권을 우리에게 양도하는 동의서에 서명하도록 한 이유입니다. 저작권을 정리해야 하지만, 우리는 여전히 코드나 문서를 공유해 준 기여자들에게 저작권을 공개적으로 귀속시키고자 합니다. 물론, 우리 커뮤니티의 모든 미래 기여도 환영합니다! 기여자로서 우리의 듀얼 라이선스에 의해 당신의 저작권이 도전받거나 침해받는다고 느끼신다면, 직접 저희에게 연락해 주세요.

## jOOQ Console은 어떻게 되었나요?

jOOQ Console 3.1은 Maven Central에서 jOOQ 3.2 사용자들에게 계속 제공되지만, jOOQ Console 개발은 현재 형태로 중단되었습니다. 이것은 Data Geekery가 가까운 미래에 개발할 새로운 상용 프로파일러 소프트웨어를 위한 자리를 만들 것입니다. 기대해 주세요.

## 왜 이것을 하나요?

우리는 jOOQ에 대한 헌신을 보여주고 싶기 때문입니다. jOOQ는 단순한 라이브러리 이상입니다. 이것은 더 나은 Java와 SQL 통합에 대한 비전입니다. 이것은 Java 생태계의 다양한 분야에서의 혁신입니다. 우리는 다음을 원합니다:

- Java와 SQL을 모두 사랑하지만 과거에 적절한 도구를 제공받지 못했던 사람들을 지원합니다.
- jOOQ와 같은 소프트웨어의 언어적 측면을 형식화함으로써 Java에서 내부 도메인 특화 언어의 통합을 더욱 발전시킵니다. jOOQ만을 위한 것이 아닙니다.
- 네이티브 Scala API - 그리고 가능하면 코어까지 - 제공함으로써 고품질 SQL 통합을 Scala 생태계로 확장합니다.
- 더 많은 고품질 개발 도구를 제공하기 위해 jOOQ Console을 더욱 개선합니다.
- 블로그와 소스 코드를 통해 고객들에게 우리의 SQL 지식을 제공합니다.

위의 계획들을 구현하기 위해, 우리는 jOOQ로부터 비즈니스를 만들어야 합니다. 그리고 우리는 이 비즈니스가 우리와 고객 모두에게 매우 흥미로울 것이라고 믿습니다. 함께, jOOQ 생태계로서, 우리는 더 나은 Java/SQL 세계를 만들 것입니다! jOOQ 3.2로 시작합시다!
