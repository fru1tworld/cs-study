# SQL로의 조기 복귀

> 원문: https://blog.jooq.org/the-premature-return-to-sql/

게시일: 2013년 10월 2일
저자: Lukas Eder
블로그: Java, SQL and jOOQ

---

온라인 커뮤니티에서 NoSQL 주제는 (ORM 주제와 마찬가지로) 감정을 자극하는 것이 보장되어 있습니다. 많은 감정이 이념적 또는 마케팅적 이유로 양쪽 진영의 전도사들에 의해 자극됩니다. 다음은 열정적인 NoSQL 및 폴리글랏 퍼시스턴스 전도사인 Alex Popescu의 흥미로운 게시물입니다. 그는 최근 SQL로 복귀하는 트렌드가 시기상조라고 주장합니다:

- http://nosql.mypopescu.com/post/62403847766/the-premature-return-to-sql

이 게시물은 SQL Performance Explained의 저자인 Markus Winand의 똑같이 흥미로운 반응을 불러일으켰습니다:

- http://use-the-index-luke.com/blog/2013-10/mysql-is-to-sql-like-mongodb-to-nosql

사람들이 얼마나 자주 "트렌드"라는 관점에서 생각하는지는 정말 흥미롭습니다. 새로운 패러다임을 도입하여 우리가 이전에 가지고 있던 모든 것을 구식으로 만든다는 그런 트렌드 말입니다. 저는 이것들이 트렌드가 아니라 실험이라고 믿습니다. 저는 이전에 NoSQL 벤더들이 DBA를 끝장낸다고 약속할 때 경계해야 한다고 블로그에 쓴 적이 있습니다. 아주 소수의 "새로운" 솔루션이나 패러다임만이 그들의 선조를 완전히 대체하거나 치환한 적이 있습니다. 또는 아이작 뉴턴의 말을 빌리자면:

> "내가 더 멀리 볼 수 있었다면 그것은 거인의 어깨 위에 서 있었기 때문이다."

우리는 "SQL로 복귀"하는 것이 아니며, 그러한 복귀가 "시기상조"인 것도 아닙니다. 네, 늙은 코끼리에게 새로운 재주를 가르치는 혁신적인 사상가들이 있고, 그것은 좋은 일입니다. 그러한 혁신적인 사상가들이 케이크의 한 조각을 얻고 그들의 발명으로 돈을 버는 것도 좋은 일입니다.

대형 데이터베이스 벤더들이 그다지 혁신적이지 않다는 것도 사실입니다. 하지만 그들은 그럴 필요가 없습니다. 그들의 자산은 신뢰성, 예측 가능성, 안정성입니다. Oracle SQL은 15년 후에도 여전히 모든 오래된 레거시를 지원할 것이며, 이것이 은행과 보험 회사에 안전한 선택이 되는 이유입니다. NoSQL이나 NewSQL 기능이 혁신적이면서 신뢰할 수 있다고 입증되면, Oracle 등은 분명히 그것을 채택하여 SQL에 통합할 것입니다. 현명한 NoSQL 벤더들은 따라서 이미 그들의 출구를 준비하고 있습니다.

이것은 물론 데이터베이스 세계 밖에서도 일어납니다:

- Scala는 혁신적이며 Java에 기여합니다 (Java 5의 제네릭, Java 8의 람다).
- 오픈 소스 개발자들 (예: JAX-RS 개발자들)은 혁신적이며 JEE에 기여합니다.
- PostgreSQL은 혁신적이며 다른 SQL 방언과 궁극적으로 SQL 표준에 기여합니다.
- Instagram은 혁신적이었으며 Facebook에 기여했습니다.
- jOOQ는 혁신적이며 JDBC와 JPA에 기여합니다 (궁극적으로, 희망컨대).

SQL은 안전한 선택이며 여기에 계속 남아 있을 것입니다.
