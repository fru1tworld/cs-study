게시일: 2013년 10월 11일, 작성자: lukaseder

## jOOQ 3.2 출시

jOOQ 3.2 릴리스에서는 두 가지 중요한 서비스 제공자 인터페이스(SPI)가 도입되었습니다:

1. CRUD 작업 동작 주입 - 사전 및 사후 작업 훅을 허용하여 전역 ID 생성기 기능을 가능하게 합니다
2. SQL 렌더링 생명주기 주입 - 임의의 SQL 변환을 지원하며, 특히 공유 스키마 멀티테넌시 패턴 구현에 유용합니다

이번 릴리스에서 코드 생성기가 상당히 개선되었습니다. 전체 세부 사항은 공식 릴리스 노트에서 확인할 수 있습니다.

## jOOQ 라이선싱 및 지원 전략

Data Geekery GmbH는 오픈소스 원칙과 상용 소프트웨어의 이점을 균형 있게 조화시키는 듀얼 라이선싱 접근 방식을 발표했습니다. 이 전략은 무료 및 상용 제품 모두를 통한 창의적 역량 강화에 대한 회사의 철학을 반영합니다.

이 라이선싱 모델에 대한 자세한 설명은 함께 제공되는 블로그 게시물에서 확인할 수 있습니다.

## SQL2jOOQ 도구 개발

GUDU Soft는 Data Geekery와 협력하여 SQL2jOOQ라는 오픈소스 변환 유틸리티를 개발했습니다. GUDU Soft의 SQL 파서를 기반으로 구축된 이 도구는 텍스트 기반 SQL 문자열과 AST를 변환하면서 jOOQ를 출력 방언으로 지원합니다. 이 도구는 레거시 SQL 구현을 jOOQ 프레임워크로 변환하는 개발자들의 마이그레이션 문제를 해결합니다.

## NoSQL 시장 동향

MongoDB의 최근 1억 5천만 달러 벤처 펀딩 라운드는 업계에서 상당한 논의를 불러일으켰습니다. 이 발표는 194개의 추적 시스템 중 MongoDB가 선두적인 비관계형 데이터베이스 관리 시스템으로서의 위상을 강조했습니다.

이 기사는 Oracle과 SQL Server와 같은 기존 관계형 데이터베이스에 대한 경쟁이 비SQL 벤더에게서 오더라도 여전히 가치가 있다고 언급합니다.
