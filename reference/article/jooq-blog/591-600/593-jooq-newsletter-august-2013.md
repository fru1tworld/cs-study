# jOOQ 뉴스레터 2013년 8월

> 원문: https://blog.jooq.org/jooq-newsletter-august-2013/

## jOOQ 사용자 설문조사의 압도적인 피드백

jOOQ는 3년 동안 Java 오픈소스 SQL 빌딩 도구로서의 선두 위치를 유지해왔으며, 소셜 플랫폼 전반에서 커뮤니티로부터 긍정적인 반응을 받고 있습니다.

최근 사용자 설문조사를 통해 몇 가지 핵심적인 통찰을 얻을 수 있었습니다:

- 사용자층이 Java와 SQL 모두에서 높은 숙련도를 보여주고 있습니다
- 주요 데이터베이스 집중 영역에는 H2, MySQL, Oracle, PostgreSQL, SQL Server가 포함됩니다
- 커뮤니티는 미션 크리티컬 애플리케이션을 위한 jOOQ의 성숙도를 높이 평가합니다
- 사용자들은 도구의 미션, 지원 품질, 전반적인 신뢰성에 대해 일반적으로 만족감을 표현하고 있습니다

설문조사에 참여해주신 분들께 감사드리며, 여러분의 피드백이 jOOQ의 미래 개발 방향을 안내할 것입니다.

## SQL 인기는 여전히 강력합니다

NoSQL 대안의 등장에도 불구하고, SQL은 상당한 시장 관련성을 유지하고 있습니다. 일부 벤더들이 다른 데이터 저장 솔루션을 제안하고 있지만, 많은 경우 표준화 이전의 SQL 접근 방식과 유사한 독점 쿼리 언어로 회귀하고 있습니다.

SQL의 지속적인 매력을 보여주는 지표에는 ANSI/ISO 표준화와 증가하는 빅데이터 채택이 포함됩니다. 일반적인 프로그래밍 실수를 강조하는 DZone의 두 가지 인기 기사는 SQL 모범 사례에 대한 커뮤니티의 지속적인 관심을 보여주었습니다.

## jOOQ 3.2 기능 전망

이 뉴스레터에서는 두 가지 중요한 계획된 개선 사항을 발표합니다:

### 쿼리 렌더링 및 변수 바인딩을 위한 새로운 SPI

이 서비스 프로바이더 인터페이스(SPI)는 사용자가 쿼리 렌더링 이벤트 중에 임의의 SQL 변환을 구현할 수 있게 해줍니다. 실용적인 애플리케이션에는 투명한 보안 레이어 구현과 공유 스키마 환경을 위한 멀티테넌시 솔루션이 포함됩니다.

### CRUD 작업을 위한 새로운 SPI

store(), insert(), update(), delete(), refresh() 작업을 지원하는 강력한 SPI를 통해 ActiveRecord 스타일 CRUD 기능이 향상됩니다. 일반적인 사용 사례에는 insert 전 데이터 초기화와 ID 생성 메커니즘이 포함됩니다.
