# 좋은 MCVE(최소 완전 검증 가능 예제)를 만드는 방법

> 원문: https://blog.jooq.org/how-to-create-a-good-mcve-minimal-complete-verifiable-example/

버그를 보고하는 것은 시간이 걸리며, 믿어주시길 바라지만, 모든 벤더는 여러분의 버그 보고에 감사합니다! 하지만 버그를 보고할 때, 이상적으로는 다음과 같은 예제가 필요합니다:

- 최소한(Minimal): 여러분의 실제 애플리케이션 코드는 거대합니다. 전체 코드를 벤더에게 덤프할 수는 없습니다. 따라서 문제를 작은 범위로 격리해야 합니다. 불필요한 코드 없이, 문제 자체만 보여주는 작은 범위로 말이죠. 그것은 어렵습니다. 왜냐하면 여러분은 몇 달 또는 몇 년에 걸쳐 프로젝트를 개발했고, 그 안에서 문제가 발생했기 때문입니다.
- 완전한(Complete): 문제를 산문으로 설명하는 것만으로는 충분하지 않습니다. 코드를 보여줘야 합니다. 관련 코드를요. 이것은 예제가 여전히 최소한이어야 한다는 요구사항과 상충됩니다.
- 검증 가능한(Verifiable): 이상적인 예제는 벤더가 문제를 재현하는 데 사용할 수 있습니다. 왜냐하면 일단 그것이 가능해지면, 벤더는 디버깅을 시작할 수 있고, 무엇이 잘못되었는지 쉽게 파악할 수 있습니다. 여러분에게 추측을 보내고 왜 이 추측이 잘못되었는지 설명해달라고 요청하는 대신에 말이죠.

이러한 예제가 포함된 버그 보고서는 벤더에게 반갑고, 모든 고객에게 도움이 됩니다. 문제가 더 빨리 발견되고 수정되기 때문입니다. 그래서 jOOQ 이슈 트래커에 보고된 문제의 경우, GitHub에 예제 프로젝트를 설정했습니다:

https://github.com/jOOQ/jOOQ-mcve

이 예제 프로젝트에는 다음이 포함되어 있습니다:

- SQL 스크립트를 설정하기 위한 Flyway
- jOOQ 코드 생성기
- 예제 JUnit 테스트

이 프로젝트를 포크하여 문제를 재현하도록 수정하기만 하면 됩니다!

버그 보고에 시간을 내주셔서 다시 한번 감사드립니다.
