# Aerospike 3.99.0.3

날짜: 2018-03-06

Jepsen은 Aerospike와 함께 그들의 차세대 합의 시스템을 검증했습니다. 검증 결과, 프로세스 일시 중지 및 충돌로 인한 데이터 손실의 두 가지 알려진 시나리오가 확인되었습니다.

## 발견된 문제

이전에 알려지지 않은 버그가 발견되었는데, 이는 내부 RPC 프록시 메커니즘의 문제로 클라이언트가 성공적으로 적용된 업데이트를 확실한 실패로 볼 수 있게 했습니다.

## Aerospike의 대응 조치

- 발견된 버그를 수정
- 클라이언트 작업 승인 전에 노드가 디스크에 쓰도록 요구하는 옵션 추가
- 합의 시스템이 허용할 수 있는 최대 클록 스큐 확장 계획
