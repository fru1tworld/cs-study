# FaunaDB 2.5.4 Jepsen 분석

## 개요
Kyle Kingsbury의 Jepsen 분석에서 FaunaDB 2.5.4는 기본 키-값 작업에서 스냅샷 격리부터 엄격한 직렬화까지 제공하는 것으로 나타났으나, "인덱스, 시간 쿼리, 이벤트 스트림 관련 쿼리에서 요청된 보장을 충족하지 못했습니다."

## 발견된 주요 문제 (19가지)

### 심각한 안정성 문제
- 불완전한 비시간적 인덱스: 읽기가 현재 상태를 반영하지 않음 (2.6.0-rc9에서 수정)
- 페이지네이션 중 누락 레코드: 미적용 트랜잭션으로 인해 페이지 끝의 레코드 생략
- 시간 쿼리 불일치: 임의의 과거 상태를 반환할 수 있음
- 롱 포크 현상: 다중 읽기가 모순되는 순서로 쓰기를 관찰
- 급성 지속적 읽기 왜곡: 노드 재시작 후 매우 불일치한 데이터

### 운영 문제
- 음의 정수 페이지네이션 실패 (2.5.5에서 수정)
- 클러스터 토폴로지 변경 중 노드 교착 상태
- 클록 스큐로 인한 가용성 문제 (미해결)
- 스키마 변경 비트랜잭션성

## 조치 및 권장사항

Fauna는 거의 모든 문제를 2.6.0-rc10까지 해결했습니다. Jepsen은 "모든 사용자가 실질적으로 빠르게 2.6.0으로 업그레이드할 것을 권장합니다."

중요한 미해결 사항으로는 스키마 작업(2.7 계획)과 클록 스큐 관련 가용성이 있습니다.

## 긍정적 평가

분석은 Calvin 프로토콜 기반의 기본 설계가 견고하며, "발견된 버그는 구현 문제"로 보이고 Fauna의 신속한 해결 의지를 인정했습니다.
