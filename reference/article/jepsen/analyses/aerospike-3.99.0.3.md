# Aerospike 3.99.0.3 Jepsen 분석

## 개요
Kyle Kingsbury의 2018년 분석은 Aerospike의 강일관성(SC) 모드를 검증했습니다. 보고서는 세 가지 주요 결함을 확인했습니다.

## 발견된 결함

### 결함 1: 동시 노드 크래시로 인한 업데이트 손실
"동시에 replication-factor보다 많은 노드가 크래시되면 커밋된 쓰기 작업이 손실될 수 있습니다." 이는 비동기 디스크 쓰기 때문입니다.

### 결함 2: 프로세스 일시 중지 또는 시계 편향 관련 문제
"27초를 초과하는 프로세스 일시 중지 또는 시계 편향이 발생하면 커밋된 업데이트가 손실될 수 있습니다." 이는 제한된 레짐 비트(6비트)로 인한 설계 제약입니다.

### 결함 3: 잘못된 클라이언트 오류 응답
"프록시 재시도로 인해 클라이언트는 성공한 쓰기가 실패했다고 받을 수 있습니다." 3.99.1.5에서 수정됨.

## 권장 조치

- SC 모드 사용자: commit-to-device true 옵션 활성화 필수
- 환경 요구사항: 반-실시간 네트워크 필요 (27초 이상 일시 중지 방지)
- AP 모드: 단일 쓰기 데이터에만 적합

## 결론

3.99.2.1 이상에서 SC 모드는 합리적인 조건에서 선형성을 제공하지만, 운영 안정성에 주의가 필요합니다.
