# Radix DLT 1.0-beta.35.1 Jepsen 분석

## 개요
이 종합적인 보안 분석은 Radix DLT 버전 1.0-beta.35.1부터 1.0.2까지를 평가하여 분산 원장 시스템에서 13가지 별개의 안전성 및 성능 문제를 식별했습니다.

## 중요한 발견사항

### 발견된 안전성 위반

분석 결과 정상 운영 중에도 지속적인 데이터 일관성 문제가 발견되었습니다. "서버별 단조성을 위반하는 부실 읽기, 중단된 읽기와 중간 읽기, 커밋된 트랜잭션의 부분적 또는 전체 손실"이 여러 버전에 걸쳐 시스템을 괴롭혔습니다.

주요 이상 현상:

- 비단조 읽기: 노드가 확인된 트랜잭션이 누락된 트랜잭션 히스토리를 반환한 후 나중에 표시—트랜잭션이 원장 상태에서 사라지는 것처럼 보임.

- 누락된 트랜잭션: 한 노드에서 보이는 커밋된 트랜잭션이 다른 노드의 로그에서 완전히 없을 수 있으며, 복구 메커니즘이 없음.

- 모순되는 로그: 동일한 노드에서 순차적 읽기가 충돌하는 트랜잭션 히스토리를 반환하며, 트랜잭션이 나타났다 사라짐.

- 충돌 시 트랜잭션 손실: 모든 검증자가 동시에 충돌할 때, 부적절한 BerkeleyDB 구성으로 인해 이전에 커밋된 트랜잭션이 영구적으로 사라짐.

- 실패 상태 이상: `FAILED`로 표시된 트랜잭션이 그럼에도 불구하고 트랜잭션 로그에 나타나 클라이언트 애플리케이션을 혼란스럽게 함.

## 성능 문제

처리량은 테스트 클러스터에서 초당 약 16개 트랜잭션으로 최고치를 기록했으며, 이는 140만 TPS의 마케팅 주장보다 훨씬 낮습니다. 단일 노드 장애로 인해 거의 모든 트랜잭션에서 지연 시간이 급격히 증가하여 일반적인 장애 시나리오 중 네트워크가 불안정해졌습니다.

## 공개 네트워크 영향

이러한 문제는 2021년 7월 28일부터 2022년 1월 27일까지 Radix Olympia 퍼블릭 네트워크에 영향을 미쳤습니다. 사용자들은 "단일 노드에서 비단조 상태, 중간 잔액, 중단된 읽기, 트랜잭션에서 누락되거나 가짜인 작업"을 관찰했을 수 있습니다.

## 해결

RDX Works는 버전 1.1.0(2022년 1월)에서 문제가 있는 아카이브 하위 시스템을 새로운 Gateway API 아키텍처로 교체했습니다. 조직은 식별된 모든 안전성 문제가 이제 해결되었다고 주장하지만, Jepsen은 이러한 주장을 독립적으로 검증하지 않았습니다.
