# YugaByte DB 1.3.1 Jepsen 분석

## 개요
Jepsen은 YugaByte DB 1.3.1의 분산 SQL 트랜잭션 지원을 평가했습니다. 보고서는 "두 가지 안전 문제"를 발견했으며, 하나는 1.3.1.2-b1에서 수정되었습니다.

## 주요 발견사항

### 안전 문제
- DEFAULT 열이 NULL로 초기화되는 현상
- 마스터 노드 충돌 시 G2-item(반의존성 순환) 이상 현상

### 운영 문제
- 테이블 생성 중 중복 키 제약 위반 오류
- 네트워크 분할 복구에 100초 이상 소요 (약 5%)
- Postgres 워커 프로세스 메모리 누수 (초당 약 5MB)
- 클럭 스큐 상황에서 선형성 위반

## 권장사항

보고서는 "DDL 운영 자동화를 피할 것"을 강력히 권고하며, 테이블 생성 후 즉시 쓰기 작업을 피하도록 조언합니다. YSQL은 여전히 베타 상태이며, 버전 2.0까지 프로덕션 환경 배포를 권장하지 않습니다.

## 결론

기본 트랜잭션 기능은 "대체로 안전"하지만, DDL과 리소스 관리에 상당한 문제가 있습니다.
