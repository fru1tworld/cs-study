# 람다 대수 입문 (A Tutorial Introduction to the Lambda Calculus)

- 원문: [A Tutorial Introduction to the Lambda Calculus](https://www.inf.fu-berlin.de/inst/ag-ki/rojas_home/documents/tutorials/lambda.pdf)
- 작성자: Raul Rojas (Freie Universitat Berlin)
- 버전: 2.0, 2015

> 이 문서는 Raul Rojas의 "A Tutorial Introduction to the Lambda Calculus" 전문을 한국어로 정리한 것입니다.

---

λ-대수는 Alonzo Church가 1930년대에 효과적 계산 가능성(effective computability)의 개념을 형식화하기 위해 만든 체계다. 변수 치환이라는 변환 규칙 하나(β-변환)와 함수 정의 방식 하나만으로 구성된, 세상에서 가장 작은 범용 프로그래밍 언어라 할 수 있다. 튜링 기계와 동치이지만, λ-대수는 기계 구현이 아닌 기호 변환 규칙에 초점을 둔다.

## 1. 정의

### 식(expression)

λ-대수의 핵심 개념은 "식(expression)"이다. 식은 재귀적으로 정의된다.

```
<expression> := <name> | <function> | <application>
  <function> := λ <name> . <expression>
<application> := <expression> <expression>
```

언어에 쓰이는 키워드는 `λ`와 `.` 뿐이다.

### 항등 함수

가장 단순한 함수 예시는 항등 함수다.

```
λx.x
```

`λx` 부분은 함수의 인자가 `x`임을 선언하고, `.` 뒤의 `x`가 함수 본체(body)다. 인자를 그대로 돌려준다.

### 적용(application)과 β-변환

함수에 인자를 적용하면, 본체에서 매개변수를 인자로 치환한다.

```
(λx.x)y → [y/x]x → y
```

`[y/x]`는 "x가 나타나는 모든 곳을 y로 치환하라"는 뜻이다.

### 좌결합(left association)

적용은 왼쪽부터 결합한다.

```
E₁E₂E₃...Eₙ  =  (...((E₁E₂)E₃)...Eₙ)
```

### α-변환

매개변수 이름은 자리표(placeholder)일 뿐이므로, 이름을 바꿔도 같은 함수다.

```
(λz.z) ≡ (λy.y) ≡ (λt.t) ≡ (λu.u)
```

이런 이름 바꾸기를 α-변환이라 한다.

## 1.1 자유 변수와 묶인 변수

λ-대수에서 이름은 정의에 대해 지역적이다(프로그래밍 언어의 지역 변수와 같다).

- `λx.x`에서 `x`는 `λx`에 의해 **묶인(bound)** 변수다.
- `λ`에 의해 묶이지 않은 변수는 **자유(free)** 변수다.

```
(λx.x)(λy.yx)
```

여기서 첫 식의 `x`는 첫 `λ`에 묶여 있고, 둘째 식의 `y`는 둘째 `λ`에 묶여 있으며, 둘째 식의 `x`는 자유 변수다. 두 식의 `x`는 서로 독립적이다.

**자유 변수의 형식적 정의:**

- `<name>`은 `<name>` 안에서 자유다. (예: `a`는 `a`에서 자유)
- `<name>`은 `λ<name₁>.<exp>`에서, `<name> ≠ <name₁>`이고 `<name>`이 `<exp>`에서 자유일 때 자유다. (예: `y`는 `λx.y`에서 자유)
- `<name>`은 `E₁E₂`에서, `E₁`이나 `E₂` 중 하나에서 자유이면 자유다.

같은 식별자가 하나의 식 안에서 자유이면서 동시에 묶일 수 있다.

```
(λx.xy)(λy.y)
```

첫 부분식에서 `y`는 자유이지만, 오른쪽 부분식에서는 묶여 있다.

## 1.2 치환

치환 시 자유 변수와 묶인 변수를 섞지 않도록 주의해야 한다.

```
(λx.(λy.xy))y
```

여기서 함수 본체에 묶인 `y`가 있고, 인자로 들어오는 `y`는 자유다. 그냥 치환하면 잘못된 결과가 나온다.

```
잘못된 결과: (λy.yy)
```

묶인 `y`를 다른 이름(예: `t`)으로 바꾼 뒤 치환해야 한다.

```
(λx.(λt.xt))y → (λt.yt)
```

**규칙:** `λx.<exp>`를 `E`에 적용할 때, `<exp>` 안의 자유로운 `x`만 `E`로 치환한다. 치환 결과 `E`의 자유 변수가 묶이게 되는 경우, 먼저 묶인 변수의 이름을 바꿔야 한다.

**정규 순서 변환(normal order reduction):** 항상 가장 왼쪽의 적용부터 변환하며, 더 이상 변환할 수 없을 때까지 반복한다.

## 2. 산술

### 처치 수(Church numerals)

λ-대수에서 숫자도 함수로 표현한다. 자연수 `n`은 "함수 `s`를 `n`번 적용하는 함수"로 정의한다.

```
0 ≡ λsz.z
1 ≡ λsz.s(z)
2 ≡ λsz.s(s(z))
3 ≡ λsz.s(s(s(z)))
```

숫자 `n`에 인자 `f`와 `a`를 적용하면, `f`를 `a`에 `n`번 적용한 결과가 나온다.

```
3fa → (λsz.s(s(s(z))))fa → f(f(fa))
```

0에 `f`와 `a`를 적용하면 `f`를 0번 적용, 즉 `a`가 그대로 나온다.

```
0fa ≡ (λsz.z)fa → a
```

### 후속 함수(successor)

```
S ≡ λnab.a(nab)
```

동작 원리: `n`이 `a`를 `n`번 적용하는 함수이므로, `nab`는 `a`를 `b`에 `n`번 적용한 결과이고, 여기에 `a`를 한 번 더 적용하면 `n+1`번 적용한 꼴이 된다.

```
S0 ≡ (λnab.a(nab))0 → λab.a(0ab) → λab.a(b) ≡ 1
S1 ≡ (λnab.a(nab))1 → λab.a(1ab) → λab.a(ab) ≡ 2
```

## 2.1 덧셈

`m + n`은 후속 함수를 `m`번 적용하면 된다.

```
덧셈: mSn
```

예를 들어 2 + 3:

```
2S3 ≡ (λsz.s(s(z)))S3 → S(S3) → S4 → 5
```

## 2.2 곱셈

```
곱셈 함수: (λxya.x(ya))
```

`x`가 `ya`를 자기 함수 자리에 놓으므로, `y`를 `a`에 적용하는 함수를 `x`번 반복 적용하는 구조가 된다.

3 × 3의 계산:

```
(λxya.x(ya))33 → (λa.3(3a))
```

`3a`는 `a`를 3번 적용하는 함수이고, 여기에 `3`을 다시 적용하면 그 함수를 3번 반복하므로 총 9번 적용. 즉 `3(3a)b = a(a(a(a(a(a(a(a(ab))))))))`.

## 3. 조건문

### 참과 거짓

논리값도 함수로 표현한다.

```
T ≡ λxy.x    (두 인자 중 첫째를 반환)
F ≡ λxy.y    (두 인자 중 둘째를 반환)
```

### 3.1 논리 연산

**AND:**

```
∧ ≡ λxy.xyF
```

`x`가 참이면 `y`의 값에 따라 결과가 결정되고, `x`가 거짓이면 `F`를 선택하므로 전체가 거짓이다.

**OR:**

```
∨ ≡ λxy.xTy
```

`x`가 참이면 `T`를 선택하므로 참이고, `x`가 거짓이면 `y`의 값에 따라 결정된다.

**NOT:**

```
¬ ≡ λx.xFT
```

`x`가 참이면 `F`를 선택하고, `x`가 거짓이면 `T`를 선택한다.

```
¬T ≡ (λx.xFT)T → TFT ≡ (λcd.c)FT → F
```

### 3.2 영 판정(zero test)

숫자가 0인지 판정하는 함수:

```
Z ≡ λx.xF¬F
```

핵심 아이디어:

- `0fa = a`이므로, `0F¬F = ¬F = T`. 0이면 참을 반환한다.
- `Fa ≡ (λxy.y)a → λy.y ≡ I` (항등 함수). `F`를 아무 값에 적용하면 항등 함수가 된다.
- 0이 아닌 수 `N`에 대해, `NF¬F`는 `F`를 `N`번 적용한 뒤 `¬`에 적용하는데, `F`를 한 번이라도 적용하면 항등 함수 `I`가 되고, 이후로도 계속 `I`이다. 따라서 `I`에 `F`를 적용하면 `IF → F`. 0이 아닌 수에는 거짓을 반환한다.

### 3.3 전임 함수(predecessor)

전임 함수의 전략: 쌍(pair) `(n, n-1)`을 만든 뒤 둘째 원소를 꺼낸다.

**쌍 표현:**

```
쌍 (a, b) ≡ (λz.zab)
```

`T`를 적용하면 첫째, `F`를 적용하면 둘째가 나온다.

```
(λz.zab)T → Tab → a
(λz.zab)F → Fab → b
```

**쌍 생성 함수 Φ:** 쌍 `(n, n-1)`에서 `(n+1, n)`을 만든다.

```
Φ ≡ (λpz.z(S(pT))(pT))
```

`pT`로 쌍의 첫째 원소를 꺼내고, 새 쌍의 첫째에는 후속 값을, 둘째에는 원래 첫째 값을 넣는다.

**전임 함수:**

```
P ≡ (λn.(nΦ(λz.z00))F)
```

초기 쌍 `(0, 0)`에 `Φ`를 `n`번 적용하면 `(n, n-1)`이 되고, 여기서 `F`로 둘째를 꺼내면 `n-1`이다. 이 정의에서 `P0 = 0`이 되는데, 이 성질은 다른 함수를 정의할 때 유용하다.

### 3.4 비교 연산

**크거나 같다(≥):**

```
G ≡ (λxy.Z(xPy))
```

`x`에서 전임 함수를 `y`번 적용해서 0이 되면 `x ≥ y`다.

**같다(=):**

```
E ≡ (λxy. ∧ (Z(xPy))(Z(yPx)))
```

`x ≥ y`이고 `y ≥ x`이면 `x = y`다.

## 4. 재귀

λ-대수에서 함수에는 이름이 없으므로, 자기 자신을 직접 호출할 수 없다. 재귀를 가능하게 하는 것이 **Y 결합자**(Y combinator)다.

```
Y ≡ (λy.(λx.y(xx))(λx.y(xx)))
```

### Y 결합자의 동작

`Y`를 함수 `R`에 적용하면:

```
YR ≡ (λx.R(xx))(λx.R(xx))
```

이를 더 변환하면:

```
→ R((λx.R(xx))(λx.R(xx)))
= R(YR)
```

즉 `YR → R(YR)`. 함수 `R`이 `YR`을 첫 인자로 받아 호출되는데, 이 `YR`이 다시 `R(YR)`로 펼쳐지므로 재귀가 성립한다.

### 예: 0부터 n까지의 합

`Σᵢ₌₀ⁿ i = n + Σᵢ₌₀ⁿ⁻¹ i`라는 재귀 관계를 이용한다.

```
R ≡ (λrn.Zn0(nS(r(Pn))))
```

- `n`이 0이면 결과는 0
- 아니면 `n`에 후속 함수를 적용한 뒤, `r`을 `Pn`(전임자)에 재귀 호출한 결과에 더한다

0부터 3까지의 합:

```
YR3 → R(YR)3 → Z30(3S(YR(P3)))
```

3은 0이 아니므로:

```
→ 3S(YR2)
```

이것은 "0부터 2까지의 합에 후속 함수를 3번 적용한 것"이다. 재귀적으로 풀면:

```
3S(2S(1S0)) = 3S(2S(1)) = 3S(3) = 6
```
