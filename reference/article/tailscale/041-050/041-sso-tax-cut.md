2023년, 우리는 [가격 정책 v3](https://tailscale.com/blog/pricing-v3)를 도입했습니다. 당시 우리는 SSO(Single Sign-On) ID 제공자를 두 가지 카테고리로 분리하기로 결정했습니다: 모든 Tailscale 가격 플랜에서 사용할 수 있는 표준 ID 제공자(무료 ID 서비스)와, Personal, Premium, Enterprise 가격 티어에서만 사용할 수 있는 고급 ID 제공자(유료 ID 서비스)입니다.

몇 주 전, 우리는 조용히 웹사이트에서 이 문구를 제거하여, Tailscale을 모든 플랜에서 OIDC 호환 SSO ID 제공자와 함께 사용할 수 있도록 했습니다.

초기 결정을 내린 지 거의 정확히 1년이 지난 지금, SSO는 다시 모든 사람에게 무료로 제공됩니다. 이 변경의 영향을 받는 사용자 수는 상대적으로 적습니다; 유료 티어를 사용하는 대부분의 사람들은 고급 ID 제공자 외에 다른 프리미엄 기능을 사용하고 있으며, 대부분의 Starter 사용자들은 표준 ID 제공자를 편안하게 사용하고 있었습니다. 하지만 우리는 우리에게 중요한 이유들 때문에 이 결정을 내렸으며, 그 이유를 이해하려면 ID(신원)에 대한 약간의 배경 지식이 필요합니다.

## SSO와 ID

먼저 맥락을 설명하겠습니다. SSO로 알려진 Single Sign-On은 사용자 접근 관리를 중앙화합니다. 사용자가 관리해야 하는 로그인 자격 증명의 수를 줄여주며, 이는 결과적으로 침해에 취약한 자격 증명의 수를 줄여줍니다. 또한 조직이 다중 인증(MFA) 사용 의무화 및 비밀번호 강도와 같은 일관된 보안 정책을 시행할 수 있게 해줍니다. SSO는 온보딩, 오프보딩 및 보안 사고 시 애플리케이션에 대한 접근을 신속하게 부여하고 취소하는 것을 더 쉽게 만들며, 사용자의 애플리케이션 접근 및 사용에 대한 감사 로그를 제공합니다.

이러한 기능들은 SSO를 보안을 진지하게 생각하는 모든 현대 기업의 기본 도구로 만들었습니다. 내부적으로 Tailscale 직원들은 보안 정책의 일부로 [SSO 사용 가능 시 SSO 사용](https://tailscale.com/security-policies/password#single-sign-on)이 의무화되어 있습니다. 게다가 Tailscale 제품은 일반 사용자명+비밀번호 인증을 전혀 지원하지 않습니다. Tailscale 사용자들은 항상 SSO를 통해서만 인증할 수 있었습니다.

왜 우리는 SSO를 요구할까요? Tailscale의 미션은 새로운 인터넷을 구축하는 것이며, 우리는 다음 세대의 인터넷이 기본적으로 보안이 되어야 한다는 강력한 입장을 취하고 있습니다. 우리는 또한 우리가 정말 잘하는 것(네트워킹)에 집중하고, ID 전문가들이 ID 관련 부분을 잘 처리하도록 맡기고 싶습니다. 이것은 간단한 일이 아닙니다! ID와 인증은 핵심 보안 문제이며, 해당 분야의 전문가들은 많은 어려운 구현 작업을 수행해 왔습니다. 사용자의 인증 요구 사항에 대해, Tailscale ID를 기존 SSO에 위임하는 것이 명백한 승리입니다.

## SSO 세금

많은 서비스 제공업체들은 아직 SSO가 기본적인 보안 요구 사항이 되었다는 우리의 철학을 공유하지 않으며, SSO ID 제공자를 통해 로그인하는 "사치"에 대해 더 많은 비용(때로는 상당히 더 많은 비용)을 청구하는 경향이 있습니다. 이 관행은 현재 보안 분야에서 악명 높게 "SSO 세금"이라고 불립니다. 이를 기리기 위해 [SSO 세금 수치의 벽](https://sso.tax)도 있습니다 - 이를 시행하는 회사들의 공개적으로 관리되는 목록입니다.

이 세금의 핵심 문제는 기본적인 보안을 예산 문제로 바꾸어, 사용자들이 좋은 보안 태세와 지갑 사이에서 선택하도록 강요한다는 것입니다. 그리고 회사가 작을수록 이 트레이드오프는 더 고통스럽습니다. 보안 우선 회사로서, 우리는 이것이 전체 인터넷의 이익을 위한 올바른 트레이드오프라고 생각하지 않습니다 - 따라서 이것은 우리가 지지하고 싶은 트렌드가 아닙니다.

## 우리가 (부분적으로) 함정에 빠진 방법

하지만 그러한 원칙을 염두에 두고도, 우리는 (부분적인) SSO 세금을 부과하는 위치에 있게 되었습니다. Tailscale은 사용자 기반과 그들의 요구 사항과 함께 매일 진화합니다. 끊임없이 변화하는 요구 사항에 맞는 가격 정책을 설계하는 것은 과거에도, 현재도 복잡한 도전입니다.

복잡성을 더하는 것: 일부 ID 관련 기능들은 주요 보안 문제를 일으키지 않으면서 진정으로 프리미엄 옵션이 될 수 있습니다. 전형적인 예가 [사용자 및 그룹 프로비저닝](https://tailscale.com/kb/1290/user-group-provisioning) (예: SCIM)입니다. 이것은 SSO 위에 구축되어 있고, 일부 종류의 사용자 관리를 훨씬 쉽게 만들지만, SSO 전반과 동일한 보안 영향을 미치지는 않습니다. 이것은 진정으로 고급 보안 옵션이지, 기본 옵션이 아닙니다.

가격 정책 v3를 설계할 때, 아이디어는 Starter가 고급 보안 기능이 필요하지 않을 수 있는 (높은 수준의 내부 신뢰를 가진) 소규모 팀에 서비스를 제공하고, Premium 및 Enterprise 플랜은 더 많은 보증이 필요한 대규모 팀을 위한 것이었습니다. 우리는 이미 비용을 지불하고 사용하는 더 기능이 풍부한 ID 제공자를 사용하는 사용자들이 네트워크 흐름 로깅, 사용자 및 그룹 프로비저닝 또는 고급 사용자 역할과 같은 더 많은 엔터프라이즈 Tailscale 기능의 혜택을 받을 것이라고 가정했습니다. 우리는 이러한 기능들을 "고급" (즉, 유료) ID 제공자와 함께 가격 플랜에 번들로 묶었습니다.

우리는 두 종류의 ID 제공자를 분리하고 SSO가 일반적으로 무료로 유지된다는 사실에서 위안을 얻으며 바늘구멍을 통과하려고 했습니다. 그것은 사실이었습니다! 하지만 또한, 계속 검토하면서, 이 가격 정책은 점점 더 (특정 ID 제공자에 대한) 부분적인 SSO 세금처럼 느껴졌습니다. 그리고 그것은 실수처럼 느껴졌습니다.

사실, 고객 피드백과 사용 사례에서, 우리는 사용자의 ID 제공자 선택이 (앞서 언급한 사용자 및 그룹 프로비저닝을 제외하고) Tailscale의 다른 기능에 대한 사용 및 요구 사항과 거의 상관관계가 없다는 것을 알게 되었습니다. 표준과 고급 ID 제공자 간의 구분은 결국 사용자들에게 어떤 가치도 제공하지 않았으며, 보안은 사치가 아니라는 우리의 정신과도 일치하지 않았습니다.

처음부터 우리 가격 정책의 목표는 사용자에게 최상의 경험과 가치를 제공하는 것이었습니다. 사용자에게 영향을 미치는 문제를 발견하면, 가능한 한 빨리 개선하거나 해결하는 것을 목표로 합니다. SSO 세금 함정에 빠졌다는 것을 인지한 때부터, 제거를 승인하고 공개 정책을 업데이트할 때까지 단 몇 주밖에 걸리지 않았습니다.

SSO 세금을 폐지하는 것은 여기서 중요한 보안 승리이며, 더 많은 회사들이 이를 따르기를 바랍니다. 마찬가지로 중요한 것은, 무엇이 작동하고 있고 - 무엇이 작동하지 않는지 - 살펴보고, 어떤 정책이 실수일 수 있는지에 대해 사용자들의 의견을 듣는 의지입니다. 그 사용자들과 신속하게 행동해 준 Tailscale 동료들에게 감사드립니다. 이제 우리의 SSO 세금은 사라졌습니다. 잘 가라, 잘 가.
