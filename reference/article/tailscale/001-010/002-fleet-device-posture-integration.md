# Fleet 디바이스 보안 상태 통합 정식 출시

- 원문: [Fleet device posture integration is now generally available](https://tailscale.com/blog/fleet-device-posture-integration)
- 작성일: 2026년 2월 18일
- 작성자: Jillian Murphy

---

조직 내 디바이스는 끊임없이 변화합니다. 노트북이 교체되고, 머신이 재이미징되고, 엔드포인트가 관리 대상에서 벗어나며, 매주 새로운 디바이스가 등장합니다.

이로 인해 인증과 디바이스 관리 사이에 격차가 발생할 수 있습니다. Tailscale은 누군가가 *누구*인지 알고 있습니다. 디바이스 관리 솔루션은 해당 디바이스가 *규정을 준수*하는지 알고 있습니다. 하지만 지금까지 이 두 가지 정보는 서로 다른 세계(음, 대시보드)에 존재했습니다.

이제 오픈 디바이스 관리 플랫폼인 Fleet과의 디바이스 보안 상태 통합이 Tailscale Enterprise 플랜에서 정식 출시되어, 이 두 점을 연결합니다.

이 통합을 통해 Tailscale 접근 정책(ACL)에서 Fleet이 관리하는 디바이스 상태를 직접 참조할 수 있으므로, 접근 결정이 디바이스가 활발히 등록되고 관리되고 있는지를 반영합니다.

## 디바이스 관리를 접근 시행에 도입

디바이스 보안 상태를 네트워크 접근 정책에 통합하면 조직이 연결성을 디바이스 관리 상태와 연결하는 실용적인 방법을 얻게 됩니다. 연결된 모든 노트북을 동일하게 취급하는 대신, 팀은 프로덕션 데이터베이스나 관리 도구와 같은 민감한 시스템에 도달하기 전에 디바이스가 기본 관리 조건을 충족하도록 요구할 수 있습니다.

다음과 같은 상황을 다루고 있다면 특히 유용합니다:

- IT가 물리적으로 프로비저닝할 수 없는 원격 팀
- 프로덕션 접근이 필요하지만 자신의 머신을 직접 관리하는 개발자
- 디바이스 등록을 수동으로 추적하는 것이 확장되지 않는 증가하는 인원
- "관리되는 디바이스만"을 지정하는 규정 준수 요구 사항

접근 시행에 디바이스 관리 상태를 적용함으로써, 조직은 최소한의 추가 오버헤드로 노출을 줄이고 접근 드리프트를 제한합니다.

## 작동 방식

관리자는 조직에서 "관리됨"이 무엇을 의미하는지 정의하고(예: 등록 상태 및 정책 요구 사항), Tailscale은 Fleet이 보고한 보안 상태 속성을 접근 시행의 일부로 적용합니다.

Fleet은 주기적으로 디바이스 관리 상태를 보안 상태 신호로 Tailscale에 동기화합니다. 그런 다음 Tailscale은 접근 정책에서 해당 속성을 사용하여, 시행이 디바이스가 현재 등록되어 있고 정의한 조건을 충족하는지를 반영합니다.

나중에 디바이스가 관리 대상에서 벗어나거나 더 이상 요구 사항을 충족하지 않으면, 수동 정책 업데이트 없이 접근이 자동으로 제한됩니다.

Fleet은 디바이스 관리 상태의 진실 소스로 유지되며, Tailscale은 해당 상태를 적용하여 테일넷 전체에서 접근 규칙을 시행합니다.

## 시작하기

Fleet 통합은 현재 Tailscale Enterprise에서 사용할 수 있습니다.

이미 Fleet과 Tailscale을 모두 사용하고 있다면 오늘 바로 활성화할 수 있습니다. 설정은 약 10분 정도 걸립니다:

1. Fleet에서 API 토큰 생성
2. Tailscale 관리 콘솔에서 Fleet 연결
3. 보안 상태에 반영할 Fleet 정책 정의
4. ACL 정책을 업데이트하여 해당 검사 참조

이 통합 및 기타 디바이스 보안 상태 통합에 대한 전체 문서는 문서에서 확인할 수 있습니다.
