## 프라이빗 엔드포인트로 로그 스트리밍 보안

Tailscale은 이제 프라이빗(테일넷) 엔드포인트로의 로그 스트리밍을 지원합니다. 퍼블릭 인터넷을 통해 로그를 전송하지 않으려는 사용자는 대신 테일넷 내의 노드로 직접 로그를 라우팅할 수 있습니다.

이 기능을 구현하려면, 관리자는 먼저 선호하는 보안 정보 및 이벤트 관리(SIEM) 시스템을 호스팅하는 테일넷에 노드를 설정한 다음, 로그 스트리밍을 구성할 때 노드의 이름이나 IP 주소를 입력해야 합니다. 컨트롤 플레인은 프라이빗 엔드포인트를 자동으로 식별하고 스트리밍 서비스와 노드를 공유하기 위한 동의를 요청합니다. 이후 로그는 지정된 노드로 흐릅니다. 로그 스트리밍을 구성하고 접근 제어 목록(ACL)을 수정할 권한이 있는 관리자만 프라이빗 엔드포인트를 설정할 수 있습니다.

## Cribl 로그 스트리밍

Tailscale은 이제 "전체 텔레메트리 데이터 솔루션 제품군으로 개방형 관찰 가능성을 현실로 만드는" 플랫폼인 Cribl과 통합됩니다. 두 회사 모두 데이터 관리와 관련하여 사용자에게 더 큰 제어, 유연성, 자유를 부여하는 것을 강조합니다.

조직은 로그 소스와 대상에 대한 다양한 요구 사항을 가지고 있습니다. 사용자 피드백에 따르면 고객은 특정 요구에 맞는 대상으로 Tailscale 로그를 라우팅하는 유연성을 원했습니다. Cribl은 이러한 유연성을 가능하게 하여, 아카이빙을 위한 S3 버킷과 같은 스토리지 서비스로의 직접 전송, 데이터 레이크를 위한 ETL 파이프라인 구축, 또는 SIEM 시스템으로의 전송을 지원합니다. Cribl을 통해 Tailscale 로그를 라우팅하는 것에 대한 추가 세부 정보는 Cribl 문서를 방문하세요.

## Tailscale Free에서 감사 로그 스트리밍

Tailscale은 구매 전에 제품 기능을 경험할 수 있게 하는 철학을 확장하여 이제 무료 플랜 사용자에게 구성 감사 로그 스트리밍을 제공합니다. 이러한 로그는 현재 지원되는 모든 로그 스트리밍 파트너로 스트리밍할 수 있습니다.

현재 지원되는 SIEM 시스템은 HTTP Event Collector를 통한 Splunk, 데이터 스트림을 통한 Elasticsearch Logstash, Panther, Cribl입니다. 포괄적인 정보는 로그 스트리밍 문서를 참조하세요. 로그 스트리밍을 시작하려면 관리 콘솔에 접속하세요.
