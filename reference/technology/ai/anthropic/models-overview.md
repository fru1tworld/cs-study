# Anthropic: Models Overview

> 공식 문서: https://platform.claude.com/docs/en/about-claude/models/overview

## 1. 개요

Anthropic은 Claude 모델을 여러 세대(제너레이션)와 성능/비용 등급(티어)의 조합으로 제공한다. 모델을 선택할 때는 무조건 "최고 성능 모델"을 고르기보다, 작업의 복잡도·응답 속도 요구사항·비용 제약을 함께 고려해 등급을 선택하는 것이 기본 원칙이다.

> 모델 이름, 컨텍스트 윈도우 크기, 정확한 가격 등 구체적인 수치는 자주 갱신되므로, 실제 프로젝트에 적용할 때는 반드시 공식 문서에서 최신 값을 확인해야 한다. 이 문서는 개념적인 등급 구조와 선택 기준을 정리한 것이다.

---

## 2. 성능/비용 등급 구조

Anthropic은 보통 세 가지 등급으로 모델 라인업을 구성한다.

| 등급 | 특징 | 적합한 용도 |
|------|------|-------------|
| 최상위 등급(예: Opus) | 가장 높은 추론 능력, 복잡한 다단계 문제 해결 | 어려운 코딩 작업, 복잡한 에이전틱 워크플로, 고난도 분석 |
| 균형 등급(예: Sonnet) | 성능과 속도/비용의 균형 | 일반적인 프로덕션 워크로드, 대부분의 에이전트/코딩 작업 |
| 경량 등급(예: Haiku) | 가장 빠르고 저렴, 상대적으로 단순한 작업에 최적화 | 대량 처리, 실시간 응답이 중요한 단순 분류/추출 작업 |

세대(버전)가 올라갈수록 같은 등급 내에서도 성능이 개선되는 경향이 있어, "이전 세대의 상위 등급"과 "다음 세대의 하위/중간 등급"이 비슷한 성능을 보이는 경우도 흔하다.

## 3. 모델 선택 기준

| 고려 요소 | 질문 |
|-----------|------|
| 작업 복잡도 | 여러 단계 추론, 장기 컨텍스트 유지가 필요한가? |
| 지연 시간(latency) 민감도 | 실시간 대화형 UX인가, 배치 처리인가? |
| 비용 민감도 | 대량 호출 시 토큰당 비용이 서비스 단가에 큰 영향을 주는가? |
| 컨텍스트 길이 요구 | 긴 문서/코드베이스 전체를 한 번에 다뤄야 하는가? |
| 도구 사용/에이전트 자율성 | 다단계 도구 호출을 스스로 판단해 수행해야 하는가? |

---

## 4. 프롬프트 캐싱과 비용 최적화

Anthropic API는 반복적으로 재사용되는 긴 프롬프트 구간(시스템 프롬프트, 대형 컨텍스트 등)을 캐싱해 이후 호출의 지연 시간과 비용을 줄이는 기능을 제공한다. 같은 프리픽스를 가진 요청이 자주 반복되는 에이전트/RAG 시스템에서 특히 효과적이다.

---

## 5. 모델 마이그레이션 시 유의점

| 항목 | 권장 사항 |
|------|-----------|
| 회귀 테스트 | 모델 세대를 올릴 때 프롬프트가 의도한 대로 동작하는지 평가 스위트로 검증 |
| 시스템 프롬프트 재검토 | 새 세대는 지시를 따르는 방식이 미묘하게 달라질 수 있어 프롬프트 튜닝이 필요할 수 있음 |
| 단계적 롤아웃 | 트래픽 일부만 새 모델로 전환해 품질/비용 영향을 관찰 후 전면 적용 |
| 구버전 지원 종료(deprecation) 일정 확인 | 공식 문서의 지원 종료 공지를 주기적으로 확인 |

---

## 6. 요약

- Anthropic 모델은 최상위/균형/경량 등급으로 나뉘며, 작업 복잡도·지연 시간·비용을 기준으로 선택한다.
- 구체적인 모델명, 가격, 컨텍스트 윈도우 크기는 자주 바뀌므로 항상 최신 공식 문서를 확인해야 한다.
- 프롬프트 캐싱은 반복되는 긴 컨텍스트가 있는 워크로드의 비용/지연 시간을 줄이는 핵심 최적화 수단이다.
