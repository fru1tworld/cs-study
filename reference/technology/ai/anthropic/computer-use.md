# Anthropic: Computer Use (Claude)

> 공식 문서: https://platform.claude.com/docs/en/agents-and-tools/tool-use/computer-use-tool

## 1. 개요

Computer Use는 Claude가 스크린샷을 통해 화면을 "보고", 마우스 클릭/키보드 입력 같은 액션을 지시해 데스크톱/브라우저 환경을 조작할 수 있게 하는 도구 사용 모드다. 웹사이트나 애플리케이션이 API를 제공하지 않아도, 사람이 화면을 보고 조작하듯 GUI 자체를 조작해 작업을 수행할 수 있게 하는 것이 핵심 목표다.

```
1. 클라이언트가 현재 화면 스크린샷을 Claude에 전달
2. Claude가 스크린샷을 분석해 다음 액션을 결정 (예: 특정 좌표 클릭)
3. 클라이언트가 실제로 그 액션을 OS/브라우저에 실행
4. 실행 후 새 스크린샷을 다시 Claude에 전달
5. 목표 달성까지 반복
```

---

## 2. 제공되는 액션 유형

| 액션 | 설명 |
|------|------|
| 스크린샷 캡처 | 현재 화면 상태를 이미지로 획득 |
| 마우스 이동/클릭/드래그 | 지정 좌표로 커서 이동, 클릭, 드래그 |
| 키보드 입력 | 텍스트 타이핑, 특수 키(Enter, Tab 등) 전송 |
| 스크롤 | 지정 방향으로 스크롤 |
| 대기 | 페이지 로딩 등을 기다리기 위한 지연 |

이 도구는 Claude가 좌표 기반으로 화면을 조작한다는 점에서, 브라우저 자동화의 [CDP Input 도메인](../../browser/cdp/input.md) 기반 접근과 개념적으로 유사하지만, Computer Use는 특정 브라우저 프로토콜에 종속되지 않고 임의의 GUI 환경(OS 데스크톱 포함)에 적용 가능하도록 설계됐다.

---

## 3. 클라이언트 구현 책임

Computer Use는 모델이 "무엇을 할지"만 결정하고, 실제 스크린샷 캡처와 입력 실행은 호출하는 애플리케이션(클라이언트)이 구현해야 한다. 즉 Anthropic API는 액션 계획만 반환하며, 실제 OS 수준 제어(가상 디스플레이, 원격 데스크톱, 헤드리스 브라우저 등)는 별도 인프라가 필요하다.

---

## 4. 안전/신뢰 고려사항

| 위험 | 대응 권고 |
|------|-----------|
| 의도치 않은 파괴적 액션(파일 삭제, 결제 등) | 격리된 샌드박스/가상 머신 환경에서 실행 |
| 민감 정보 노출 | 스크린샷에 개인정보/자격 증명이 노출되지 않도록 환경 구성 |
| 프롬프트 인젝션 (화면에 표시된 악의적 지시를 모델이 따르는 경우) | 사람의 확인이 필요한 액션을 구분해 두고, 중요한 액션 전 승인 단계 삽입 |
| 무한 루프/반복 실패 | 최대 반복 횟수 제한, 실패 시 사람에게 에스컬레이션 |

---

## 5. 다른 도구 사용 모드와의 관계

Computer Use는 [일반 Tool Use](./tool-use.md)의 한 특수 형태로, "화면"이라는 범용 인터페이스를 도구로 다루는 방식이다. 구조화된 API가 있는 서비스라면 Tool Use로 직접 함수를 호출하는 편이 더 정확하고 빠르며, Computer Use는 API가 없는 레거시 시스템이나 임의의 데스크톱 애플리케이션을 다뤄야 할 때 유효하다.

---

## 6. 요약

- Computer Use는 Claude가 스크린샷을 보고 좌표 기반 액션(클릭, 타이핑, 스크롤)을 지시하는 도구 사용 모드다.
- 실제 액션 실행은 클라이언트 애플리케이션의 책임이며, 격리된 환경에서 실행할 것이 권장된다.
- 구조화된 API가 있는 작업은 [Tool Use](./tool-use.md)가 더 적합하고, Computer Use는 GUI 전용 환경을 위한 보완 수단이다.
