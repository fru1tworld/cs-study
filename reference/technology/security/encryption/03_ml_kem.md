# ML-KEM (Module-Lattice-Based Key-Encapsulation Mechanism)

> 원문: https://en.wikipedia.org/wiki/ML-KEM

## 개요

**ML-KEM**(Module-Lattice-Based Key-Encapsulation Mechanism), 이전 명칭 **Kyber**는 미래의 양자 컴퓨터를 이용한 암호 분석 공격에 견디도록 설계된 **키 캡슐화 메커니즘(KEM)** 이다. 2024년에 표준화되었다.

ML-KEM은 두 통신 당사자 간에 공유 비밀(shared secret)을 수립하여, IND-CCA2 공격자가 전송 내용을 해독할 수 없게 한다. 이 비대칭 암호 체계는 **오류가 있는 학습(Learning With Errors, LWE)** 격자 문제의 변형을 기본 트랩도어 함수로 사용한다.

ML-KEM은 NIST의 최초 양자 내성 암호 표준 경쟁에서 승리하여 **FIPS 203**으로 표준화되었다.

## 속성

### 수학적 기반

이 암호 체계는 **모듈 오류가 있는 학습(Module-LWE, M-LWE)** 문제와 **원분체 환(cyclotomic ring)** 에 기반한다. 최근의 연구 결과로, ring-LWE 문제에서 MLWE 문제로의 엄밀한 형식적 수학적 보안 환원(security reduction)이 확립되었다.

경쟁하는 양자 내성 방식들과 비교할 때, 격자 기반 접근법은 다음 측면에서 뚜렷한 이점이 있다:
- 연산 실행 시간(runtime) 효율성
- 암호문(ciphertext)의 크기
- 키 소재(key material)의 크기

### 보안 수준 및 키 사양

ML-KEM은 서로 다른 NIST 보안 등급을 가진 세 가지 변형이 존재한다:

| 변형 | NIST 보안 수준 | 비밀 키 크기 | 공개 키 크기 | 암호문 크기 | 동등한 대칭 보안 |
|------|---------------|-------------|-------------|-----------|----------------|
| ML-KEM-512 | 레벨 1 | 1632 바이트 | 800 바이트 | 768 바이트 | AES-128 |
| ML-KEM-768 | 레벨 3 | 2400 바이트 | 1184 바이트 | 1088 바이트 | AES-192 |
| ML-KEM-1024 | 레벨 5 | 3168 바이트 | 1568 바이트 | 1568 바이트 | AES-256 |

NIST 보안 수준의 의미:
- **레벨 1**: AES-128과 동등한 보안 강도 (양자 공격 포함)
- **레벨 3**: AES-192와 동등한 보안 강도
- **레벨 5**: AES-256과 동등한 보안 강도

### 구현 효율성

최적화된 구현으로 암호 연산은 약 **4킬로바이트의 메모리**만으로 수행할 수 있다. 이는 리소스가 제약된 임베디드 환경에서도 ML-KEM을 사용할 수 있게 한다.

#### 성능 비용 분석 (ECDH Curve25519 대비)

liboqs를 사용한 채팅 암호화 시나리오에서, 고효율 비양자 내성 ECDH 키 교환(Curve25519)을 ML-KEM으로 대체할 때의 영향:

| 지표 | 증가 배율 | 범위 |
|------|----------|------|
| 실행 시간 | 약 2.3배 | 1.5~7배 |
| 에너지 소비 | 약 2.3배 | 1.4~3.1배 |
| 데이터 오버헤드 | 약 70배 | 48~92배 |

내부 해싱 연산이 실행 시간의 대부분을 차지하며, 하드웨어 가속의 주요 후보가 된다.

## 역사

### 기원

표준화 이전에 ML-KEM은 **Kyber**라는 이름으로 알려져 있었다. 이 알고리즘은 2005년에 컴퓨터 과학자 **Oded Regev**가 발표한 방법론에서 파생되었다.

### 개발진

유럽과 북미의 정부 대학, 연구 기관, 민간 기업에 소속된 연구원들이 공동 개발에 참여했다. 지원 기관으로는 유럽 위원회, 스위스, 네덜란드, 독일 등이 있다.

개발자들은 상보적인 서명 체계 **Dilithium**(이후 ML-DSA로 표준화)도 함께 개발했으며, 이 두 구성 요소를 "대수적 격자를 위한 암호 스위트(Cryptographic Suite for Algebraic Lattices, **CRYSTALS**)"라고 명명했다.

### 내부 해시 사용

유사한 양자 내성 KEM 방식들과 마찬가지로, Kyber는 내부적으로 광범위한 해싱을 수행한다. 구체적으로 **Keccak(SHA-3/SHAKE)** 의 변형들을 사용하여 유사 난수를 생성하고 기타 용도에 활용한다.

## NIST 표준화 과정

### 2017년: 제출

이 방식은 양자 안전 암호 프리미티브를 목표로 하는 NIST의 공개 선발 과정에 제출되었다. Kyber는 표준화를 위해 선정된 **최초의 키 캡슐화 메커니즘**이 되었으며, 제3라운드까지 유일한 KEM 선정작으로 남았다.

결정 보고서의 각주에 따르면, 표준화는 다양한 특허 관련 합의의 이행을 조건으로 했으며, **NTRU**가 대체 옵션으로 지정되었다. 2025년의 제4 표준화 라운드에서는 **HQC** 체계가 추가 대체 옵션으로 선정되었다.

### 라운드 2 수정 사항 (Kyber v2)

제2라운드 제출에서 상당한 변경이 이루어졌다:

| 변경 사항 | 이전 | 이후 | 이유 |
|----------|------|------|------|
| 공개 키 압축 | 사용 | 제거 | NIST의 보안 증명 관련 의견에 대응 |
| 파라미터 q | 7681 | 3329 | 더 작은 모듈러스로 효율성 개선 |
| 암호문 압축 파라미터 | 이전 값 | 수정됨 | 보안-효율성 균형 조정 |
| NTT 정의 | 기존 방식 | NTRU 방식 준용 | 가속된 다항식 곱셈을 위해 |
| 노이즈 파라미터 η | 이전 값 | 2 | 더 빠른 노이즈 샘플링 |
| 공개 키 표현 | 표준 도메인 | NTT 도메인 | 별도 NTT 연산 제거 |

### 라운드 3 수정 사항

- **후지사키-오카모토 변환(FO transform)** 적용 방식 수정
- 레벨 1 파라미터 세트에서 **노이즈 수준 증가** 및 **암호문 압축 감소**
- **샘플링 알고리즘** 최적화

### FIPS 203 최초 공개 초안 (IPD)

FIPS 203 IPD에서 추가 변경이 이루어졌다:

- 공유 비밀 키 길이를 **256비트로 고정**
- 다른 **FO 변환 변형** 구현
- **NIST 승인 난수 생성** 방식 채택, 이전에 결함 있는 난수에 대비하던 연산 제거
- **명시적 입력 검사** 추가

### IPD 이후 수정 사항

- 키 생성에 **도메인 분리(domain separation)** 추가
- FIPS 203 IPD에서 부주의하게 생성된 **인덱스 불일치** 수정

### 하이브리드 접근법

현재 ML-KEM은 대부분 **하이브리드 모드**로 배치되고 있다. 이는 전통적인 키 교환(예: ECDH)과 ML-KEM을 동시에 수행하여:
- 기존의 보안 보장을 유지하면서
- 양자 컴퓨터에 대한 추가적인 보호를 제공한다.
- 두 방식 중 하나라도 안전하면 전체 시스템이 안전하다.

## 기술적 기반

### 모듈 LWE 문제

ML-KEM의 보안은 M-LWE 문제의 난해성에 기반한다. M-LWE 문제는 다음과 같이 정의된다:

행렬 **A** ∈ R_q^(k×k)와 벡터 **s** ∈ R_q^k, 오류 벡터 **e** ∈ R_q^k가 주어질 때:
- **b** = **A** · **s** + **e** (mod q)

에서 (**A**, **b**)가 주어졌을 때 **s**를 복원하는 것이 계산적으로 어렵다.

여기서:
- R_q = Z_q[x]/(x^n + 1)는 정수 모듈러 q에 대한 원분체 환이다.
- n = 256 (다항식의 차수)
- q = 3329 (모듈러스)
- k는 보안 수준에 따라 2, 3, 4 중 하나

### KEM 연산

ML-KEM은 세 가지 알고리즘으로 구성된다:

1. **KeyGen()**: 키 쌍 생성
   - 비밀 키 sk와 공개 키 pk를 생성
   - 행렬 **A**와 비밀 벡터 **s**, 오류 벡터 **e**를 샘플링
   - pk = (**A**, **b** = **A**·**s** + **e**)

2. **Encaps(pk)**: 캡슐화
   - 공개 키를 사용하여 공유 비밀 K와 암호문 c를 생성
   - 임시 비밀 **r**과 오류 벡터들을 샘플링
   - 암호문은 메시지 m을 **b**^T · **r** + e₂ + ⌈q/2⌋ · m으로 인코딩

3. **Decaps(sk, c)**: 역캡슐화
   - 비밀 키와 암호문을 사용하여 공유 비밀 K를 복원
   - 후지사키-오카모토 변환을 통해 CCA2 보안 달성

### IND-CCA2 보안

ML-KEM은 **후지사키-오카모토(FO) 변환**을 적용하여 CPA 안전한 공개 키 암호화 체계를 CCA2 안전한 KEM으로 변환한다. 이 변환은:
- 역캡슐화 시 캡슐화를 재수행하여 암호문이 유효한지 검증
- 유효하지 않은 암호문에 대해 무작위처럼 보이는 공유 비밀을 반환 (implicit rejection)
- 이를 통해 적응적 선택 암호문 공격(CCA2)에 대한 안전성을 보장

### NIST의 부채널 공격 대응 고려

NIST는 표준화 과정에서 **마스킹된 런타임 접근법**의 런타임 비용과 구현 복잡도에 특별한 주의를 기울였다. 마스킹은 부채널 공격을 방지하기 위해 설계된 대응책으로, 비밀 값을 여러 개의 공유(share)로 분할하여 처리한다.
