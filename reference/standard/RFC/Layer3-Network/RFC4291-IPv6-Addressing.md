# RFC 4291 - IP Version 6 Addressing Architecture

> 2006년 2월 발행, RFC 3513을 대체하는 IPv6 주소 체계 표준 명세서
> 128비트 IPv6 주소의 구조, 종류, 텍스트 표현 방식을 정의
> 이후 RFC 5952, 6052, 7136, 7346, 7371, 8064에 의해 갱신(Updated)됨

## 1. 개요

RFC 4291은 IPv6 프로토콜의 주소 체계(Addressing Architecture)를 정의합니다. 128비트 주소 공간의 구조, 주소 종류, 텍스트 표현 방법, 그리고 각 주소 유형별 형식을 규정합니다.

### IPv6 주소의 3가지 종류

| 종류 | 설명 | 전달 방식 |
|------|------|-----------|
| Unicast (유니캐스트) | 단일 인터페이스 식별자 | 해당 인터페이스로 1:1 전달 |
| Anycast (애니캐스트) | 여러 인터페이스(보통 다른 노드)의 집합 식별자 | 가장 가까운 하나의 인터페이스로 전달 |
| Multicast (멀티캐스트) | 여러 인터페이스의 집합 식별자 | 집합에 속한 모든 인터페이스로 전달 |

> "IPv6에는 브로드캐스트 주소가 없으며, 그 기능은 멀티캐스트 주소로 대체되었습니다."

## 2. 주소 모델 (Addressing Model)

- IPv6 주소는 **노드가 아닌 인터페이스**에 할당됩니다.
- 하나의 인터페이스는 여러 개의 IPv6 주소를 가질 수 있습니다.
- 모든 인터페이스는 최소한 하나의 Link-Local 유니캐스트 주소를 가져야 합니다.
- 유니캐스트 주소나 그 집합은 노드를 식별하는 데 사용될 수 있습니다 (인터페이스 중 하나의 주소로 노드 식별 가능).

### 인터페이스 식별 단위

> "IPv6 유니캐스트 주소는 인터페이스 식별자(Interface Identifier)와 서브넷 프리픽스(Subnet Prefix)로 구성되며, 토폴로지 정보를 기반으로 라우팅됩니다."

## 3. 텍스트 표현 (Text Representation)

IPv6 주소는 세 가지 일반적인 텍스트 표현 형식을 가집니다.

### 3.1 기본 형식 (Preferred Form)

```
x:x:x:x:x:x:x:x
```

- 각 `x`는 16비트(2옥텟)를 나타내는 16진수 값
- 8개의 16비트 그룹을 콜론(`:`)으로 구분
- 예시:
  ```
  ABCD:EF01:2345:6789:ABCD:EF01:2345:6789
  2001:DB8:0:0:8:800:200C:417A
  ```

### 3.2 0 압축 (`::` 표기)

연속된 0의 그룹을 `::`로 압축할 수 있습니다.

> "이중 콜론(::)은 하나 이상의 16비트 0 그룹을 나타냅니다.
 한 주소에서 ::는 단 한 번만 사용할 수 있습니다."

| 원본 주소 | 압축 형식 |
|-----------|-----------|
| `2001:DB8:0:0:8:800:200C:417A` (유니캐스트) | `2001:DB8::8:800:200C:417A` |
| `FF01:0:0:0:0:0:0:101` (멀티캐스트) | `FF01::101` |
| `0:0:0:0:0:0:0:1` (루프백) | `::1` |
| `0:0:0:0:0:0:0:0` (미지정) | `::` |

- `::`는 한 주소에서 **단 한 번만** 사용 가능 (그렇지 않으면 0 그룹 개수가 모호해짐)

### 3.3 IPv4 혼합 표기 (Mixed Notation)

IPv4와 IPv6가 혼재된 환경에서 사용하는 대체 형식입니다.

```
x:x:x:x:x:x:d.d.d.d
```

- 앞쪽 6개의 `x`는 16비트 16진수 그룹
- 뒤쪽 `d.d.d.d`는 IPv4 주소의 점 십진 표기 (32비트)
- 예시:
  ```
  0:0:0:0:0:0:13.1.68.3       →  ::13.1.68.3
  0:0:0:0:0:FFFF:129.144.52.38 →  ::FFFF:129.144.52.38
  ```

### 3.4 프리픽스 표현 (CIDR 표기)

```
ipv6-address/prefix-length
```

- `prefix-length`는 프리픽스를 구성하는 좌측 비트 수 (10진수)
- 예시: `2001:DB8::/48`, `2001:DB8:0:CD30::/60`

> 노드 주소와 그 프리픽스를 결합하여 쓸 수도 있습니다.
 예) `2001:DB8:0:CD30:123:4567:89AB:CDEF/60`

## 4. 주소 타입 식별

주소의 종류는 주소의 선두 비트(프리픽스)로 식별합니다.

| 주소 유형 | 이진 프리픽스 | IPv6 표기 |
|-----------|---------------|-----------|
| 미지정 (Unspecified) | 00...0 (128비트) | `::/128` |
| 루프백 (Loopback) | 00...1 (128비트) | `::1/128` |
| 멀티캐스트 | `11111111` | `FF00::/8` |
| Link-Local 유니캐스트 | `1111111010` | `FE80::/10` |
| 글로벌 유니캐스트 | 그 외 모든 값 | — |

> "애니캐스트 주소는 유니캐스트 주소 공간에서 할당되며, 유니캐스트 주소와 문법적으로 구별되지 않습니다."

## 5. 유니캐스트 주소 (Unicast Addresses)

### 5.1 글로벌 유니캐스트 주소 구조

```
|         n bits          |   m bits  |       128-n-m bits         |
+-------------------------+-----------+----------------------------+
| global routing prefix   | subnet ID |       interface ID         |
+-------------------------+-----------+----------------------------+
```

- Global Routing Prefix: 사이트(서브넷 집합)에 할당된 값
- Subnet ID: 사이트 내 서브넷(링크) 식별자
- Interface Identifier: 링크 내 인터페이스 식별자

### 5.2 인터페이스 식별자 (Interface Identifier)

> "000(이진수)으로 시작하는 것을 제외한 모든 유니캐스트 주소의 인터페이스 식별자는 64비트 길이이며, Modified EUI-64 형식으로 구성되어야 합니다."

> 참고: 이 MUST 규정은 RFC 7136에 의해 완화되었다.
 RFC 7136은 인터페이스 식별자의 내부 구조(U/L 비트 등)에 의미를 부여하지 말고 단순히 64비트 불투명 값으로 취급해야 한다고 명시하며, Modified EUI-64 형식 사용을 더 이상 강제하지 않는다.

#### Modified EUI-64 형식

- IEEE EUI-64 식별자를 기반으로 함
- IEEE 802 48비트 MAC 주소로부터 생성 시:
  1. MAC 주소 중앙(OUI와 NIC 사이)에 `FFFE`(16비트) 삽입 → 64비트
  2. 선두에서 7번째 비트인 **U/L 비트(Universal/Local)를 반전**

```
MAC 주소:  00-1B-63-84-45-E6
                  ↓ FFFE 삽입
          00-1B-63-FF-FE-84-45-E6
                  ↓ U/L 비트 반전 (00 → 02)
          02-1B-63-FF-FE-84-45-E6
                  ↓
EUI-64 인터페이스 ID: 021B:63FF:FE84:45E6
```

| 비트 | 의미 |
|------|------|
| U/L (Universal/Local) | 0 = 로컬 관리, 1 = 전역 고유 (반전 후) |
| I/G (Individual/Group) | 0 = 개별 주소, 1 = 그룹 주소 |

### 5.3 특수 유니캐스트 주소

#### 미지정 주소 (Unspecified)

```
::  (0:0:0:0:0:0:0:0)
```

- 인터페이스에 주소가 없음을 의미
- 출발지 주소로만 사용 가능 (예: 주소 설정 초기화 중 DAD)
- 목적지 주소로 사용 불가, 라우터가 포워딩하지 않음

#### 루프백 주소 (Loopback)

```
::1  (0:0:0:0:0:0:0:1)
```

- 노드가 자기 자신에게 패킷을 보낼 때 사용 (IPv4의 `127.0.0.1`에 해당)
- 인터페이스에 할당 불가, 노드 외부로 전송 불가

### 5.4 IPv4 관련 임베디드 주소

#### IPv4-Compatible IPv6 주소 (폐기됨)

```
|        80 bits        | 16 |     32 bits     |
+-----------------------+----+-----------------+
|0000.............0000  |0000| IPv4 address    |
+-----------------------+----+-----------------+
```

> RFC 4291에서 이 형식은 **더 이상 사용되지 않음**(deprecated)으로 지정되었습니다.

#### IPv4-Mapped IPv6 주소

```
|        80 bits        | 16 |     32 bits     |
+-----------------------+----+-----------------+
|0000.............0000  |FFFF| IPv4 address    |
+-----------------------+----+-----------------+
```

- IPv4 노드의 주소를 IPv6 주소로 표현 (예: `::FFFF:192.168.1.1`)
- IPv4-only 노드와 통신하는 IPv6 애플리케이션에서 사용

### 5.5 Link-Local 주소

```
|   10 bits  |         54 bits         |          64 bits           |
+------------+-------------------------+----------------------------+
| 1111111010 |           0             |       interface ID         |
+------------+-------------------------+----------------------------+
```

- 프리픽스: `FE80::/10`
- 단일 링크 상에서만 유효 (이웃 탐색, 자동 주소 설정 등)
- 라우터는 Link-Local 출발지/목적지 패킷을 다른 링크로 포워딩하면 안 됨

> 참고: Site-Local 주소(`FEC0::/10`)는 본 RFC가 발행되기 전 RFC 3879에 의해 이미 **폐기**되었습니다.

## 6. 애니캐스트 주소 (Anycast Addresses)

> "애니캐스트 주소는 여러 인터페이스(보통 서로 다른 노드)에 할당되며, 해당 주소로 보낸 패킷은 라우팅 거리상 '가장 가까운' 하나의 인터페이스로 전달됩니다."

- 유니캐스트 주소 공간에서 할당되며, 문법적으로 유니캐스트와 동일
- 여러 인터페이스에 같은 주소를 할당함으로써 애니캐스트가 됨

### Subnet-Router 애니캐스트 주소 (예약됨)

```
|                         n bits                 |   128-n bits    |
+------------------------------------------------+-----------------+
|                   subnet prefix                | 00000000000000  |
+------------------------------------------------+-----------------+
```

- 서브넷 프리픽스 + 인터페이스 ID 전체를 0으로 설정
- 해당 서브넷의 모든 라우터에 자동 할당
- 특정 서브넷의 라우터 중 하나에 도달해야 할 때 사용

## 7. 멀티캐스트 주소 (Multicast Addresses)

### 멀티캐스트 주소 형식

```
|   8    |  4 |  4 |                  112 bits                   |
+--------+----+----+---------------------------------------------+
|11111111|flgs|scop|                  group ID                   |
+--------+----+----+---------------------------------------------+
```

- 프리픽스: `11111111` (`FF00::/8`)

#### flgs (4비트 플래그)

```
0 R P T
```

| 플래그 | 의미 |
|--------|------|
| T (Transient) | 0 = 영구 할당(well-known), 1 = 임시(non-permanent) |
| P (Prefix) | 1 = 유니캐스트 프리픽스 기반 멀티캐스트 (RFC 3306) |
| R (Rendezvous) | 1 = RP 주소 내장 (RFC 3956) |

#### scop (4비트 스코프)

| 값 | 스코프 |
|----|--------|
| 1 | Interface-Local (노드 내부) |
| 2 | Link-Local |
| 4 | Admin-Local |
| 5 | Site-Local |
| 8 | Organization-Local |
| E | Global |

### 사전 정의된 멀티캐스트 주소

| 주소 | 의미 |
|------|------|
| `FF01::1` | Interface-Local All Nodes |
| `FF02::1` | Link-Local All Nodes |
| `FF01::2` | Interface-Local All Routers |
| `FF02::2` | Link-Local All Routers |
| `FF05::2` | Site-Local All Routers |

### Solicited-Node 멀티캐스트 주소

이웃 탐색(Neighbor Discovery)에서 핵심적으로 사용됩니다.

```
FF02:0:0:0:0:1:FFXX:XXXX
```

- 프리픽스 `FF02::1:FF00:0/104` + 유니캐스트/애니캐스트 주소의 하위 24비트
- 모든 노드는 자신의 유니캐스트·애니캐스트 주소마다 대응하는 Solicited-Node 멀티캐스트 그룹에 가입해야 함
- 예: `4037::01:800:200E:8C6C` → `FF02::1:FF0E:8C6C`

> "Solicited-Node 멀티캐스트 주소는 효율적인 주소 해석(Address Resolution)을 위해 사용되어 브로드캐스트 부하를 줄입니다."

## 8. 노드가 가져야 할 필수 주소 (Required Node Addresses)

### 호스트가 인식해야 하는 주소

- 모든 인터페이스의 Link-Local 주소
- 할당된 유니캐스트/애니캐스트 주소
- 루프백 주소 (`::1`)
- All-Nodes 멀티캐스트 주소 (`FF01::1`, `FF02::1`)
- 가입한 각 유니캐스트·애니캐스트 주소의 Solicited-Node 멀티캐스트 주소
- 가입한 그 외 그룹의 멀티캐스트 주소

### 라우터가 추가로 인식해야 하는 주소

- 설정된 모든 인터페이스의 Subnet-Router 애니캐스트 주소
- 설정된 모든 애니캐스트 주소
- All-Routers 멀티캐스트 주소 (`FF01::2`, `FF02::2`, `FF05::2`)

## 9. 요약

RFC 4291은 IPv6 주소 체계의 근간을 다음과 같이 정의합니다:

- **3가지 주소 종류**: 유니캐스트, 애니캐스트, 멀티캐스트 (브로드캐스트 없음)
- **주소는 인터페이스에 할당**되며, 하나의 인터페이스가 여러 주소 보유 가능
- **텍스트 표현**: 기본 형식, `::` 0 압축, IPv4 혼합 표기
- **64비트 인터페이스 식별자**: Modified EUI-64 형식
- **특수 주소**: 미지정(`::`), 루프백(`::1`), Link-Local(`FE80::/10`)
- **Solicited-Node 멀티캐스트**: 효율적 이웃 탐색의 핵심
