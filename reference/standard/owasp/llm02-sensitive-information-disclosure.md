# OWASP LLM02: Sensitive Information Disclosure

> 공식 문서: https://genai.owasp.org/llmrisk/llm022025-sensitive-information-disclosure/

## 1. 개요

LLM 애플리케이션이 학습 데이터, 시스템 프롬프트, 사용자 대화 기록, 연동된 백엔드 데이터 등에 포함된 민감 정보를 의도치 않게 노출하는 위험을 다룬다. 전통적인 데이터 유출과 달리, LLM은 자연어 응답이라는 형태로 정보를 흘려보내기 때문에 기존 DLP(Data Loss Prevention) 도구가 탐지하기 어려운 경로로 유출이 발생할 수 있다.

---

## 2. 유출 경로 분류

| 경로 | 설명 |
|------|------|
| 학습 데이터 유출 | 모델이 학습 과정에서 본 개인정보/기밀 정보를 응답에 그대로 재현 |
| 시스템 프롬프트 유출 | [Prompt Injection](./llm01-prompt-injection.md)이나 정교한 질의로 내부 지시사항/설정을 그대로 노출 |
| 대화 기록 오염 | 다중 사용자 세션 간 컨텍스트가 잘못 섞여 다른 사용자의 대화 내용이 노출 |
| RAG/도구 연동 데이터 유출 | 검색된 문서나 도구 응답에 포함된 민감 데이터가 걸러지지 않고 최종 응답에 그대로 포함 |
| 로그/모니터링 유출 | 프롬프트/응답을 저장하는 로깅 시스템 자체의 접근 통제 미흡 |

---

## 3. 대표적인 악용/사고 시나리오

| 시나리오 | 위험 |
|----------|------|
| 조회 권한 없는 데이터 노출 | RAG 시스템이 사용자별 접근 권한(Row-Level Security 등)을 반영하지 않고 전체 문서를 검색 대상으로 삼음 |
| 프롬프트 캐시/공유 인프라의 데이터 혼입 | 멀티테넌트 환경에서 테넌트 간 컨텍스트 격리 실패 |
| 세밀한 질의를 통한 정보 재구성 | 직접 공개하지 않아도, 여러 번의 우회 질문으로 민감 정보를 유추/재구성 |
| 파인튜닝 데이터의 개인정보 잔존 | 파인튜닝에 사용한 사내 데이터에 포함된 PII가 응답에 재현됨 |

---

## 4. 완화 전략

| 전략 | 설명 |
|------|------|
| 데이터 최소화(Data Minimization) | 학습/파인튜닝/컨텍스트에 포함하는 데이터 자체에서 불필요한 민감 정보를 사전 제거 |
| 출력 필터링 | 응답을 반환하기 전 PII/기밀 패턴을 탐지해 마스킹하는 후처리 계층 도입 |
| 접근 제어 연동 | RAG 등에서 사용자의 실제 권한에 맞는 문서만 검색 대상에 포함 (권한 필터를 검색 이전 단계에 적용) |
| 테넌트/세션 격리 | 멀티테넌트 환경에서 컨텍스트, 캐시, 로그가 테넌트 경계를 넘지 않도록 아키텍처 설계 |
| 시스템 프롬프트 최소 신뢰 | 시스템 프롬프트에 민감 정보(자격 증명, 내부 정책 원문 등)를 직접 담지 않기 |
| 로그 접근 통제 | 프롬프트/응답 로그 저장소에도 일반 민감 데이터와 동일한 수준의 접근 통제 적용 |

---

## 5. Prompt Injection과의 연관성

[LLM01 Prompt Injection](./llm01-prompt-injection.md)은 정보 유출을 유발하는 대표적인 "공격 벡터"이고, LLM02는 그 결과로 발생하는 "피해 유형" 중 하나로 볼 수 있다. 즉 두 위험은 독립적이라기보다, 프롬프트 인젝션 방어가 부실하면 민감 정보 노출로 이어지는 인과 관계에 있는 경우가 많다.

---

## 6. 요약

- Sensitive Information Disclosure는 학습 데이터, 시스템 프롬프트, 대화 기록, 연동 데이터 등 다양한 경로로 발생할 수 있는 정보 유출 위험이다.
- 멀티테넌트/RAG 환경에서는 권한 필터링과 테넌트 격리가 핵심 방어선이다.
- Prompt Injection 방어와 밀접하게 연관되어 있어, 두 위험을 함께 고려한 설계가 필요하다.
