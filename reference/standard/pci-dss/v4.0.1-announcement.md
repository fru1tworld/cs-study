# PCI DSS v4.0.1 발행 공지

> 공식 공지: https://blog.pcisecuritystandards.org/just-published-pci-dss-v4-0-1

## 1. 개요

PCI DSS(Payment Card Industry Data Security Standard)는 신용/직불카드 결제 데이터를 처리·저장·전송하는 모든 조직이 준수해야 하는 보안 표준으로, PCI SSC(Payment Card Industry Security Standards Council)가 관리한다. v4.0.1은 2022년 발표된 v4.0을 명확화(clarification)하고 오류를 정정한 개정판으로, 새로운 요구사항을 추가한 것이 아니라 v4.0의 해석 모호성을 줄이는 데 초점을 맞췄다.

| 구분 | 설명 |
|------|------|
| 발행 형태 | v4.0의 마이너 개정(minor revision) |
| 주요 변경 성격 | 요구사항 문구 명확화, 오탈자/모순 수정, 적용 시점 안내 보완 |
| 신규 보안 요구사항 여부 | 원칙적으로 없음 (v4.0 대비 실질적 보안 요구 추가는 아님) |

---

## 2. v4.0 → v4.0.1 변경의 성격

PCI DSS는 버전이 올라갈수록 다음과 같은 큰 흐름의 개정을 거쳐 왔다.

| 버전 흐름 | 특징 |
|-----------|------|
| v3.2.1 → v4.0 | 대규모 개정: 커스터마이즈 접근법 도입, MFA 요구 확대, 암호화/로깅 요구 강화 |
| v4.0 → v4.0.1 | 소규모 정정: 실무 적용 과정에서 발견된 모호한 문구·오류를 정리 |

v4.0.1 자체는 조직이 대응해야 할 새로운 통제(control)를 추가하지 않지만, 기존 v4.0 요구사항의 해석이 바뀌거나 명확해진 부분이 있으므로 컴플라이언스 담당자는 변경 로그(Summary of Changes)를 확인해 자사 정책 문서와 대조할 필요가 있다.

---

## 3. v4.0의 핵심 방향 (배경 지식)

v4.0.1을 이해하려면 v4.0이 도입한 핵심 변화를 함께 알아야 한다.

| 핵심 변화 | 설명 |
|-----------|------|
| Customized Approach | 표준이 명시한 방법(Defined Approach) 대신, 조직이 스스로 설계한 통제로 요구사항의 목표(Objective)를 달성했음을 증명하는 대안 경로 도입 |
| MFA 요구 확대 | 카드 데이터 환경(CDE)에 접근하는 모든 계정에 다중 인증 요구 범위 확대 |
| 암호화/키 관리 강화 | 저장 데이터 암호화, 암호화 키 수명주기 관리 요구 세분화 |
| 로깅/모니터링 강화 | 로그 검토 주기, 자동화된 이상 탐지 요구 명시 |
| 타겟 준수 날짜(Future-dated requirements) | 일부 신규 요구사항은 즉시가 아니라 유예 기간 이후(2025년 3월 31일 등) 강제 적용되도록 단계적 시행 일정 부여 |

---

## 4. 조직이 확인해야 할 실무 체크리스트

| 항목 | 확인 사항 |
|------|-----------|
| 변경 로그 검토 | v4.0.1의 "Summary of Changes" 문서에서 자사 CDE 범위와 관련된 항목 확인 |
| 유예 기간(Future-dated) 요구사항 캘린더 관리 | 아직 발효되지 않은 v4.0 요구사항의 적용 시점을 놓치지 않도록 추적 |
| QSA/내부 감사팀과 해석 정합성 확인 | 명확화된 문구가 기존 감사 방식에 영향을 주는지 확인 |
| Customized Approach 채택 여부 재검토 | 명확화 내용이 커스터마이즈 접근법 설계에 영향을 주는지 확인 |

---

## 5. 요약

- PCI DSS v4.0.1은 v4.0의 실질적 요구사항을 바꾸지 않는 명확화·정정 목적의 마이너 개정판이다.
- v4.0이 도입한 Customized Approach, 확대된 MFA, 강화된 암호화/로깅 요구는 v4.0.1에서도 그대로 유지된다.
- 조직은 변경 로그와 유예 기간이 있는 요구사항의 적용 일정을 별도로 관리해야 한다.
