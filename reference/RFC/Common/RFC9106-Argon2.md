> 패스워드 해싱 및 작업 증명 애플리케이션을 위한 Argon2 메모리 하드 함수

## 1. 개요 (Abstract)

이 문서는 패스워드 해싱 및 작업 증명(Proof-of-Work) 애플리케이션을 위한 **Argon2 메모리 하드 함수**를 설명합니다. 이 문서의 목적은 인터넷 프로토콜에서 Argon2의 채택을 간소화하는 것이며, 구현자를 위한 지침과 테스트 벡터를 제공합니다.

---

## 2. 소개 (Introduction)

Argon2는 x86 아키텍처에 최적화된 메모리 하드 함수로, 현대 프로세서의 캐시와 메모리 구성을 활용합니다.

### 2.1 Argon2의 세 가지 변형

| 변형 | 메모리 접근 방식 | 용도 | 특징 |
|------|-----------------|------|------|
| **Argon2d** | 데이터 의존적 | 암호화폐, 작업 증명 | 타이밍 공격이 위협이 아닌 환경에 적합 |
| **Argon2i** | 데이터 독립적 | 패스워드 해싱 | 부채널 공격 보호 |
| **Argon2id** | 하이브리드 | 범용 | 첫 번째 패스의 전반부는 Argon2i, 이후는 Argon2d 방식 사용 |

**중요**: 이 문서를 구현하는 모든 구현체는 **Argon2id를 반드시(MUST) 지원**해야 하며, Argon2d와 Argon2i는 선택적으로(MAY) 지원할 수 있습니다.

## 4. Argon2 알고리즘 명세 (Argon2 Specification)

### 4.1 입력 및 출력

#### 4.1.1 필수 매개변수

| 매개변수 | 기호 | 범위 | 설명 |
|---------|------|------|------|
| 메시지/패스워드 | P | 0 ~ 2^32-1 바이트 | 해싱할 패스워드 |
| 논스/솔트 | S | 0 ~ 2^32-1 바이트 | 솔트 값 (16바이트 권장) |
| 병렬 처리 정도 | p | 1 ~ 2^24-1 | 레인(lane) 수 |
| 태그 길이 | T | 4 ~ 2^32-1 바이트 | 출력 해시 길이 |
| 메모리 크기 | m | 8×p ~ 2^32-1 KiB | 사용할 메모리 (kibibytes) |
| 반복 횟수 | t | 1 ~ 2^32-1 | 패스(pass) 횟수 |
| 버전 번호 | v | 0x13 | 현재 버전 |
| 타입 | y | 0, 1, 2 | 0=Argon2d, 1=Argon2i, 2=Argon2id |

#### 4.1.2 선택적 매개변수

| 매개변수 | 기호 | 범위 | 설명 |
|---------|------|------|------|
| 비밀 키 | K | 0 ~ 2^32-1 바이트 | 추가 비밀 값 |
| 연관 데이터 | X | 0 ~ 2^32-1 바이트 | 추가 연관 데이터 |

#### 4.1.3 출력

- **태그**: T 바이트 길이의 출력 해시

### 4.2 메모리 구조

메모리는 **B[i][j]** 행렬로 구성됩니다:
- **행(row)**: p개의 레인 (i = 0 ~ p-1)
- **열(column)**: q개의 블록 (j = 0 ~ q-1)
- **블록 크기**: 1024 바이트
- **총 메모리**: m' = 4 × p × floor(m / 4p) 블록

```
메모리 레이아웃:

        열 0    열 1    열 2    ...    열 q-1
      ┌───────┬───────┬───────┬─────┬───────┐
레인 0 │B[0][0]│B[0][1]│B[0][2]│ ... │B[0][q-1]│
      ├───────┼───────┼───────┼─────┼───────┤
레인 1 │B[1][0]│B[1][1]│B[1][2]│ ... │B[1][q-1]│
      ├───────┼───────┼───────┼─────┼───────┤
  ...  │  ...  │  ...  │  ...  │ ... │  ...  │
      ├───────┼───────┼───────┼─────┼───────┤
레인p-1│B[p-1][0]│B[p-1][1]│B[p-1][2]│...│B[p-1][q-1]│
      └───────┴───────┴───────┴─────┴───────┘

        ←─────────────── 4개의 슬라이스 ───────────────→
        │  슬라이스 0  │  슬라이스 1  │  슬라이스 2  │  슬라이스 3  │
```

### 4.3 알고리즘 단계

#### 단계 1: H₀ 초기화

64바이트 값 H₀를 BLAKE2b로 계산합니다:

```
H₀ = H^64(LE32(p) || LE32(T) || LE32(m) || LE32(t) ||
          LE32(v) || LE32(y) ||
          LE32(length(P)) || P ||
          LE32(length(S)) || S ||
          LE32(length(K)) || K ||
          LE32(length(X)) || X)
```

여기서 H^64는 64바이트 다이제스트를 생성하는 BLAKE2b 함수입니다.

#### 단계 2: 메모리 할당

m' = 4 × p × floor(m / 4p) 개의 1024바이트 블록을 할당합니다.

#### 단계 3-4: 초기 블록 생성

각 레인 i (0 ≤ i < p)에 대해:

```
B[i][0] = H'^1024(H₀ || LE32(0) || LE32(i))
B[i][1] = H'^1024(H₀ || LE32(1) || LE32(i))
```

여기서 H'^1024는 1024바이트 출력을 생성하는 가변 길이 해시 함수입니다.

#### 단계 5: 메인 계산 (첫 번째 패스)

j = 2부터 q-1까지 각 블록에 대해:

```
B[i][j] = G(B[i][j-1], B[l][z])
```

여기서:
- G는 압축 함수
- B[l][z]는 인덱싱 규칙에 따라 선택된 참조 블록

#### 단계 6: 추가 패스 (t > 1인 경우)

패스 r = 1부터 t-1까지:

```
B[i][0] = G(B[i][q-1], B[l][z]) XOR B[i][0]
B[i][j] = G(B[i][j-1], B[l][z]) XOR B[i][j]   (j > 0)
```

#### 단계 7: 최종 블록 계산

```
C = B[0][q-1] XOR B[1][q-1] XOR ... XOR B[p-1][q-1]
```

#### 단계 8: 출력 생성

```
Tag = H'^T(C)
```

### 4.4 가변 길이 해시 함수 H'

H'^T는 BLAKE2b를 확장하여 임의 길이 출력을 생성합니다:

```
만약 T ≤ 64:
    H'^T(A) = H^T(LE32(T) || A)

그렇지 않으면:
    r = ceil(T/32) - 2
    V₁ = H^64(LE32(T) || A)
    V₂ = H^64(V₁)
    ...
    V_r = H^64(V_{r-1})
    V_{r+1} = H^(T - 32×r)(V_r)

    H'^T(X) = W₁ || W₂ || ... || W_r || V_{r+1}
```

여기서 W_i는 V_i의 처음 32바이트입니다.

### 4.5 인덱싱 및 참조 블록 선택

메모리 행렬은 4개의 수직 **슬라이스**로 분할됩니다. 각 슬라이스-레인 교차점은 길이 q/4의 **세그먼트**를 형성합니다.

#### 4.5.1 J₁ 및 J₂ 계산

**Argon2d의 경우:**
```
J₁ = int32(extract(B[i][j-1], 0))
J₂ = int32(extract(B[i][j-1], 1))
```
이전 블록 B[i][j-1]의 처음 64비트에서 파생됩니다.

**Argon2i의 경우:**

의사 난수 인덱스를 생성합니다:
```
입력 Z = LE64(r) || LE64(l) || LE64(sl) || LE64(m') ||
         LE64(t) || LE64(y)
```

여기서:
- r = 패스 번호
- l = 레인 번호
- sl = 슬라이스 번호
- m' = 총 블록 수
- t = 총 패스 수
- y = Argon2 타입

BLAKE2b를 적용하여 q/SL 세트의 1024바이트 값을 생성하고, 이를 처리하여 J₁, J₂ 쌍을 생성합니다.

**Argon2id의 경우:**
- 패스 0, 슬라이스 0-1: Argon2i 로직 사용
- 그 외: Argon2d 로직 사용

#### 4.5.2 참조 블록 인덱스 매핑

**레인 인덱스:**
```
l = J₂ mod p
```

**집합 W 선택:**

1. **같은 레인인 경우** (l == i):
   - 마지막 3개 완료된 세그먼트의 블록 포함
   - 현재 세그먼트의 블록 포함 (B[i][j-1] 제외)

2. **다른 레인인 경우** (l != i):
   - 마지막 3개 완료된 세그먼트의 블록 포함
   - B[i][j]가 세그먼트의 첫 번째 블록이면 마지막 인덱스 제외

**블록 인덱스 계산:**
```
x = J₁² / 2^32
y = (|W| × x) / 2^32
zz = |W| - 1 - y
z = W[zz]
```

## 5. 압축 함수 G (Compression Function G)

압축 함수 G는 두 개의 1024바이트 블록 X, Y를 입력받아 1024바이트 블록을 출력합니다.

### 5.1 동작 과정

```
1. R = X XOR Y

2. R을 8×8 행렬의 16바이트 레지스터로 해석:

   ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
   │R[0] │R[1] │R[2] │R[3] │R[4] │R[5] │R[6] │R[7] │
   ├─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┤
   │R[8] │R[9] │R[10]│R[11]│R[12]│R[13]│R[14]│R[15]│
   ├─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┤
   │ ... │ ... │ ... │ ... │ ... │ ... │ ... │ ... │
   ├─────┼─────┼─────┼─────┼─────┼─────┼─────┼─────┤
   │R[56]│R[57]│R[58]│R[59]│R[60]│R[61]│R[62]│R[63]│
   └─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘

3. 각 행에 순열 P 적용 → Q
4. 각 열에 순열 P 적용 → Z
5. 출력: Z XOR R
```

### 5.2 순열 P (Permutation P)

P는 8개의 16바이트 입력을 처리합니다. 이를 4×4 행렬의 64비트 워드로 해석합니다:

```
┌────┬────┬────┬────┐
│ v₀ │ v₁ │ v₂ │ v₃ │
├────┼────┼────┼────┤
│ v₄ │ v₅ │ v₆ │ v₇ │
├────┼────┼────┼────┤
│ v₈ │ v₉ │v₁₀ │v₁₁ │
├────┼────┼────┼────┤
│v₁₂ │v₁₃ │v₁₄ │v₁₅ │
└────┴────┴────┴────┘
```

**GB 함수를 다음 순서로 적용:**

1. 열 방향:
   - GB(v₀, v₄, v₈, v₁₂)
   - GB(v₁, v₅, v₉, v₁₃)
   - GB(v₂, v₆, v₁₀, v₁₄)
   - GB(v₃, v₇, v₁₁, v₁₅)

2. 대각선 방향:
   - GB(v₀, v₅, v₁₀, v₁₅)
   - GB(v₁, v₆, v₁₁, v₁₂)
   - GB(v₂, v₇, v₈, v₁₃)
   - GB(v₃, v₄, v₉, v₁₄)

### 5.3 GB 함수

GB 함수는 BLAKE2b 라운드 함수를 기반으로 하지만, **곱셈 연산**을 추가하여 ASIC 저항성을 높입니다.

```
GB(a, b, c, d):

    a = (a + b + 2 × trunc(a) × trunc(b)) mod 2^64
    d = (d XOR a) >>> 32
    c = (c + d + 2 × trunc(c) × trunc(d)) mod 2^64
    b = (b XOR c) >>> 24

    a = (a + b + 2 × trunc(a) × trunc(b)) mod 2^64
    d = (d XOR a) >>> 16
    c = (c + d + 2 × trunc(c) × trunc(d)) mod 2^64
    b = (b XOR c) >>> 63
```

**핵심 차이점**: 표준 BLAKE2b와 달리 64비트 곱셈을 사용하여 회로 깊이를 증가시키고, ASIC 구현을 어렵게 만듭니다.

## 6. 매개변수 선택 지침 (Parameter Choice)

### 6.1 용도별 권장 설정

#### 6.1.1 작업 증명 (Proof-of-Work) - Argon2d

| 용도 | 레인 | 메모리 | 시간 |
|------|------|--------|------|
| 암호화폐 채굴 | 2 | 250 MB | ~0.1초 (2 GHz, 1코어) |

#### 6.1.2 범용 (General Purpose) - Argon2id

| 용도 | 레인 | 메모리 | 시간 |
|------|------|--------|------|
| 백엔드 인증 | 8 | 4 GiB | ~0.5초 (4코어) |
| 디스크 암호화 | 4 | 6 GiB | ~3초 (2코어) |
| 프론트엔드 인증 | 4 | 1 GiB | ~0.5초 (2코어) |

### 6.2 권장 기본 설정

#### 첫 번째 권장 옵션 (First Recommended)

| 매개변수 | 값 | 설명 |
|---------|-----|------|
| 타입 | Argon2id | |
| 반복 횟수 (t) | 1 | |
| 병렬 처리 (p) | 4 레인 | |
| 메모리 (m) | 2^21 (2 GiB) | |
| 솔트 길이 | 128비트 (16바이트) | |
| 태그 길이 | 256비트 (32바이트) | |

> 이 설정은 **모든 환경에서 기본값으로 권장**됩니다. 부채널 공격에 대한 보안을 제공하고, 전용 무차별 대입 하드웨어에서 공격자 비용을 최대화합니다.

#### 두 번째 권장 옵션 (Second Recommended)

메모리가 제한된 환경에서 사용합니다:

| 매개변수 | 값 | 설명 |
|---------|-----|------|
| 타입 | Argon2id | |
| 반복 횟수 (t) | 3 | |
| 병렬 처리 (p) | 4 레인 | |
| 메모리 (m) | 2^16 (64 MiB) | |
| 솔트 길이 | 128비트 (16바이트) | |
| 태그 길이 | 256비트 (32바이트) | |

### 6.3 매개변수 선택 절차

1. **타입 선택**: 특별한 제약이 없다면 Argon2id 선택
2. **병렬 처리 설정**: p=4 레인으로 시작
3. **메모리 결정**: 사용 가능한 최대 메모리 → m 매개변수
4. **허용 시간 측정**: 최대 허용 실행 시간 결정
5. **솔트 길이 선택**: 128비트 표준 (공간 제약 시 64비트 허용)
6. **태그 길이 선택**: 대부분의 애플리케이션에서 128비트
7. **메모리 초기화**: 부채널 위협이 있으면 메모리 와이핑 활성화
8. **벤치마크**: 다양한 t 값으로 테스트하여 시간 예산 내 최대 t 사용

### 6.4 솔트 요구사항

| 요구사항 | 설명 |
|---------|------|
| 최소 길이 | 16바이트 (128비트) |
| 고유성 | 각 패스워드에 대해 무작위로 생성된 고유한 값 |
| 저장 | 해시된 패스워드와 함께 저장 (비밀이 아님) |

## 7. 인코딩 (Encoding)

### 7.1 PHC 문자열 형식

Argon2 해시는 다음 형식으로 인코딩될 수 있습니다:

```
$argon2<T>$v=<num>$m=<num>,t=<num>,p=<num>$<base64-salt>$<base64-hash>
```

**구성 요소:**

| 필드 | 설명 | 예시 |
|------|------|------|
| `<T>` | 타입 식별자 | d, i, 또는 id |
| `v=<num>` | 버전 번호 | v=19 (0x13) |
| `m=<num>` | 메모리 (KiB) | m=65536 |
| `t=<num>` | 반복 횟수 | t=3 |
| `p=<num>` | 병렬 처리 | p=4 |
| `<base64-salt>` | Base64 인코딩된 솔트 | |
| `<base64-hash>` | Base64 인코딩된 해시 | |

**예시:**
```
$argon2id$v=19$m=65536,t=3,p=4$c29tZXNhbHQ$RdescudvJCsgt3ub+b+dWRWJTmaaJObG
```

## 9. 보안 고려사항 (Security Considerations)

### 9.1 충돌 저항성 및 프리이미지 저항성

| 속성 | 보안 수준 |
|------|----------|
| 충돌 저항성 | 2^256 연산 (BLAKE2b와 동등) |
| 프리이미지 저항성 | 2^512 연산 (BLAKE2b와 동등) |

### 9.2 시간-공간 트레이드오프 공격

#### 9.2.1 Argon2i

| 패스 수 | 공격 방식 | 감소 계수 |
|--------|----------|----------|
| 1-2 패스 | 저장소 공격 | ~5× |
| 3+ 패스 (1 GiB) | 저장소 공격 | ~3× |

점근적 한계: O(m^0.233) 이점, 최적 증명: O(m^0.25)

#### 9.2.2 Argon2d

| 공격 방식 | 감소 계수 |
|----------|----------|
| 랭킹 트레이드오프 공격 | ~1.33× |

#### 9.2.3 Argon2id

| 패스 수 | 감소 계수 |
|--------|----------|
| 1 패스 | ~2.1× (복합 공격) |
| 다중 패스 | ~1.33× |

### 9.3 예방 조치

트레이드오프 공격을 방지하기 위해:

```
패스 수(t)는 반드시(MUST) (log₂(m) - 26)보다 커야 합니다.
```

예: 2 GiB (m = 2^21) 메모리의 경우, t > 21 - 26 = -5 이므로 t ≥ 1이면 됩니다.

### 9.4 시간 제한 방어자 시나리오

고정된 방어자 계산 시간에 대해:
- **Argon2i**: 3 패스가 거의 최적
- **Argon2d/Argon2id**: 1 패스가 공격자 비용 최대화

### 9.5 부채널 공격

| 변형 | 부채널 보호 |
|------|------------|
| Argon2d | 취약 (데이터 의존적 메모리 접근) |
| Argon2i | 보호됨 (데이터 독립적 메모리 접근) |
| Argon2id | 부분 보호 (첫 패스 전반부만 Argon2i) |

### 9.6 권장 사항 요약

> **Argon2id with t=1 and 2 GiB memory**가 **첫 번째 권장 옵션**이며, 모든 환경에서 기본 설정으로 제안됩니다. 이 설정은 부채널 공격에 대해 안전하고, 전용 무차별 대입 하드웨어에서 공격자 비용을 최대화합니다.
