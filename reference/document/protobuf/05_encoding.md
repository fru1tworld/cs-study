## 개요

Protocol Buffers는 메시지를 일련의 **(태그, 값) 쌍**으로 인코딩한 바이너리 스트림으로 직렬화합니다. 핵심 구성 요소는 가변 길이 정수인 **varint**와, 필드 번호 + 와이어 타입을 담은 **태그(tag)**입니다.

---

## Varint 인코딩

varint는 부호 없는 64비트 정수를 1~10바이트로 인코딩합니다. 값이 작을수록 더 적은 바이트를 사용합니다. 각 바이트는 다음으로 구성됩니다.

- **MSB(최상위 비트)**: 연속 비트. 1이면 다음 바이트가 더 있음, 0이면 마지막 바이트.
- **하위 7비트**: 실제 데이터.

**예: 150 인코딩**

```
150 → 10010110 00000001   (hex: 96 01)
```

해석 과정: 각 바이트에서 MSB를 제거하고, 리틀엔디언(little-endian) 그룹을 빅엔디언으로 뒤집어 이어 붙입니다.

```
0010110 0000001  →  0000001 0010110  →  00000010010110 = 150
```

## 메시지 구조: 태그-값 쌍

메시지는 필드-값 쌍의 나열입니다. 각 쌍은 다음으로 구성됩니다.

- **태그(varint)**: 필드 번호와 와이어 타입을 함께 인코딩
- **값**: 와이어 타입 규칙에 따라 인코딩

**태그 계산식**

```
tag = (field_number << 3) | wire_type
```

예: 필드 1번, VARINT(0) 타입 → `(1 << 3) | 0 = 8` (hex `08`).

---

## 와이어 타입

| 값 | 이름 | 사용 타입 |
|----|------|-----------|
| 0 | VARINT | int32, int64, uint32, uint64, sint32, sint64, bool, enum |
| 1 | I64 | fixed64, sfixed64, double |
| 2 | LEN | string, bytes, 임베디드 메시지, packed repeated 필드 |
| 5 | I32 | fixed32, sfixed32, float |

> 와이어 타입 3, 4는 더 이상 쓰지 않는 group(SGROUP/EGROUP)에 쓰였습니다.

## 부호 있는 정수와 ZigZag 인코딩

- **int32 / int64**: 2의 보수(two's complement)를 사용합니다. 음수는 항상 64비트로 취급되어 varint로 10바이트를 차지합니다(음수에 비효율적).
- **sint32 / sint64**: **ZigZag 인코딩**을 사용해 작은 절댓값의 음수를 효율적으로 표현합니다.

ZigZag 변환식:

```
sint32:  (n << 1) ^ (n >> 31)
sint64:  (n << 1) ^ (n >> 63)
```

매핑 예시: `0→0, -1→1, 1→2, -2→3, 2→4 ...` 처럼 0을 중심으로 양수/음수가 번갈아 작은 값에 매핑됩니다.

> 음수가 자주 나오는 필드에는 `sint32`/`sint64`를, 큰 값이 자주 나오면 고정 길이의 `fixed32`/`fixed64`를 고려하세요.

## 길이 구분 필드 (LEN)

string, bytes, 임베디드 메시지는 `LEN`(와이어 타입 2)을 사용하며 형식은 다음과 같습니다.

```
[태그: LEN] [길이(varint)] [페이로드 바이트]
```

**예: 필드 2번에 문자열 "testing"**

```
12 07 74 65 73 74 69 6e 67
```

- `12`: 태그 (필드 2, 와이어 타입 LEN → `(2<<3)|2 = 0x12`)
- `07`: 길이 7바이트
- 나머지: "testing"의 UTF-8 바이트

임베디드 메시지도 동일하게, 먼저 길이를 쓰고 그 뒤에 메시지 바이트를 둡니다.

---

## Packed repeated 필드

기본적으로(Edition 2023 이상, proto3 스칼라) 원시(primitive) repeated 필드는 하나의 `LEN` 레코드로 압축 인코딩됩니다.

```proto
message Test4 {
  repeated int32 e = 5;   // e = [1, 2, 3]
}
```

```
5: {1 2 3}    // 개별 레코드 3개가 아니라, 하나의 packed 레코드
```

즉 `[태그(LEN)] [전체 길이] [값1 값2 값3 ...]` 형태로 태그 반복을 줄여 크기를 절약합니다. 비스칼라(메시지) repeated 필드는 packed 대상이 아니며 요소마다 별도 레코드로 인코딩됩니다.

## 필드 순서와 마지막 값 우선 규칙

**필드 순서**

- 직렬화 순서는 구현에 따라 다르며 보장되지 않습니다.
- 파서는 어떤 순서의 필드든 받아들여야 합니다.
- 따라서 같은 메시지라도 직렬화 결과 바이트가 매번 동일하지 않을 수 있습니다.

**마지막 값 우선(Last-One-Wins)** — 비반복 필드가 와이어상에 여러 번 나타날 때:

- **스칼라**: 파서는 마지막 값을 채택합니다.
- **임베디드 메시지**: 인스턴스들을 병합(merge)합니다(나중 필드가 앞을 덮어씀).
- **repeated 필드**: 모두 이어 붙입니다(concatenate).

이 규칙 덕분에 메시지 병합을 단순 연결로 구현할 수 있습니다.

```
Parse(str1 + str2)  ==  Parse(str1); Parse(str2); Merge()
```
