# 1.1 코틀린 맛보기

```kotlin
data class Person(val name: String, val age: Int? = null)

fun main() {
    val persons = listOf(Person("Alice"), Person("Bob", age = 29))
    val oldest = persons.maxByOrNull { it.age ?: 0 }
    println("The oldest is: $oldest")
}
// 출력: The oldest is: Person(name=Bob, age=29)
```

- 코틀린은 자바 플랫폼에서 돌아가는 새로운 프로그래밍 언어
- 기존 자바 라이브러리나 프레임워크와 함께 잘 작동하며 성능도 자바와 같은 수준
- JetBrains에서 개발한 크로스 플랫폼 범용 프로그래밍 언어
- 2017년 5월, 구글에서 안드로이드 공식 개발 언어로 채택

# 1.2 코틀린의 주요 특성

## 1.2.1 코틀린 용례: 안드로이드, 서버, 자바가 실행되는 모든 곳, 그 외의 용도

- **서버 개발**: 웹 애플리케이션 백엔드, 마이크로서비스
- **안드로이드 개발**: 네이티브 모바일 앱 개발
- **데스크톱 애플리케이션**: JavaFX, Swing 등과 함께 사용
- **멀티플랫폼**: Kotlin/JS(브라우저, Node.js), Kotlin/Native(iOS, 임베디드)

## 1.2.2 정적 타입 지정으로 인해 코틀린 성능, 신뢰성, 유지 보수성이 모두 좋아짐

컴파일러가 문맥을 고려해 변수 타입을 결정하는 기능을 **타입 추론(type inference)**이라 부른다.

```kotlin
val x = 1  // Int 타입으로 추론
val name = "Kotlin"  // String 타입으로 추론
```

코틀린이 타입 추론을 제공한다는 말은 여러분이 직접 타입을 지정할 필요가 없어서 정적 타입 지정 언어에서 프로그래머가 직접 타입을 선언해야 함에 따라 생기는 불편함이 대부분 사라진다는 뜻이다.

**정적 타입 지정의 장점:**
- **성능**: 런타임에 타입을 결정할 필요가 없어 메서드 호출이 빠름
- **신뢰성**: 컴파일러가 프로그램의 정확성을 검증하여 런타임 오류 감소
- **유지 보수성**: 코드에서 다루는 객체가 어떤 타입인지 알 수 있어 코드 이해가 쉬움
- **도구 지원**: IDE에서 더 정확한 코드 완성, 리팩토링 제공

## 1.2.3 함수형 프로그래밍과 객체지향 프로그래밍의 조합이 코틀린을 안전하고 유연하게 한다

코틀린은 **널이 될 수 있는 타입(nullable type)**을 지원한다.

```kotlin
val notNull: String = "Hello"      // null 불가
val nullable: String? = null       // null 가능
```

**함수형 프로그래밍의 핵심 개념:**

1. **일급 시민인 함수(First-class functions)**
   - 함수를 변수에 저장하고, 파라미터로 전달하고, 반환값으로 사용 가능
   ```kotlin
   val sum = { x: Int, y: Int -> x + y }
   val result = listOf(1, 2, 3).map { it * 2 }
   ```

2. **불변성(Immutability)**
   - 일단 만들어지면 내부 상태가 절대 변하지 않는 불변 객체 사용
   ```kotlin
   data class User(val name: String, val age: Int)
   ```

3. **부수 효과 없음(No side effects)**
   - 입력이 같으면 항상 같은 출력, 외부 상태 변경 없음
   - 다중 스레드 환경에서도 안전

**함수형 스타일의 장점:**
- 간결성: 람다를 통한 코드 추상화
- 다중 스레드 안전성: 불변 데이터 구조와 순수 함수 사용
- 테스트 용이성: 부수 효과가 없어 독립적 테스트 가능

## 1.2.4 코루틴을 쓰면 동시성, 비동기 코드를 자연스럽고 구조적으로 사용할 수 있다

```kotlin
suspend fun fetchData(): String {
    delay(1000)  // 비동기 대기
    return "Data"
}

fun main() = runBlocking {
    val result = async { fetchData() }
    println(result.await())
}
```

- 코루틴은 경량 스레드로, 수천 개의 코루틴을 효율적으로 실행 가능
- 콜백 지옥 없이 비동기 코드를 동기 코드처럼 작성
- 구조화된 동시성으로 취소와 예외 처리가 용이

## 1.2.5 코틀린을 모든 목적에 사용할 수 있다. 코틀린은 오픈소스이며, 여러분의 기여에 대해 열려 있다

- Apache 2.0 라이선스로 무료 사용 가능
- GitHub에서 개발되며 커뮤니티 기여 환영
- IntelliJ IDEA, Android Studio 등 IDE 지원

## 1.3.3 다중 플랫폼: iOS, JVM, JS 및 그 외의 플랫폼에서 비즈니스 로직을 공유하고 코드 중복을 최소화하기

# 1.4 코틀린의 철학

## 1.4.1 코틀린은 실용적인 언어다

- 다른 프로그래밍 언어가 채택한 이미 성공적으로 검증된 해법과 기능에 의존
- 언어의 복잡도를 줄이고 이미 알고 있는 기존 개념을 통해 쉽게 배울 수 있음
- 특정 프로그래밍 스타일이나 패러다임을 강제하지 않음
- 자바에서 사용해온 익숙한 스타일과 기법을 그대로 활용 가능
- IDE와 함께 발전하여 뛰어난 도구 지원

## 1.4.2 코틀린은 간결하다

- 프로그래머가 작성하는 코드에서 의미가 없는 부분을 줄임
- getter, setter, 생성자 파라미터를 필드에 대입하기 위한 로직 등 준비 코드를 묵시적으로 제공

```java
// Java
public class Person {
    private String name;
    private int age;

    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() { return name; }
    public int getAge() { return age; }
}
```

```kotlin
// Kotlin - 동일한 기능을 한 줄로
data class Person(val name: String, val age: Int)
```

- 코드가 간결하면 읽기 쉽고, 유지보수가 용이하며, 생산성이 높아짐

## 1.4.3 코틀린은 안전하다

- **NullPointerException 방지**: 컴파일 시점에 null 검사를 수행
  ```kotlin
  val s: String? = null
  println(s.length)  // 컴파일 에러!
  println(s?.length) // 안전 호출, null 반환
  ```

- **ClassCastException 방지**: 스마트 캐스트로 타입 검사와 캐스트를 한 번에
  ```kotlin
  if (obj is String) {
      println(obj.length)  // 자동으로 String으로 캐스트
  }
  ```

- JVM 사용을 통해 메모리 안전성 보장, 버퍼 오버플로 방지

## 1.4.4 코틀린은 상호운용성이 좋다

- 자바 메서드 호출, 클래스 상속, 인터페이스 구현, 어노테이션 적용 모두 가능
- 기존 자바 라이브러리를 최대한 활용
- 자체 컬렉션 라이브러리를 제공하지 않고, 자바 표준 라이브러리 클래스에 의존
- 코틀린 컬렉션은 자바 컬렉션과 동일한 클래스이므로 변환 불필요

```kotlin
// 자바 라이브러리 직접 사용
import java.util.ArrayList

val list = ArrayList<String>()
list.add("Kotlin")
```

# 1.5 코틀린 도구 사용

## 1.5.1 코틀린 코드 설정과 실행

- **IntelliJ IDEA**: 코틀린 플러그인 기본 탑재
- **Android Studio**: 코틀린 완벽 지원
- **VS Code**: Kotlin 확장 프로그램 사용
- **명령줄**: kotlinc 컴파일러 사용

```bash
# Kotlin 컴파일러 설치 (SDKMAN 사용)
sdk install kotlin

# REPL 실행
kotlinc

# 스크립트 실행
kotlin script.kts
```

## 1.5.2 코틀린 코드 컴파일

```
[코틀린 소스코드] → [코틀린 컴파일러] → [.class 바이트코드] → [JVM 실행]
      .kt                kotlinc               .class              java
```

- 코틀린 컴파일러는 자바 바이트코드(.class)를 생성
- 생성된 바이트코드는 표준 자바 패키징 방식을 따름
- Gradle, Maven 등 빌드 도구와 통합 가능

```groovy
// build.gradle.kts (Kotlin DSL)
plugins {
    kotlin("jvm") version "1.9.0"
}

dependencies {
    implementation(kotlin("stdlib"))
}
```

# 요약

- 코틀린은 타입 추론을 지원하는 **정적 타입 지정 언어**로, 소스코드의 정확성과 성능을 보장하면서도 간결한 코드 작성이 가능
- **함수형 프로그래밍**과 **객체지향 프로그래밍**을 모두 지원
- **서버 애플리케이션**, **안드로이드 앱**, **멀티플랫폼** 개발에 적합
- 자바와 **100% 상호운용** 가능하며 기존 자바 코드베이스에 점진적으로 도입 가능
- **실용성, 간결성, 안전성, 상호운용성**이 코틀린의 핵심 철학
- **널 안전성**과 **스마트 캐스트**로 런타임 오류 방지
- **코루틴**으로 비동기 프로그래밍을 간단하게 처리
