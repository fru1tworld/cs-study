# 2.1 기본 요소: 함수와 변수

## 2.1.1 첫 번째 코틀린 프로그램 작성: Hello, World!

```kotlin
fun main() {
    println("Hello, world!")
}
```

- 함수를 선언할 때 `fun` 키워드 사용
- 함수를 모든 코틀린 파일의 **최상위 수준**에 정의할 수 있으므로 클래스 안에 함수를 넣어야 할 필요가 없음
- `main` 함수는 프로그램의 진입점
- 세미콜론(`;`)을 붙이지 않아도 됨

## 2.1.2 파라미터와 반환값이 있는 함수 선언

```kotlin
fun max(a: Int, b: Int): Int {
    return if (a > b) a else b
}
```

함수 선언 구조:
```
fun 함수이름(파라미터: 타입, ...): 반환타입 {
    // 함수 본문
}
```

### 문(statement)과 식(expression)의 구분

코틀린에서 **if는 식(expression)이지 문(statement)이 아니다.**

| 구분 | 설명 | 예시 |
|------|------|------|
| **식(Expression)** | 값을 만들어내며 다른 식의 하위 요소로 계산에 참여 | `if`, `when`, `try` |
| **문(Statement)** | 자신을 둘러싸고 있는 가장 안쪽 블록의 최상위 요소로 존재하며 값을 만들어내지 않음 | `for`, `while` |

코틀린에서는 **루프(for, while)를 제외한 대부분의 제어 구조가 식**이라는 점이 자바와 다르다.

```kotlin
// if 식
val x = if (myBoolean) 3 else 5

// when 식
val direction = when (inputString) {
    "u" -> UP
    "d" -> DOWN
    else -> UNKNOWN
}

// try 식
val number = try {
    inputString.toInt()
} catch (nfe: NumberFormatException) {
    -1
}
```

## 2.1.3 식 본문을 사용해 함수를 더 간결하게 정의

```kotlin
// 블록 본문 함수 (block body function)
fun max(a: Int, b: Int): Int {
    return if (a > b) a else b
}

// 식 본문 함수 (expression body function)
fun max(a: Int, b: Int): Int = if (a > b) a else b

// 타입 추론으로 반환 타입 생략 가능 (식 본문 함수만 가능)
fun max(a: Int, b: Int) = if (a > b) a else b
```

- 본문이 중괄호로 둘러싸인 함수: **블록 본문 함수(block body function)**
- 등호와 식으로 이뤄진 함수: **식 본문 함수(expression body function)**
- 식 본문 함수는 반환 타입을 생략할 수 있음 (타입 추론)
- 블록 본문 함수는 반환 타입과 return 문을 명시해야 함

## 2.1.4 데이터를 저장하기 위해 변수 선언

```kotlin
// 타입 추론
val question = "삶, 우주, 그리고 모든 것에 대한 궁극적인 질문"
val answer = 42

// 타입 명시
val answer: Int = 42

// 초기화 없이 선언 (타입 필수)
val answer: Int
answer = 42
```

- 코틀린은 **타입 추론**을 지원하므로 대부분의 경우 타입 지정 생략 가능
- 초기화 식이 없으면 변수 타입을 반드시 명시해야 함

## 2.1.5 변수를 읽기 전용 변수나 재대입 가능 변수로 표시

```kotlin
// val (value) - 변경 불가능한 참조, 자바의 final과 유사
val languages = arrayListOf("Java")
languages.add("Kotlin")  // OK: 참조하는 객체의 내부는 변경 가능
// languages = arrayListOf("Scala")  // 에러: 참조 자체는 변경 불가

// var (variable) - 변경 가능한 참조
var count = 0
count = 1  // OK: 값 변경 가능
// count = "hello"  // 에러: 타입은 변경 불가
```

| 키워드 | 의미 | 특성 |
|--------|------|------|
| `val` | 불변 참조 (immutable reference) | 초기화 후 재대입 불가, 자바의 `final`과 유사 |
| `var` | 가변 참조 (mutable reference) | 값 변경 가능, 단 타입은 고정 |

**권장사항**: 기본적으로 모든 변수를 `val`로 선언하고, 나중에 꼭 필요할 때만 `var`로 변경

### val의 조건부 초기화

```kotlin
val message: String
if (canPerformOperation()) {
    message = "Success"
} else {
    message = "Failed"
}
```

`val` 참조 자체는 불변이지만, 블록 실행 시 조건에 따라 다른 값으로 초기화될 수 있다.

## 2.1.6 더 쉽게 문자열 형식 지정: 문자열 템플릿

```kotlin
val name = "Kotlin"
println("Hello, $name!")  // Hello, Kotlin!

// 복잡한 식은 중괄호로 감싸기
println("Hello, ${name.uppercase()}!")  // Hello, KOTLIN!
println("1 + 2 = ${1 + 2}")  // 1 + 2 = 3

// $ 문자 자체를 출력하려면 이스케이프
println("\$100")  // $100
```

- `$변수명`: 변수 값을 문자열에 삽입
- `${표현식}`: 복잡한 표현식을 문자열에 삽입
- `\$`: 달러 기호 자체를 출력

**주의**: `$`로 변수를 지정할 때 변수명 바로 뒤에 한글을 붙여 사용하면 코틀린 컴파일러는 영문자와 한글을 한꺼번에 식별자로 인식해서 unresolved reference 오류를 발생시킨다.

```kotlin
val name = "코틀린"
println("$name이 좋아요")     // 에러! name이 아닌 name이를 찾음
println("${name}이 좋아요")   // OK: 코틀린이 좋아요
```

# 2.2 행동과 데이터 캡슐화: 클래스와 프로퍼티

자바에서는 필드와 접근자를 한데 묶어 **프로퍼티(property)**라고 부르며, 프로퍼티의 개념을 활용하는 프레임워크가 많다.

## 2.2.1 클래스와 데이터를 연관시키고, 접근 가능하게 만들기: 프로퍼티

```java
// Java - 전형적인 자바빈 클래스
public class Person {
    private final String name;
    private boolean isMarried;

    public Person(String name, boolean isMarried) {
        this.name = name;
        this.isMarried = isMarried;
    }

    public String getName() { return name; }
    public boolean isMarried() { return isMarried; }
    public void setMarried(boolean married) { isMarried = married; }
}
```

```kotlin
// Kotlin - 동일한 기능을 간결하게
class Person(
    val name: String,        // 읽기 전용 프로퍼티: 비공개 필드 + getter
    var isMarried: Boolean   // 쓰기 가능 프로퍼티: 비공개 필드 + getter + setter
)
```

**프로퍼티 접근:**
```kotlin
val person = Person("Bob", true)
println(person.name)       // getter 호출
println(person.isMarried)  // getter 호출
person.isMarried = false   // setter 호출
```

- 코틀린은 `val`로 선언된 프로퍼티에 대해 getter를 자동 생성
- `var`로 선언된 프로퍼티에 대해 getter와 setter를 자동 생성
- 자바에서는 `person.getName()`, 코틀린에서는 `person.name`으로 접근

## 2.2.2 프로퍼티 값을 저장하지 않고 계산: 커스텀 접근자

```kotlin
class Rectangle(val height: Int, val width: Int) {
    // 커스텀 getter - 호출할 때마다 계산
    val isSquare: Boolean
        get() = height == width
}

val rectangle = Rectangle(41, 43)
println(rectangle.isSquare)  // false
```

**파라미터 없는 함수 vs 커스텀 getter:**
```kotlin
// 둘 다 동일하게 동작
fun isSquare(): Boolean = height == width  // 함수
val isSquare: Boolean get() = height == width  // 프로퍼티

// 권장: 클래스의 특성을 나타내면 프로퍼티, 동작을 나타내면 함수
```

## 2.2.3 코틀린 소스코드 구조: 디렉터리와 패키지

```kotlin
// 패키지 선언
package geometry.shapes

import java.util.Random

class Rectangle(val height: Int, val width: Int) {
    val isSquare: Boolean get() = height == width
}

// 최상위 함수
fun createRandomRectangle(): Rectangle {
    val random = Random()
    return Rectangle(random.nextInt(), random.nextInt())
}
```

- 패키지 안에 클래스, 함수, 프로퍼티를 직접 선언 가능
- 자바와 달리 여러 클래스를 한 파일에 넣을 수 있고, 파일 이름과 클래스 이름이 달라도 됨
- 그러나 자바 관례를 따르는 것이 권장됨 (특히 자바와 혼용할 때)

# 2.3 선택 표현과 처리: 이넘과 when

## 2.3.1 이넘 클래스와 이넘 상수 정의

```kotlin
// 단순한 enum 클래스
enum class Color {
    RED, ORANGE, YELLOW, GREEN, BLUE, INDIGO, VIOLET
}

// 프로퍼티와 메서드가 있는 enum 클래스
enum class Color(
    val r: Int, val g: Int, val b: Int  // 상수의 프로퍼티 정의
) {
    RED(255, 0, 0),
    ORANGE(255, 165, 0),
    YELLOW(255, 255, 0),
    GREEN(0, 255, 0),
    BLUE(0, 0, 255);  // 반드시 세미콜론 필요!

    fun rgb() = (r * 256 + g) * 256 + b  // enum 클래스 안에서 메서드 정의
}

println(Color.BLUE.rgb())  // 255
```

- `enum`은 **소프트 키워드(soft keyword)**이다. `class` 앞에 있을 때만 특별한 의미를 가짐
- enum 클래스에서 상수 목록과 메서드 정의 사이에 **반드시 세미콜론(`;`)**을 사용해야 함
- 코틀린에서 유일하게 세미콜론이 필수인 곳

## 2.3.2 when으로 이넘 클래스 다루기

```kotlin
fun getMnemonic(color: Color) =
    when (color) {
        Color.RED -> "Richard"
        Color.ORANGE -> "Of"
        Color.YELLOW -> "York"
        Color.GREEN -> "Gave"
        Color.BLUE -> "Battle"
        Color.INDIGO -> "In"
        Color.VIOLET -> "Vain"
    }

println(getMnemonic(Color.BLUE))  // Battle
```

- 자바의 `switch`와 달리 `break`를 넣지 않아도 됨
- 한 분기에 여러 값을 매치시킬 수 있음:

```kotlin
fun getWarmth(color: Color) = when (color) {
    Color.RED, Color.ORANGE, Color.YELLOW -> "warm"
    Color.GREEN -> "neutral"
    Color.BLUE, Color.INDIGO, Color.VIOLET -> "cold"
}
```

## 2.3.3 when식의 대상을 변수에 포획

```kotlin
fun mix(c1: Color, c2: Color) =
    when (setOf(c1, c2)) {
        setOf(Color.RED, Color.YELLOW) -> Color.ORANGE
        setOf(Color.YELLOW, Color.BLUE) -> Color.GREEN
        else -> throw Exception("Dirty color")
    }
```

## 2.3.4 when의 분기 조건에 임의의 객체 사용

```kotlin
fun mixOptimized(c1: Color, c2: Color) =
    when {
        (c1 == Color.RED && c2 == Color.YELLOW) ||
        (c1 == Color.YELLOW && c2 == Color.RED) -> Color.ORANGE
        else -> throw Exception("Dirty color")
    }
```

- `when`의 분기 조건으로 **임의의 객체**를 허용
- 분기 조건에 객체 사이의 **동등성(equality)**을 사용

## 2.3.5 인자 없는 when 사용

```kotlin
fun mixOptimized(c1: Color, c2: Color) =
    when {
        (c1 == Color.RED && c2 == Color.YELLOW) -> Color.ORANGE
        (c1 == Color.YELLOW && c2 == Color.BLUE) -> Color.GREEN
        else -> throw Exception("Dirty color")
    }
```

- when에 아무 인자도 없으면 각 분기의 조건이 **불리언 결과를 계산하는 식**이어야 함
- 추가 객체를 만들지 않아 성능상 이점이 있을 수 있음

## 2.3.6 스마트 캐스트: 타입 검사와 타입 캐스트 조합

```kotlin
interface Expr
class Num(val value: Int) : Expr
class Sum(val left: Expr, val right: Expr) : Expr

fun eval(e: Expr): Int {
    if (e is Num) {
        val n = e as Num  // 불필요한 캐스트 (스마트 캐스트됨)
        return n.value
    }
    if (e is Sum) {
        return eval(e.right) + eval(e.left)  // 스마트 캐스트
    }
    throw IllegalArgumentException("Unknown expression")
}
```

- `is`를 사용해 변수 타입을 검사 (자바의 `instanceof`와 유사)
- 타입 검사 후에 컴파일러가 자동으로 캐스팅 수행: **스마트 캐스트(smart cast)**
- 스마트 캐스트는 `val` 변수에만 적용 (var이거나 커스텀 접근자가 있으면 불가)
- 명시적 캐스트: `as` 키워드 사용

```kotlin
val n = e as Num  // 명시적 캐스트 (e가 Num이 아니면 ClassCastException)
```

## 2.3.7 리팩터링: if를 when으로 변경

```kotlin
// if 식 사용
fun eval(e: Expr): Int =
    if (e is Num) {
        e.value
    } else if (e is Sum) {
        eval(e.right) + eval(e.left)
    } else {
        throw IllegalArgumentException("Unknown expression")
    }

// when 식으로 리팩터링
fun eval(e: Expr): Int =
    when (e) {
        is Num -> e.value
        is Sum -> eval(e.right) + eval(e.left)
        else -> throw IllegalArgumentException("Unknown expression")
    }
```

- `when` 분기에서 **타입 검사와 스마트 캐스트**가 함께 작동

## 2.3.8 if와 when의 분기에서 블록 사용

```kotlin
fun evalWithLogging(e: Expr): Int =
    when (e) {
        is Num -> {
            println("num: ${e.value}")
            e.value  // 블록의 마지막 식이 반환값
        }
        is Sum -> {
            val left = evalWithLogging(e.left)
            val right = evalWithLogging(e.right)
            println("sum: $left + $right")
            left + right  // 블록의 마지막 식이 반환값
        }
        else -> throw IllegalArgumentException("Unknown expression")
    }
```

- 블록이 본문인 경우 **블록의 마지막 식이 블록 전체의 결과**
- 이 규칙은 함수에는 적용되지 않음 (함수는 return 필요)

# 2.4 대상 이터레이션: while과 for 루프

## 2.4.1 조건이 참인 동안 코드 반복: while 루프

```kotlin
// while 루프
while (조건) {
    // 조건이 참인 동안 실행
}

// do-while 루프
do {
    // 본문을 최소 한 번 실행
} while (조건)
```

- 자바의 while, do-while과 동일하게 동작
- 코틀린은 이들을 **문(statement)**으로 취급 (식이 아님)

## 2.4.2 수에 대해 이터레이션: 범위와 순열

```kotlin
// 범위 (range) - 폐구간, 양 끝 포함
val oneToTen = 1..10  // 1, 2, 3, ..., 10

// for 루프로 범위 이터레이션
for (i in 1..100) {
    print("$i ")  // 1 2 3 ... 100
}

// 역방향 범위
for (i in 100 downTo 1) {
    print("$i ")  // 100 99 98 ... 1
}

// step으로 증가값 지정
for (i in 1..100 step 2) {
    print("$i ")  // 1 3 5 7 ... 99 (홀수만)
}

// until - 끝 값 제외 (반폐구간)
for (i in 0 until 10) {
    print("$i ")  // 0 1 2 ... 9 (10 제외)
}
// 위와 동일: for (i in 0..<10)
```

**FizzBuzz 예제:**
```kotlin
for (i in 1..100) {
    println(
        when {
            i % 15 == 0 -> "FizzBuzz"
            i % 3 == 0 -> "Fizz"
            i % 5 == 0 -> "Buzz"
            else -> "$i"
        }
    )
}
```

val binaryReps = mutableMapOf<Char, String>()
for (c in 'A'..'F') {

## 2.4.4 in으로 컬렉션이나 범위의 원소 검사

```kotlin
// in으로 범위에 속하는지 검사
fun isLetter(c: Char) = c in 'a'..'z' || c in 'A'..'Z'
fun isNotDigit(c: Char) = c !in '0'..'9'

println(isLetter('q'))    // true
println(isNotDigit('x'))  // true

// when에서 in 사용
fun recognize(c: Char) = when (c) {
    in '0'..'9' -> "It's a digit!"
    in 'a'..'z', in 'A'..'Z' -> "It's a letter!"
    else -> "I don't know..."
}
```

```kotlin
// Comparable 구현 객체에도 범위 사용 가능
println("Kotlin" in "Java".."Scala")  // true (알파벳 순 비교)
println("Kotlin" in setOf("Java", "Scala"))  // false (컬렉션에 포함 여부)
```

# 2.5 코틀린에서 예외 던지고 잡아내기

## 2.5.1 try, catch, finally를 사용한 예외 처리와 오류 복구

```kotlin
fun readNumber(reader: BufferedReader): Int? {
    try {
        val line = reader.readLine()
        return Integer.parseInt(line)
    } catch (e: NumberFormatException) {
        return null
    } finally {
        reader.close()
    }
}
```

**코틀린 예외 처리의 특징:**
- `throw`는 **식**이므로 다른 식에 포함될 수 있음:
  ```kotlin
  val percentage =
      if (number in 0..100) number
      else throw IllegalArgumentException("...")
  ```
- 코틀린은 **검사 예외(checked exception)**와 **비검사 예외(unchecked exception)**를 구분하지 않음
- 함수가 던지는 예외를 선언할 필요 없음 (throws 절 없음)
- 자바와 달리 예외를 반드시 처리하도록 강제하지 않음

## 2.5.2 try를 식으로 사용

```kotlin
fun readNumber(reader: BufferedReader) {
    val number = try {
        Integer.parseInt(reader.readLine())
    } catch (e: NumberFormatException) {
        null  // 예외 발생 시 null 반환
    }
    println(number)
}
```

```kotlin
// catch에서 return 사용
fun readNumber(reader: BufferedReader) {
    val number = try {
        Integer.parseInt(reader.readLine())
    } catch (e: NumberFormatException) {
        return  // 예외 발생 시 함수 종료
    }
    println(number)
}
```

- `try`는 **식**이므로 결과를 변수에 대입 가능
- try 블록의 마지막 식이 결과값
- catch 블록도 값을 반환해야 함 (또는 return/throw로 빠져나가기)

# 요약

- **함수**: `fun` 키워드로 선언, 최상위 수준에 정의 가능
- **변수**: `val`(불변), `var`(가변). 타입 추론 지원
- **문자열 템플릿**: `$변수명`, `${표현식}`으로 문자열 내 값 삽입
- **클래스**: 간결한 프로퍼티 선언, 커스텀 접근자 지원
- **when**: 자바 switch보다 강력, 임의 객체 비교 가능, 스마트 캐스트와 결합
- **for 루프**: `for (x in collection)`, 범위(`..`, `until`, `downTo`, `step`)
- **in 연산자**: 범위나 컬렉션에 속하는지 검사, `!in`으로 부정
- **예외 처리**: 검사/비검사 예외 구분 없음, try는 식으로 사용 가능
- **스마트 캐스트**: `is`로 타입 검사 후 자동 캐스트
- 코틀린은 대부분의 제어 구조가 **식(expression)**이라는 점이 자바와 다름
